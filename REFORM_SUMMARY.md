## 核心洞察

### 为什么需要这次改造？

你当前的架构已经可用，但面临以下挑战：

1. **状态管理混乱**
   - 全局变量分散在各处
   - 状态变化难以追踪
   - 缺乏观察者通知机制
   - 难以测试

2. **工具接口不统一**
   - 每个工具有不同的方法签名
   - 权限检查各自为政
   - 新增工具需要理解整个系统
   - UI 渲染逻辑重复

3. **权限防线单薄**
   - 权限检查位置不统一
   - 缺乏明确的防线顺序
   - 审计日志不完整
   - 易遗漏危险操作

4. **Agent 系统刚性**
   - 内置 Agent 硬编码
   - 无法加载自定义 Agent
   - 多 Agent 协作时上下文污染
   - 工具集无法动态限制

5. **上下文管理被动**
   - Token 溢出时被动删除消息
   - 无法预测成本
   - 消息重要性难以判断
   - 频繁超出限制导致重试

### Claude Code 怎么解决的？

Claude Code 是一个 1900+ 文件的生产级产品，它采用了 7 个核心设计模式来解决这些问题：

1. **三层状态架构**
   - Bootstrap: Session 级全局状态（最小化）
   - AppState: 应用级共享状态（Store 模式）
   - ToolContext: 工具执行上下文（单次隔离）

2. **统一 Tool 协议**
   - 所有工具实现同一接口
   - Builder 模式简化实现
   - 完整的生命周期（验证→权限→执行→渲染）

3. **工具注册表**
   - 单一来源（getAllBaseTools）
   - 三层条件注册（编译期 + 加载期 + 运行时）
   - 支持 40+ 内置工具 + MCP 无限外部工具

4. **多源 Agent 加载**
   - 内置 Agent
   - 自定义 Agent（.claude/agents/*.md）
   - 插件 Agent（plugin/）
   - 优先级明确（后者覆盖前者）

5. **Context 隔离**
   - 每个 Agent 有独立的上下文
   - 可生成子 Agent
   - 工具集可动态限制
   - 记忆系统多维度

6. **Token 预算 + 智能压缩**
   - 清晰的预算分配（system/tools/history/context）
   - 主动压缩策略（删除 + 摘要）
   - 可预测的 token 消耗

7. **多层防线权限**
   - 输入验证（validate_input）
   - 权限检查（check_permissions）
   - 资源限制（rate_limit/quota）
   - 审计日志（complete logging）

## 改造方案总体设计

### 分层架构

改造后的项目将采用 **4 层分层架构**，清晰的依赖方向：

```
第 4 层: 表现层（API + CLI）
       ↓ 依赖
第 3 层: 应用逻辑（ContextController + AgentManager）
       ↓ 依赖
第 2 层: 核心抽象（Store + Tool + Agent + TokenBudget）
       ↓ 依赖
第 1 层: 基础设施（Bootstrap + Config + Permission）
```

**关键特性**：
- ✅ DAG 依赖（无循环）
- ✅ 清晰的职责边界
- ✅ 易于测试（可 mock 任何层）
- ✅ 易于扩展（新工具/Agent 只需遵循协议）

### 核心文件清单

**第 1 层（基础设施）**：
- `backend/state/bootstrap.py` - Session 级全局状态
- `backend/system/config_system.py` - 6 层配置合并
- `backend/system/permission_system.py` - 权限防线

**第 2 层（核心抽象）**：
- `backend/state/app_state.py` - AppState Store（35 行）
- `backend/tools/tool.py` - 统一 Tool 接口
- `backend/tools/registry.py` - Tool 注册表
- `backend/graph/agent_definition.py` - Agent 定义
- `backend/context_management/token_budget.py` - Token 预算
- `backend/context_management/context_compactor.py` - 智能压缩

**第 3 层（应用逻辑）**：
- `backend/context_management/context_controller.py` （修改）
- `backend/graph/agent.py` （修改）
- `backend/skill_system/registry.py` （修改）

**第 4 层（表现层）**：
- `backend/api/chat.py` （修改）
- CLI 相关模块 （新增或修改）
