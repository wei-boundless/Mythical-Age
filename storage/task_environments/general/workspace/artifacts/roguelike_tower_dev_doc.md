# 五层地下塔 v2 开发设计文档

## 1. 游戏总览与设计目标

### 1.1 游戏简介
《五层地下塔》是一款基于网页的单人回合制 Roguelike 地牢探索游戏。玩家通过方向键在随机生成的 ASCII 风格地牢中移动，击败敌人、拾取物品、规避陷阱并逐层深入，最终抵达第五层的楼梯以获取胜利。v2 版本在保留原有核心玩法的基础上，新增强大的主动技能系统、随机事件机制、战斗视觉反馈、小地图迷雾探索和本地存档功能，大幅提升了游戏的策略深度与可重玩性。

### 1.2 设计目标
- **零依赖纯前端运行**：单个 HTML 文件包含所有 CSS 和 JavaScript，可直接在浏览器中打开，无需服务器或构建工具。
- **回合制战术体验**：每一步移动都触发一次完整的回合结算，包括冷却减少、敌人行动、状态更新等。
- **渐进难度曲线**：随楼层提升，敌人数量和属性线性增长，玩家可通过拾取物品和祭坛祝福来强化自身。
- **丰富策略选择**：三种主动技能（火焰冲击/治疗/护盾）各有冷却时间和战术价值，玩家需根据战况灵活选择。
- **完整存档支持**：结合自动存档与手动存档，利用 localStorage 持久化游戏进度，支持死亡后读档重试。

## 2. 系统架构与数据结构

### 2.1 全局游戏状态 (`game` 对象)
```javascript
game = {
  floor: 1,               // 当前楼层 (1–5)
  map: [],                // 21×15 二维字符数组 ('#' 或 '.')
  explored: [],           // 21×15 二维布尔数组，已探索为 true
  player: {
    x, y, hp, maxHp, atk, def, exp
  },
  enemies: [              // 敌人数组
    { x, y, name, hp, maxHp, atk, def, exp }
  ],
  items: [                // 物品数组
    { x, y, name, effect } // effect ∈ {heal, atk, def, exp}
  ],
  traps: [
    { x, y }
  ],
  shrines: [
    { x, y, used: bool }
  ],
  chestMimics: [
    { x, y, hp, atk, def, exp }
  ],
  skills: {
    fire:  { cooldown: 0, maxCd: 6 },
    heal:  { cooldown: 0, maxCd: 8 },
    shield:{ cooldown: 0, maxCd: 10, active: 0 }
  },
  selectedSkill: 'fire',
  stairPos: { x, y } | null,
  msgLog: [],
  turn: 0,
  gameOver: false,
  victory: false
}
```

### 2.2 地图生成系统
使用 **BSP 风格随机房间+走廊连接** 算法：
1. 生成 21×15 全墙地图。
2. 尝试 25 次随机放置 3×3 至 7×5 的房间，检查与已有房间的碰撞（含 1 格边距）。
3. 若有效房间少于 3 个，提供保底中心房间。
4. 按房间顺序用 L 形走廊连接相邻房间，并在房间数 ≥3 时封闭回路（连接首尾房间）。
5. 在房间内随机放置实体（敌人、物品、陷阱、祭坛、宝箱怪），避开玩家出生点和楼梯位置。

### 2.3 战斗系统
- **玩家攻击**：`damage = max(1, player.atk - enemy.def)`。
- **敌人反击**：在同一回合内进行，伤害 = `max(1, enemy.atk - player.def)`。
- **护盾减伤**：若护盾激活，敌人反击伤害额外 -2（最低 1）。
- **致死判定**：HP ≤ 0 时触发 `playerDeath()`，设置 `gameOver = true`，UI 半透明并提示读档。

### 2.4 主动技能系统
| 技能 | 图标 | 冷却 | 效果 |
|------|------|------|------|
| 火焰冲击 | 🔥 | 6 回合 | 对玩家周围 3×3 区域所有敌人造成 `atk+2 - def` 伤害 |
| 治疗术 | 💚 | 8 回合 | 恢复 6–12 点生命值，不超最大 HP |
| 护盾 | 🛡️ | 10 回合 | 激活 2 回合减伤状态，敌人伤害 -2 |

- **释放方式**：空格键释放当前选中技能；数字键 1/2/3 或点击技能槽切换。
- **冷却 UI**：技能冷却时图标变灰并显示剩余回合数，冷却完毕显示 "✓"。

### 2.5 随机事件系统
- **陷阱房**：玩家踩上触发 3–7 点伤害（护盾可减免），陷阱消失。
- **祝福祭坛**：踩上触发随机祝福（ATK 永久 +1~2、DEF 永久 +1、最大 HP 永久 +4），祭坛每个仅可使用一次。
- **宝箱怪**：伪装成物品 "$"，玩家踩上时触发战斗，属性随楼层提升。

### 2.6 小地图/迷雾系统
- 使用 Canvas 绘制 120×120 像素迷你地图。
- **已探索区域**以亮色绘制，未探索区域以深黑色覆盖。
- 玩家位置显示为蓝色方块，楼梯（若已探索）显示为橙色方块。
- 探索逻辑：玩家每步揭示周围 3×3 区域（含自身格）。

### 2.7 本地存档系统
- **自动存档**：每次踏上楼梯进入下一层前调用 `autoSave()`，无 UI 提示。
- **手动存档**：点击"💾 手动存档"按钮调用 `saveGame()`，写入 localStorage 并显示提示。
- **读档**：点击"📂 读档"按钮调用 `loadGame()`，从 localStorage 读取并恢复完整游戏状态，包括楼层、地图、探索度、所有实体、技能冷却、回合计数，并显示存档时间戳。
- **死亡恢复**：`gameOver` 后仍可读档，读档会清除 `gameOver` 和 `victory` 状态。

### 2.8 战斗视觉反馈
- **闪白效果**：攻击方与被攻击方所在格子短暂添加 `flash_white` CSS 类（150ms 后移除）。
- **浮动伤害数字**：通过 CSS `@keyframes floatUp` 动画实现从受击位置向上飘移并淡出的红色（伤害）或绿色（治疗）数字。

## 3. UI/UX 说明

### 3.1 布局结构
- 左侧：21×15 地牢网格（CSS Grid，每格 24px），背景深色。
- 右侧面板：状态栏 → 技能栏 → 小地图 → 消息日志 → 按钮行。
- 移动端（≤700px 宽）：面板移至下方，格子缩小至 20px。

### 3.2 颜色与符号映射
| 符号 | 类型 | 背景色 | 文字色 |
|------|------|--------|--------|
| # | 墙 | #0d0d1a | #222 |
| . | 地面 | #1e1e32 | #666 |
| @ | 玩家 | #2a4a6a | #8cf |
| E | 敌人 | #4a1a1a | #f66 |
| $ | 物品/宝箱怪 | #1a4a1a | #8f8 |
| > | 楼梯 | #3a2a1a | #fa0 |
| ! | 陷阱 | #2a1a3a | #f0f |
| ✝ | 祭坛 | #1a3a3a | #0ff |

### 3.3 消息系统
- 消息存储在 `msgLog` 数组中，最新消息显示在顶部。
- 支持多种消息样式：`msg-damage`（红）、`msg-heal`（绿）、`msg-info`（蓝）、`msg-warn`（橙）、`msg-loot`（黄）。

## 4. 数值平衡思路

### 4.1 玩家成长
- 初始属性：HP 30, ATK 8, DEF 2。
- 通过拾取物品获得：HP+8、ATK+1、DEF+1、EXP+5。
- 通过祭坛获得：ATK +1~2、DEF +1、MaxHP +4（含当前 HP）。
- EXP 目前为累积数值，未设计升级机制，但预留为未来扩展。

### 4.2 敌人成长
| 楼层 | 骷髅 HP | 史莱姆 HP | 暗影 HP | ATK 基准 | DEF 基准 |
|------|---------|-----------|---------|----------|----------|
| 1 | 8 | 6 | 10 | 3/2/5 | 1/0/2 |
| 3 | 12 | 10 | 14 | 6/5/8 | 不变 |
| 5 | 16 | 14 | 18 | 8/7/10 | 不变 |

敌人数量 = 3–6 + floor，保证后期更多遭遇。

### 4.3 技能冷却平衡
- 火焰冲击 (6 CD)：AOE 清场，适合密集敌人。
- 治疗 (8 CD)：续航保障，高楼层更重要。
- 护盾 (10 CD)：高风险战斗的减伤手段。

## 5. 代码组织说明

整个游戏为单一 HTML 文件，采用**自执行初始化模式**：
1. `<style>` 块定义所有 CSS（约 150 行）。
2. `<div id="game-shell">` 定义 DOM 结构。
3. `<script>` 块包含完整游戏逻辑（约 400 行），函数按以下顺序排列：
   - 常量与全局状态声明
   - 工具函数（rng, clamp, addMsg, renderMsgLog）
   - 地图生成（generateDungeon, carveCorridor, placeEntities）
   - 楼层控制（generateFloor）
   - UI 渲染（renderDungeon, renderStatus, renderSkills, renderMiniMap）
   - 战斗动画（flashCell, spawnDamageFloat）
   - 技能系统（reduceCooldowns, useSkill）
   - 移动与回合（handlePlayerMove, combat, endTurn, playerDeath）
   - 存档系统（saveGame, autoSave, loadGame）
   - 重启（restartGame）
   - 初始化与事件绑定（initGame, keydown 监听, click 监听）

## 6. 已知限制

1. **无音效/背景音乐**：纯视觉体验。
2. **无升级/等级系统**：EXP 仅为累积数值，未设计等级成长或技能树。
3. **敌人无 AI 寻路**：敌人固定位置，不会主动移动或追击玩家。
4. **地图连通性保证有限**：25 次房间尝试在极低概率下可能只生成少数房间，但保底机制降低了风险。
5. **存档大小限制**：依赖 localStorage，单次存档约 5–10KB，浏览器限制通常为 5–10MB，足够使用。
6. **无跨设备同步**：存档仅存储在当前浏览器中。
7. **移动端体验**：虽支持响应式布局，但格子较小，手指操作不如键盘精确。

## 7. 未来扩展方向

1. **等级与技能树**：利用 EXP 设计升级系统，解锁新技能或强化现有技能。
2. **敌人 AI**：实现简单的巡逻或追击行为。
3. **装备系统**：武器、防具、饰品槽位，提供更多 Build 组合。
4. **BOSS 战**：每层楼梯前设置守关 BOSS。
5. **更多楼层/无尽模式**：超越 5 层的无尽挑战。
6. **音效与粒子特效**：增强打击感和氛围。
7. **成就系统**：基于击杀数、收集数等触发成就。
8. **多人排行榜**：结合后端服务实现楼层/分数排行。
