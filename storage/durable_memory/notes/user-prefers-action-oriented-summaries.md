## Canonical Memory
用户在处理报告分析任务时，明确要求将结论压成带行动动词的行动建议，偏好面向业务/管理层的简洁、可执行输出格式。此外，用户要求复杂问题结论前置再补依据，称呼用户为'岩'，信息不足时先明确说缺什么不直接猜，而是基于已有证据给出最稳妥判断并标注可靠程度，同时指出优先补充哪块信息。

## Why Stored
信息不足时的三步处理法是用户明确提出的交互约定，与已有的结论前置、称呼偏好属于同一组稳定偏好，应合并保存以便跨会话一致执行

## How To Apply
以后遇到信息不足的情况，按三步执行：1) 明确告诉用户缺哪类信息及原因；2) 基于已有证据给出当前最稳妥判断并标注可靠程度；3) 指出优先补充哪块信息最划算。不要停留在'再看看''不好说'等悬空表态上。

## Source Evidence
用户问'如果信息不足，你应该怎么处理？'，助手回答：'先说缺什么，不直接猜。具体做法是三步：1. 明确缺口——直接告诉你当前缺少哪一类信息，以及为什么缺了它就没法下判断。2. 给出当前最稳妥的判断——即使信息不全，也不停在'再看看''不好说'这种悬空表态上。基于已有证据，给出当下最可取的落点，同时标注这个判断的可靠程度。3. 指出下一步该先确认什么——让你知道优先补哪块信息最划算，而不是把缺口摊开就结束。'
