# AGENTS.md instructions for D:\AI应用\langchain-agent

## 固定本地节点

- 本项目本地运行节点固定为：
  - 前端 Next.js：`http://127.0.0.1:3000`
  - 后端 FastAPI / Uvicorn：`http://127.0.0.1:8003`
  - 前端 API Base：`http://127.0.0.1:8003/api`
- 除非用户明确要求，不允许临时切换到 `3001`、`3002`、`8002`、`8004`、`8007` 或其他随机端口。
- 如果固定端口被占用，必须先查明占用进程并关闭本项目旧进程；不要为了省事换端口。
- 前端掉线、请求失败、SSE 断开、监控断开时，第一检查项必须是：
  - `3000` 是否只有一个前端进程监听。
  - `8003` 是否只有一个后端进程监听。
  - 前端实际请求目标是否为 `http://127.0.0.1:8003/api`。
- Electron、浏览器直开、Next 代理和前端配置必须统一指向上述固定节点；不要同时维护多套默认端口。
- 前端开发服务必须从干净的 `.next` 启动。遇到 `Cannot find module ./.next/server/...`、旧导入残留、页面热更新异常时，不要换端口，先关闭项目旧前端进程并用固定 `3000` 重新启动。

## 项目协作规则

- 除非用户明确要求，否则没有用处的旧残留代码请严格清理，严禁以兼容、兜底为理由保留旧代码和旧的逻辑结构。包括旧的测试文件也不要留着。
- 如果对代码或者结构有较大的改动，必须先写一个计划书，然后根据计划书严格实施；除非遇到严重问题，否则要一次性将计划书实施结束，不可以只停在某一阶段，阶段计划只可以用来检测计划是否跑偏。
- 如果用户要求重构，请认真考虑重构方案，禁止在旧壳上堆新壳。遇到问题，先考虑是不是结构性问题，少做补丁式的修改，尤其不能单独针对特定问题来填补漏洞。
- 前端设计时，根据架构分层，不同层级用卡片按钮做页面切换，不要将不同层级的页面混在一页。不要把开发式的语句当作副标题。
- 不允许自己产出结果来应付测试，也禁止用任何方法绕过测试，必须确保测试的功能能够真实通过。
- 当要用到浏览器时，请使用本地 Edge 浏览器。
- 对用户提出的功能要求，请严格按照高标准来做，禁止用差不多的理由选择降级方案。

## Agent Prompt 规则

- 时刻记住本项目是在做 agent。给 agent 的 prompt 必须让 agent 理解自己的角色、职责、边界和裁决要求，不要把开发说明当作 prompt 发给 agent。
- 不要写成：

```text
这是 runtime 节点。
根据任务图执行 world_review。
这个节点用于校验资产。
```

- 应该写成：

```text
你是一名世界观审核员。
你只负责评审当前世界观设定是否完整、一致、可支撑后续写作。
你不负责替创作者扩写设定。
你需要指出问题、给出裁决、说明是否允许进入下一阶段。
```
