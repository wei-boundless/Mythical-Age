# 前端美化与灵魂系统上架计划书

> 日期：2026-04-24
> 状态：执行中
> 目标：在不破坏现有聊天、会话、Inspector、RAG 开关等工作链路的前提下，完成前端整体视觉升级，并将四灵魂切换系统正式上架到前端。

---

## 1. 本轮改造目标

本轮前端改造只做三件事：

1. 重做整体视觉风格，使工作台从“可用面板”升级为“有明确世界观和品牌感的本地 Agent 工作台”。
2. 将灵魂系统正式接入前端，支持河伯、四岳、祝融、玄女四种人格的浏览、切换和当前生效状态展示。
3. 为四个灵魂补上可直接进入工程的角色形象资源，并接入界面关键展示位。

本轮不做：

1. 重写聊天协议或 SSE reducer。
2. 大改后端提示词装配链。
3. 重新设计 Inspector 的编辑能力。
4. 在本轮引入复杂动画库或三维渲染。

---

## 2. 当前现状

当前前端具备完整工作台骨架：

1. `page.tsx` 负责三栏布局。
2. `Navbar / Sidebar / ChatPanel / InspectorPanel` 已经形成基础信息架构。
3. `WorkspaceRuntime` 已经有会话、RAG、技能列表、文件编辑等基础能力。

但现状问题也很明确：

1. 视觉风格仍偏“基础工具台”，识别度不够强。
2. 灵魂系统只存在于后端静态文件中，前端没有明确入口。
3. 当前前端无法显示“当前人格是谁、为什么是这个人格、如何切换人格”。
4. 缺少能承载人格感知的角色形象资源。

---

## 3. 设计方向

## 3.1 总体视觉方向

本次前端采用统一方向：

`神话档案馆 / 本地灵魂工作台`

关键词：

1. 档案感
2. 仪式感
3. 指挥台
4. 东方神话气质
5. 非赛博紫色、非通用 SaaS 仪表盘

具体策略：

1. 页面以深色墨面、矿物色高光、金属线框和纸档案层次构成。
2. 不再使用“普通半透明白卡片”作为主要气质。
3. 灵魂切换后，界面主强调色、标题气质和人物展示区发生联动，但不影响核心交互结构。

## 3.2 四灵魂视觉基调

### 河伯

- 气质：清澈、收束、冷静
- 主色：深青、雾蓝、水纹银
- 适配：复杂问题收口、主线辨流

### 四岳

- 气质：稳重、结构、定盘
- 主色：岩灰、松墨、古铜
- 适配：架构规划、系统分层、规则整理

### 祝融

- 气质：推进、执行、破局
- 主色：炽橙、深赤、熔金
- 适配：排障修复、卡点突破、执行推进

### 玄女

- 气质：洞察、语义、辨析
- 主色：夜青、黛紫、月白
- 适配：复杂语义理解、策略推演、隐含条件辨识

## 4. 技术路线

## 4.1 状态层

在前端 store/runtime 增加灵魂系统状态：

1. `soulOptions`
2. `activeSoulKey`
3. `switchSoul()`
4. `refreshSoulState()`

数据来源：

1. 读取 `backend/soul/agent_core/ACTIVE_SEED.md`
2. 读取 `backend/soul/agent_core/seeds/*.md`
3. 切换时通过现有 `/api/files` 能力将目标 seed 内容写回 `ACTIVE_SEED.md`

这样可以保证：

1. 前端切换的就是当前正式人格。
2. 后端下轮提示词装配自然生效。
3. 不额外引入新的后端配置接口。

## 4.2 视觉实现

重点修改文件：

1. `frontend/src/app/globals.css`
2. `frontend/src/app/layout.tsx`
3. `frontend/src/app/page.tsx`
4. `frontend/src/components/layout/Navbar.tsx`
5. `frontend/src/components/layout/Sidebar.tsx`
6. `frontend/src/components/chat/ChatPanel.tsx`
7. `frontend/src/components/chat/ChatMessage.tsx`
8. `frontend/src/components/chat/ChatInput.tsx`
9. `frontend/src/components/editor/InspectorPanel.tsx`

原则：

1. 信息架构基本不推倒。
2. 视觉结构、区块层次、标题系统、空状态、切换面板重新设计。
3. 保持桌面可用性优先，同时确保移动端不会直接坏掉。

## 4.3 人物资产策略

理想方案：

1. 四张统一风格的高质量角色贴图。

当前执行策略：

1. 先落工程内可控的 SVG 立绘资产。
2. 资源路径固定，后续可无痛替换为位图 PNG/WebP。
3. 本轮先保证“页面里已经有正式人格形象”，而不是等外部绘图流程完成再接。

1. 文件切换标签更清晰

---

## 6. 风险控制

本轮重点防三类问题：

1. 改视觉时把原有会话/流式交互打坏。
2. 灵魂切换做成纯前端假切换，没有真正落到 `ACTIVE_SEED.md`。
3. 资源接入方式太重，导致前端构建不稳定。

对应控制策略：

1. 状态层先加，不先改 UI 行为。
2. 灵魂切换直接复用 `/api/files`，不造新协议。
3. 人物形象先使用 SVG 工程资源，避免外部依赖阻塞。
4. 每完成一层后运行 `npm run build` 验证。

---

## 7. 验收标准

完成后至少满足：

1. 前端视觉明显升级，不再像默认控制台。
2. 可以在前端看到四个灵魂并切换。
3. 切换后 `ACTIVE_SEED.md` 被正式更新。
4. 页面关键位置能看到对应灵魂形象。
5. `npm run build` 可以通过。
