# 任务图编辑器意图化优化重构计划

## 1. 纠偏说明

本计划替代上一版“意图化简化”表述。重构目标不是把旧页面折叠到更深层，也不是把 10 个入口压成 3 个入口就算完成。真正目标是优化编辑器，让用户以更少操作表达任务图语义，并让系统主动生成、校验和修复正确结构。

因此，本轮重构必须遵守：

- 不用“隐藏高级配置”冒充优化。
- 不用“二级 tab 收纳旧页面”冒充重构。
- 不删除图任务现有能力。
- 不把后端运行协议重新搬回前端。
- 不保留旧入口作为兼容壳。

正确目标是：

```text
用户表达意图
-> 编辑器生成标准节点/边草稿
-> Inspector 只要求必要配置
-> 预检指出缺失项并给出修复动作
-> 后端编译为运行协议
```

## 2. 当前问题

当前任务图编辑器功能完整，但用户要先理解内部层级再编辑：

- `TaskGraphLayerNav.tsx` 暴露 10 个层：图级配置、拓扑、图契约、执行者、节点契约、阶段、资源流、风险、质量门、发布运行。
- `TaskGraphWorkbench.tsx` 使用 `activeLayer` 在 10 个页面之间切换，页面边界按实现域划分，而不是按用户操作划分。
- `TaskGraphTopologyPage.tsx` 已经是正确主入口，但选中节点/边后仍展示大量后端式字段和契约 binding section。
- `TaskGraphStudioShell.tsx` 同时展示图身份、工作集、层说明、标准对象视图、页面主体、问题条，首屏负担过重。

这些不是单纯 UI 拥挤，而是编辑器权责没有优化：前端还在要求用户理解“系统如何运行”，而不是帮助用户表达“这个任务图要怎么协作”。

## 3. 目标产品模型

任务图编辑器应变成“语义化工作台”：

### 3.1 主工作区

主工作区始终以任务图画布为核心。

- 左侧：语义动作栏。
- 中间：任务图画布。
- 右侧：选中对象 Inspector。
- 底部：问题与发布状态。

### 3.2 语义动作栏

用户不直接添加底层节点/边，而是选择成熟任务图动作：

- 添加写作节点。
- 添加审核节点。
- 添加返修关系。
- 添加记忆读取。
- 添加候选记忆写入。
- 添加记忆提交。
- 添加产物仓库。
- 添加人工门控。
- 应用写作-审核-返修模板。

动作生成最小标准草稿，前端只写语义 ID 和必要参数，不生成后端运行协议。

### 3.3 智能 Inspector

Inspector 由选中对象和语义类型决定，而不是展示固定全字段表。

节点 Inspector：

- 角色与 Prompt。
- 输入输出契约。
- 资源读取和产物目标。
- 执行者与模型需求。
- 审核/人工门控要求。
- 高级策略，仅在需要时展开。

边 Inspector：

- 边语义。
- 来源和目标。
- 必填参数。
- 契约绑定。
- 记忆/产物/返修参数。
- 高级策略，仅在需要时展开。

### 3.4 问题驱动修复

检查区不是旧页面收纳箱，而是修复工作台：

- 每个问题必须定位到节点、边或图级配置。
- 能自动补齐的，提供修复按钮。
- 不能自动补齐的，打开对应 Inspector 分区。
- 标准对象视图只作为检查结果和发布依据，不在主编辑流中压迫用户。

## 4. 权责边界

### 前端负责

- 表达用户意图。
- 创建语义化节点/边草稿。
- 展示必要配置。
- 定位问题。
- 调用保存、预检、发布、运行。

### 后端负责

- 解析语义边。
- 展开契约族。
- 生成记忆读写协议。
- 编译 GraphHarnessConfig。
- 校验发布可运行性。
- 组装运行包。

### 前端禁止

- 禁止重新推导 GraphHarnessConfig。
- 禁止生成 memory protocol 细节。
- 禁止把 `contract_bindings.semantic` 等后端协议回填为 UI 默认值。
- 禁止用万能 JSON 表单替代结构化配置。
- 禁止保留 10 层旧导航作为隐藏备用路径。

## 5. 目标交互路径

### 创建写作审核图

1. 用户选择“写作-审核-返修模板”。
2. 编辑器生成写手节点、审核节点、返修边、必要记忆/产物节点。
3. 用户只编辑写手 Prompt、审核 Prompt、契约要求。
4. 预检指出缺失输入输出或环境绑定。
5. 用户一键修复或进入对应 Inspector 分区。

### 添加记忆读取

1. 用户选中一个任务节点。
2. 点击“读取记忆”。
3. 编辑器要求选择仓库和 collection。
4. 系统生成标准记忆读取边。
5. 后端负责解析 selector 和运行装载协议。

### 添加审核返修

1. 用户选中写作节点和审核节点。
2. 点击“审核关系”。
3. 编辑器生成审核边和必要返修候选。
4. 用户只配置审核标准、返修目标、结果 key。
5. 不让用户手写底层 edge_type 组合。

## 6. 文件级实施计划

### 阶段 1：建立语义动作模型

新增：

- `TaskGraphEditorActions.ts`
  - 定义前端可展示的语义动作。
  - 每个动作只描述 UI 标签、适用对象、必要参数、调用方式。
  - 不包含后端运行协议。

修改：

- `taskGraphSemanticRelations.ts`
  - 保持轻量，只提供语义边 ID 和参数投影。
  - 不加入 contract binding 或 memory protocol 默认值。

完成标准：

- 用户添加边时看到的是“审核 / 返修 / 读取记忆 / 写入记忆 / 产物引用”，不是 edge_type 原始表。

### 阶段 2：重构主编辑器布局

修改：

- `TaskGraphStudioShell.tsx`
  - 移除总是显示的层说明条。
  - 标准对象视图移到检查工作台。
  - 主编辑区固定为动作栏、画布、Inspector。

- `TaskGraphLayerNav.tsx`
  - 不再渲染 10 层卡片。
  - 替换为任务图工作区切换：编辑、检查、发布。
  - 该切换只改变工作区，不承载旧页面兼容入口。

完成标准：

- 首屏不再要求用户选择内部层。
- 用户默认进入画布编辑。

### 阶段 3：拆分拓扑页为可维护结构

从 `TaskGraphTopologyPage.tsx` 拆出：

- `TaskGraphActionRail.tsx`
  - 展示语义动作。
  - 调用节点/边创建函数。

- `TaskGraphCanvasPanel.tsx`
  - 包装现有 `CoordinationTopologyGraph`。
  - 保留当前图任务画布能力。

- `TaskGraphObjectInspector.tsx`
  - 统一节点与边配置。
  - 根据对象语义显示必要字段。

完成标准：

- `TaskGraphTopologyPage.tsx` 只负责布局组装。
- 添加节点/边的操作从底层按钮升级为语义动作。

### 阶段 4：优化 Inspector，而不是折叠 Inspector

修改：

- `TaskGraphContractBindingInspector.tsx`
  - 从“全 section 表单”改为“语义所需字段 + 缺失提示 + 高级策略”。

新增：

- `TaskGraphNodeSmartInspector.tsx`
- `TaskGraphEdgeSmartInspector.tsx`

要求：

- 节点默认展示 Prompt、输入、输出、资源、执行者。
- 边默认展示语义、起点终点、必填参数、契约。
- runtime/governance 不是隐藏，而是在有实际需要或问题定位时展开。

完成标准：

- 用户配置一个写手节点或审核边时，不需要看完整后端字段矩阵。
- 预检问题能打开具体字段，而不是只跳到页面。

### 阶段 5：重构检查工作台

新增：

- `TaskGraphValidationWorkbench.tsx`

整合能力：

- 预检问题。
- 标准对象视图。
- 契约质量。
- 阶段/循环。
- 风险治理。

处理方式：

- 原 `TaskGraphContractQualityPage.tsx`、`TaskGraphRiskGovernancePage.tsx`、`TaskGraphTimelinePage.tsx`、`TaskGraphModuleCompositionPage.tsx` 的能力拆成检查卡片或问题面板。
- 若旧页面失去独立职责，实施结束删除旧页面文件。

完成标准：

- 检查工作台回答三个问题：
  - 当前为什么不能发布。
  - 哪些问题可以自动修。
  - 哪些问题需要用户补配置。

### 阶段 6：发布运行页收束

保留：

- `TaskGraphPublishRunPage.tsx`

修改：

- 只保留保存、预检、发布、运行绑定、监控入口。
- 预检问题点击后回到画布对象或检查工作台。

完成标准：

- 发布页不再承担编辑配置。
- 发布闭环清晰。

### 阶段 7：清理旧导航和样式

修改：

- `frontend/src/app/globals.css`

删除：

- 旧 10 层卡片导航样式。
- 不再使用的层说明条样式。
- 拆分后失效的重复 topology inspector 样式。

完成标准：

- 没有新旧两套任务图导航。
- 没有保留空壳页面。
- 样式减少而不是换一种方式膨胀。

## 7. 不做事项

- 不把任务图编辑器重新放回任务系统管理页。
- 不把旧 10 个页面简单塞进检查页 tab。
- 不用折叠控件掩盖信息架构问题。
- 不删除现有图任务功能。
- 不重写后端任务图模型。
- 不在本轮强制换画布引擎。现有画布能力先保留；如果后续要做复杂拖拽、自动布局、边手柄和大图性能，再单独评估成熟图编辑库。

```powershell

```

页面：

- 固定前端：`http://127.0.0.1:3000`
- 固定后端：`http://127.0.0.1:8003`
- 使用 Edge 检查：
  - 默认进入画布编辑。
  - 能添加写作、审核、返修、记忆、产物动作。
  - 节点 Inspector 能编辑 Prompt 和契约。
  - 边 Inspector 能编辑语义关系参数。
  - 检查工作台能定位并修复问题。
  - 发布运行不退化。

## 9. 验收标准

- 用户减少需要理解的系统层级，而不是只把层级藏起来。
- 常见写作图、审核图、记忆读写图可以通过语义动作快速搭建。
- 节点 Prompt、节点契约、边关系、记忆读写、产物交接都完整可配置。
- 高级配置仍可达，但由问题或语义上下文触发。
- 前端不重新承担后端编译和运行协议职责。
- 图任务保存、预检、发布、运行链路不退化。
