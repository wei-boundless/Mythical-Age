# RAG 知识库外置迁移实施记录

## 目标边界

- 保留记忆库在项目 `storage` 内：`durable_memory`、`formal_memory`、`session_memory`、`working_memory`、`task_durable_memory` 不迁移。
- 外置 RAG 数据根：知识资产、检索/向量索引、文档转换缓存、模态派生产物。
- 保持用户和 agent 的逻辑路径不变：仍然使用 `knowledge/...` 引用知识库文件。

## 目标目录

默认外置根：

```text
../langchain-agent-data
```

子目录：

```text
knowledge
indexes
document_cache
modality_artifacts
```

可通过环境变量覆盖：

```text
APP_EXTERNAL_DATA_ROOT
APP_KNOWLEDGE_ROOT
APP_INDEXES_ROOT
APP_DOCUMENT_CACHE_ROOT
APP_MODALITY_ARTIFACTS_ROOT
```

## 权威入口

`ProjectLayout` 是唯一目录权威：

- `knowledge_storage_dir` 指向外置知识库。
- `indexes_dir` 指向外置检索索引根。
- `document_cache_dir` 指向外置文档转换缓存。
- 记忆库属性继续指向项目内 `storage`。

`WorkspaceFileService` 负责逻辑路径桥接：

- `knowledge/...` 解析到外置 `knowledge_storage_dir`。
- 对外展示仍返回 `knowledge/...`，避免破坏已有会话、绑定、证据包和用户习惯。

## 迁移策略

`ensure_storage_dirs()` 会迁移旧目录：

- `knowledge/` -> 外置 `knowledge`
- `backend/knowledge/` -> 外置 `knowledge`
- `storage/knowledge/` -> 外置 `knowledge`
- `storage/indexes/` 和 `storage/indexes_v2/` -> 外置 `indexes`
- `storage/document_cache/` 和 `storage/document_cache_v2/` -> 外置 `document_cache`
- `storage/modality_artifacts/` -> 外置 `modality_artifacts`

迁移合并不会覆盖同名不同内容文件；如果目标已有同名冲突，旧目录会保留以便人工处理。

## 验证点

- layout 默认外置根正确。
- env override 正确。
- memory 原始库仍在项目 `storage`。
- RAG collections 使用外置索引和知识源。
- 文件工具可通过 `knowledge/...` 读写外置知识库。
- PDF/结构化数据目录返回稳定的 `knowledge/...` 逻辑路径。
