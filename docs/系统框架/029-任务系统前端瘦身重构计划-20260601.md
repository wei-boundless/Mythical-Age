# 任务系统前端瘦身重构计划

## 问题结论

这次任务系统分层重构的旧入口已经删除，但实现方式不合格：新功能主要堆在三个大型 TSX 组件里，导致一次页面改造产生过大的代码增量。

主要增量来源：

- `TaskEnvironmentManagementWorkbench.tsx`：约 800 行，环境类型、资源装载、Prompt、任务、任务图都塞在一个文件。
- `ContractManagementWorkbench.tsx`：约 690 行，目录、筛选、Inspector、Schema、产物、验收、运行策略都在一个文件。
- `NodeConfigurationWorkbench.tsx`：约 670 行，节点目录、详情、执行装配、契约绑定、权限、预览都在一个文件。
- `globals.css`：新增数百行页面专用样式，继续污染全局样式文件。
- `backend/storage/.../replacement_*.json`：运行态文件误入版本库，已开始清理。

旧契约页和旧节点页被删除了，但“用新大组件替代旧大组件”的方式仍然不成熟。

## 目标

保留已经正确建立的数据能力：

- 环境类型模板
- 环境资源装载
- 环境 Prompt
- 环境内任务 inventory
- 环境任务图健康摘要
- 契约使用影响索引
- 节点配置一等对象
- runtime preview

但重构前端实现形态：

- 单个页面组件不超过约 350 行。
- 重复 UI 原语只写一次。
- 全局 CSS 只保留布局级变量和共享基础样式，页面专用样式下沉到局部组件样式或复用既有原语。
- 任务图工作台组件不再物理归属 `task-system` 目录，避免目录所有权误导。

## 处理方案

### 阶段 1：清理无效提交内容

- 从 git 跟踪中移除 `backend/storage/**` 运行态文件。
- 补强 `.gitignore`，确保后续不会再把运行态 storage 加入提交。

### 阶段 2：抽管理台共享原语

新增小型共享组件，只负责展示结构，不包含领域决策：

- `ManagementWorkbenchFrame`
- `ManagementFilterRail`
- `ManagementCatalogTable`
- `ManagementInspector`
- `ManagementTabs`
- `MetricStack`
- `JsonObjectEditor`
- `TextListEditor`

环境、契约、节点页面只传配置和领域处理函数，不再重复写三套布局。

### 阶段 3：拆分三个巨型页面

将当前三个大文件拆成领域子页：

- `environment/EnvironmentTypesPage.tsx`
- `environment/EnvironmentLoadoutPage.tsx`
- `environment/EnvironmentPromptsPage.tsx`
- `environment/EnvironmentTasksPage.tsx`
- `environment/EnvironmentGraphsPage.tsx`
- `contracts/ContractCatalogPage.tsx`
- `contracts/ContractInspectorTabs.tsx`
- `nodes/NodeCatalogPage.tsx`
- `nodes/NodeInspectorTabs.tsx`

拆分不是单纯搬文件，必须删除重复布局和重复 JSON 编辑器。

### 阶段 4：收束 CSS

- 删除这次新增的重复全局样式。
- 只保留通用 workbench 原语样式。
- 领域页面使用同一套 rail/table/inspector/tabs 样式。

### 阶段 5：目录所有权修正

- 将任务图工作台相关组件迁出 `views/task-system/`。
- 新目录为 `views/task-graph-workbench/components/` 或等价结构。
- `TaskSystemView` 不引用任何任务图拓扑编辑组件。

- `python -m pytest backend/tests/task_environment_api_regression.py backend/tests/task_environment_registry_regression.py backend/tests/task_environment_kind_template_regression.py backend/tests/task_environment_inventory_regression.py backend/tests/task_contract_usage_index_regression.py backend/tests/task_node_configuration_registry_regression.py -q`
- Playwright/Edge 检查 `http://127.0.0.1:3000/?view=task-system` 的环境、契约、节点三个域。

## 不做

- 不删除已经需要的后端领域模型，只为减少行数而回退架构。
- 不把契约、节点配置重新合并成旧的万能 JSON 表单。
- 不保留新旧两套任务系统页面入口。
