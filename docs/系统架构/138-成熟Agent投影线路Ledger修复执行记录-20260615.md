# 138-成熟Agent投影线路 Ledger 修复执行记录

日期：2026-06-15

依据计划：`137-成熟Agent投影线路Ledger权威修复方案-20260614.md`

## 1. 本次落地结论

已把主聊天投影权威收束为：

```text
runtime event
-> public event bridge
-> chatrun:* public ledger
-> session_timeline display_state/main_chat_surface
-> frontend hydrate/reducer
-> ChatMessage 三态显示
```

主显示规则已落地：

| 状态 | 主聊天表面 | 工具窗口 |
| --- | --- | --- |
| `normal_turn` | `body_only`，只显示 assistant 正文 | 不 replay 到主聊天 |
| `task_live` | `live_timeline`，显示 live timeline/工具窗口 | 按 `tool_call_id` 合并显示 |
| `task_closed` | `closeout_summary`，显示收口正文/摘要 + 日志入口 | 不在主聊天展开，只进日志/trace |
| `log_only` | 不进入主聊天 | 只做 trace/log |

## 2. 关键修复

### 2.1 后端 ledger replay

- `backend/runtime/shared/stream_replay.py`
  - 增加 `list_public_events()` / `list_public_event_records()`。
- `backend/harness/runtime/session_timeline.py`
  - 新增 stream run attachment，从 `run_registry.list_session_runs(session_id)` 找 `chatrun:*`。
  - 从 public ledger 读取 `public_projection_frame`。
  - 输出 `display_state`、`main_chat_surface`、`tool_event_count`、`closeout_summary`、`log_ref`。
  - task/turn raw trace attachment 改为 `log_only`，`public_projection_frames: []`。
  - 删除 raw event tail 重建主聊天投影的旧主路径。

### 2.2 前端显示分流

- `frontend/src/lib/store/runtime.ts`
  - `hydrateSessionRuntimeProjection()` 只对 `main_chat_surface === "live_timeline"` replay frames 到主聊天。
  - `body_only` 不 replay 工具窗口。
  - `closeout_summary` 只写入收口正文/日志引用/工具次数。
- `frontend/src/lib/store/events.ts`
  - live SSE 按 task anchor 和 terminal/commit 信号设置 `normal_turn/body_only`、`task_live/live_timeline`、`task_closed/closeout_summary`。
- `frontend/src/components/chat/ChatMessage.tsx`
  - 只有 `live_timeline` 渲染 `PublicTimelineActivity`。
  - `task_closed` 渲染正文/收口摘要 + 轻量日志入口，不渲染工具窗口。
- `frontend/src/components/chat/PublicTimelineActivity.tsx`
  - 已确认只承担 live 工具窗口显示；工具轨迹以可折叠窗口呈现，默认隐藏内部 id/offset/source 噪声。
- `frontend/src/lib/store/utils.ts`
  - 修复历史 assistant 消息过滤：带 `tool_calls` 但有正文时保留正文；只有纯工具调用、无正文的 assistant 记录才过滤。

### 2.3 工具调用与反馈身份

- `backend/api/chat.py`
  - public bridge 支持 `tool_calls[]`。
  - 同一模型 action 的公开反馈只输出一次，然后按工具数组顺序输出 request/permission。
  - 每个工具调用使用独立 `tool_call_id`；缺失时生成稳定 id，但不把 `request_id` 直接当作 `tool_call_id`。
  - permission id 按 `tool_call_id` 作用域化。
- `backend/harness/runtime/projection/projector.py`
  - 模型反馈 frame/item id 使用 `feedback_identity`，不再按文本相等误去重。
- `frontend/src/components/chat/ChatMessage.tsx`
  - 模型反馈去重改为 identity 级，不按纯文本误删。

## 3. 对线矩阵

| 信号 | 生产者 | 传输/存储 | 前端消费 | 主聊天结果 |
| --- | --- | --- | --- | --- |
| assistant 正文 | `assistant_text_*` public event | `chatrun:*` ledger + session history | `toUiMessages()` / projection body | 普通 turn 和收口均可显示正文 |
| 工具 request/start/completed | public bridge / projector | `public_projection_frame.tool_call_id` | reducer 按 `tool_call_id` 合并 | 只在 `live_timeline` 显示工具窗口 |
| 多工具调用 | `tool_calls[]` | 每个工具独立 `tool_call_id` | reducer 合并同 id 生命周期 | 多个工具窗口不串线 |
| 模型公开反馈 | `runtime_step_summary.feedback_identity` | public frame item/frame id | ChatMessage identity 去重 | 不误删不同 action 的相同反馈 |
| task terminal/commit | `turn_completed` / `session_output_commit_*` | ledger + session_timeline resolver | `main_chat_surface=closeout_summary` | 收口后不显示工具窗口 |
| raw task/turn trace | runtime event log | `log_only` attachment | runtime log/trace | 不进入主聊天 |
