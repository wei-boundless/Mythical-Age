## 3. UI 对照

| 129 要求 | 当前代码 |
| --- | --- |
| 折叠态只显示 `<工具动作> <目标> <状态>` | `ChatMessage.tsx::projectionToolWindowTitle()` 只组合 action、target、status。 |
| 展开态只显示 `$ <tool_name> <target/args>` 和返回摘要 | `PublicTimelineActivity.tsx::ToolWindow()` 渲染单个 console；`toolWindowCommandLine()` 和 `toolWindowOutputText()` 提供命令行和摘要。 |
| 默认不显示 ID、offset、source_event_type | `PublicTimelineActivity.tsx` 对 tool window 的 `meta` 置空；内部 section 会过滤 `调用号/offset/event/source`。 |
| 正文和工具按同一时序但不同表面显示 | `ChatMessage.tsx::orderedProjectionMessageBlocks()` 将 body block 与 activity 按 offset 混排；model feedback 先转正文，工具仍进 `PublicTimelineActivity`。 |

## 4. 注意

- `model_action_admission_checked` 在 task bridge 真实事件流里可能同时携带 `model_action_request` 和 `admission`。当前桥接会从这一条运行事件投出公开 `tool_call_requested -> tool_permission_decided`，否则 task bridge 工具窗口会没有 request 锚点。
- 这不改变前端规则：前端只看 public projection 信号和 `tool_call_id`，不会把 permission 当成新窗口。
- 工具请求事件已禁止携带正文反馈字段；模型公开反馈只能通过 `runtime_step_summary/status_trace` 进入正文区。
