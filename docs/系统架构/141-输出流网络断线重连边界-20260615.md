# 输出流网络断线重连边界

日期：2026-06-15

## 结论

自动重连保护只属于已建立的 chat run 输出流传输断线。

前端已经拿到 `stream_run_id` 后，SSE 读取链路如果发生网络/传输层中断，才允许用同一个 `stream_run_id + event_offset` 续接 `/chat/runs/{stream_run_id}/events?after_offset=...`。

## 允许重连的信号

| 信号 | 判定 | 行为 |
| --- | --- | --- |
| `fetch` 抛出 `TypeError` | 浏览器网络请求失败 | 保持当前 run，按 offset 重连 |
| SSE reader / fetch 抛出 `AbortError`，且前端 `AbortController.signal` 未进入 `aborted` | 浏览器底层连接被中断，不等同于用户停止 | 保持当前 run，按 offset 重连 |
| `TimeoutError` / `RequestTimeoutError`，且前端 `AbortController.signal` 未进入 `aborted` | 传输层等待超时，不等同于用户停止 | 保持当前 run，按 offset 重连 |
| `NetworkError` / `Failed to fetch` / `Load failed` / `The network connection was lost` | 浏览器或运行环境报告网络断线 | 保持当前 run，按 offset 重连 |
| SSE reader 正常结束但没有 `turn_completed` | 输出流无终态关闭，视为传输断线 | 保持当前 run，按 offset 重连 |

## 不允许重连的信号

| 信号 | 原因 | 行为 |
| --- | --- | --- |
| `AbortError` 且前端 `AbortController.signal.aborted === true` | 用户显式停止或会话被移除，属于本地控制，不是网络保护 | 抛给 runtime，由 runtime 按显式停止/移除处理 |
| HTTP 非 2xx | 后端明确拒绝或失败，不是网络断线 | 触发 `stream_reconnect_failed` 后失败 |
| SSE payload 解析失败 | 协议错误，不是网络断线 | 触发 `stream_reconnect_failed` 后失败 |
| 后端 `turn_completed` | run 已有终态 | 清理 cursor，结束输出流 |

## 当前修复

- 文件：`frontend/src/lib/api.ts`
- `isReconnectableChatStreamTransportError` 识别网络/传输错误，并把非本地 signal 触发的 `AbortError` / `TimeoutError` 归入可重连传输断线。
- HTTP 错误和缺失 readable body 改为 `ChatStreamProtocolError`，不会进入重连循环。
- 非网络错误会先发送 `stream_reconnect_failed`，再交给外层 runtime 显示失败。

## 2026-06-15 追加修复

- 文件：`frontend/src/lib/store/runtime.ts`
- 外层 runtime 不再用异常名推断 stopped。只有 `stopCurrentStream()` 写入的 `stoppedStreamingSessionIds` 才能把输出流判为用户停止。
- 已建立 run 的断线链路保持为：`stream_run_id + event_offset` -> `/chat/runs/{stream_run_id}/events?after_offset=...` -> `stream_reconnecting` -> `stream_reconnected` -> 后续公开投影。
- 验证：`frontend/src/lib/api/client.test.ts` 增加 SSE reader 抛 `AbortError` 的结构测试，要求事件顺序为 `stream_reconnecting -> stream_reconnected -> turn_completed`。
