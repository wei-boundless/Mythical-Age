# 投影线路严格时序修复执行记录

日期：2026-06-14

状态：继续按 `125-投影线路严格时序修复计划书-20260614.md` 收口生产主链路。本记录只保留真实投影线路、身份和时序证据；语义测试不作为本轮验收依据。

## 1. 已修复线路

| 线路 | 修复结果 |
| --- | --- |
| public frame offset | `PublicProjectionFrame.event_offset` 只使用 public sequence，不再读取 payload `event_offset/offset`。 |
| lifecycle offset 校验 | 工具 lifecycle 状态机只使用 public sequence 校验 request/permission/start/completed 顺序。 |
| active steer projection | `active_task_steer_accepted` 改为可见 `status` item，不进入 assistant body。 |
| tool identity | 新增 `runtime.shared.tool_identity`，集中生成/读取 canonical `tool_call_id` 和 fallback `permission_decision_id`。 |
| chat bridge | `_tool_call_requested_data()` 不再用 `request_id` 充当 `tool_call_id`。 |
| permission fallback | task executor、single agent turn、chat completion fallback 统一到 canonical helper；缺 admission id 时使用 `admission:{tool_call_id}`。 |
| frontend body ordering | `ProjectionLedger.body.blocks` 保存 body segment offset，ChatMessage 按 body block 与 timeline item 混排。 |
| frontend tool lifecycle | 工具 timeline display key 改为 `tool_lifecycle_id || tool_call_id`，completed 更新同一项，不再按 frame id 拆成第二项。 |
| frontend block rendering | 分块正文使用各自 `block.text` 判断显示，不再依赖整条消息的全局正文状态；带图片消息不会把正文块错误渲染成图片。 |
| frontend timeline contract | `PublicChatTimelineItem` 补齐 `source_event_type/sourceEventType`，前端不再丢失事件来源类型。 |
| status timeline identity | 非工具 status timeline item 改用 `frame_id/source_event_id`，不再用稳定 `task-stage-status:*` 合并到旧位置；`projection.status` 仍保留最新状态。 |
| tool completion identity | `tool_item_completed` 从 observation invocation/admission ref 取 `tool_lifecycle_id` 和 `permission_decision_id`，不再把 `tool_call_id` 或 `admission:{tool_call_id}` 当完成身份。 |
| model progress status | `model_action_admission` 中的 `public_progress_note/current_judgment/next_action` 先投成 `runtime_step_summary` status 帧，再投工具请求帧；正文不消费这类进展文本。 |

## 3. 本轮真实证据

本轮没有继续维护语义测试。已做的主链路验证：

```powershell
python -m py_compile backend/api/chat.py backend/runtime/shared/tool_identity.py
npx tsc --noEmit
```

真实历史事件回放确认：

- `task_tool_observation_recorded` offset `24` 重新投影后，`tool_lifecycle_id=toolinv:42375f4f255b651a`。
- 同一 completion 的 `permission_decision_id=admission:model-action:taskrun:turn:session-cb1d7772f53a4823:3:b9163208:epoch:1:invocation:1:ef5427966fab:tool:1`。
- 不再输出旧错接 `admission:{tool_call_id}`。

真实 chatrun 证据：

- `runtime_step_summary` visible status 帧存在于 offsets `92, 109, 120, 130, 136, 142, 194, 204, 216, 217`。
- 这些 status 帧旧数据共用同一个 `item_id=task-stage-status:d24514fed10e2f5f`，但各自有唯一 `publicframe:*` 和递增 `event_offset`。

样例 raw `model_action_admission` 桥接确认新顺序：

## 4. 已清理的旧主权威

当前主显示权威：

- 正文显示主权威是 `MessagePublicProjection.bodyBlocks[]` 的 `firstOffset/lastOffset`。
- 工具/状态显示主权威是 `MessagePublicProjection.timeline[]` 的 `eventOffset/updatedEventOffset`。
- `bodyEventOffset` 只保留为兼容元数据，不再用于 ChatMessage 的正文/工具混排。
- `frame:tool-completed` 这类按完成帧拆工具行的旧显示方式已改为 lifecycle-keyed upsert。
- 后端 public frame offset 由 public sequence 产生，不从 payload `event_offset/offset` 反读。

- 当前项目栈在固定端口保持启动状态：frontend `3000`，backend `8003`。
- 当前 store 没有浏览器侧调试注入入口；未为了截图新增临时后门。页面级验证采用固定端口 + 本地 Edge 渲染，严格时序显示能力由真实 reducer/component 代码路径覆盖。
