# 模型等待态占位投影与清洗器权威修复记录

日期：2026-06-15

## 当前结论

`正在思考` 不是后端公共 timeline 事件，也不是可持久化的正文或工具记录。

它只能是前端在等待模型正文/工具控制轨迹到来前显示的本地占位。只要真实信号到达，例如正文 `body_append`、工具控制轨迹、计划状态或终态，占位就必须消失。

## 权威边界

| 信号 | 权威层 | 公开投影 | 前端显示 |
| --- | --- | --- | --- |
| 大模型正文 | 模型输出流 | `body_append/body_finalize` | 正文区域按 offset 追加 |
| 工具控制轨迹 | 工具生命周期投影 | `tool_call_requested/tool_started/tool_completed` 对应工具窗口 | 折叠工具窗口 |
| `agent_todo` | 任务计划账本 | 不进入工具窗口；只可折成 `todo_plan_status` 或隐藏 trace | 计划状态，不显示命令窗口 |
| 模型等待心跳 | 执行器内部等待态 | 不生成公开事件；历史残留也降为 hidden trace | 前端本地占位 |
| 系统原因码 | 后端运行态 | 映射为中文可读状态或隐藏 | 不泄露原始 reason code |

## 当前线路

```text
task_model_action_wait_heartbeat
  -> _project_public_stream_event(...)
  -> []

历史残留 runtime.model_wait / model_wait_placeholder
  -> projection._status_spec(...)
  -> main_visibility = hidden
  -> retention = trace
```

## 清洗器职责

公共文本清洗器只负责删除私有引用、结构化协议痕迹、原始工具输出和裸机器状态码。它无权按“文案泛化程度”吞掉合法内容。

已经删除旧的泛化进度文案黑名单，避免把模型合法反馈、用户可读反馈或状态摘要吞掉。
