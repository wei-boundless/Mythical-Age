# 协议修复提示误投影修复记录

日期：2026-06-15

## 现象

前端主视图出现：

```text
当前步骤输出格式不完整，正在自动修正后继续。
```

## 根因

该文本来自任务执行器内部协议修复分支：

- 事件：`task_model_action_protocol_repair_required`
- step：`model_action_protocol_repair_required:*`
- 语义：模型给系统的 action JSON 不完整，runtime 生成 repair observation，让模型重新提交合法动作。

这不是大模型给用户的正文，也不是工具调用窗口内容。旧链路把它写成普通 `step_summary_recorded`，chat API 再转成 `runtime_step_summary`，投影器因此当作可见运行进度展示。

## 修复

- `backend/harness/loop/task_executor.py`
  - 为协议修复 step 写入 `presentation_source="runtime.protocol_repair"`。
  - 该来源不再写入 `latest_public_progress_note` / `latest_public_status`。
- `backend/harness/runtime/projection/projector.py`
  - `runtime.protocol_repair` 进入 trace-only，不进入主投影。
- `backend/harness/runtime/run_monitor/projector.py`
  - run monitor 的 latest public step 排除 `runtime.protocol_repair`。

## 边界

协议修复记录仍保留在 runtime trace 中用于诊断；用户主界面只显示模型正文、模型公开反馈、工具调用窗口和真正的用户可见状态。
