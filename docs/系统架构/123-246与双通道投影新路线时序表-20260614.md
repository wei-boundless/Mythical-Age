日期：2026-06-14

## 1. 新路线核心约束

本路线采用成熟 coding agent 风格的公开投影模型：

```text
Runtime canonical event log
-> single public event clock
-> PublicProjectionFrame(event_family/channel/lossless/event_offset)
-> Frontend ProjectionLedger
-> Message surface
```

关键裁决：

1. 公开投影只有一个排序时钟：`event_offset`。
2. 正文和工具不是两条物理 SSE，而是同一有序事件流上的两个逻辑通道。
3. assistant 正文只能来自 `assistant_text_delta`、`assistant_text_final`、`assistant_stream_repair`。
4. 工具轨迹只能来自工具生命周期事件：`tool_call_requested`、`tool_permission_decided`、`tool_item_started`、`tool_item_completed`。
5. `source_task_event_offset`、`source_turn_event_offset`、`source_handoff_event_offset` 只作为证据引用，不参与前端排序。
6. terminal、snapshot、工具 observation、runtime status、replacement stop closeout 都不能反推或合成 assistant 正文。
7. runtime 私有产物不进入普通 workspace 文件面，也不进入用户可见正文。

## 2. 事件族与显示面

| Event family | Channel | 事件来源 | 可见位置 | 是否可成为正文 | 排序权威 |
| --- | --- | --- | --- | --- | --- |
| `assistant_body` | `body` | 模型正文流 | assistant 正文块 | 是 | `event_offset` |
| `tool_control` | `control` | 模型工具请求、权限、执行器生命周期 | 工具/控制轨迹块 | 否 | `event_offset` |
| `runtime_commit` | `commit` | session output commit gate | commit 状态、trace/pinned failure | 否 | `event_offset` |
| `turn_anchor_terminal` | `terminal` | turn anchor、task bridge、turn completed | trace/terminal 状态 | 否 | `event_offset` |
| `status_trace` | `status` | runtime status、steer ack、diagnostic | 状态栏、trace、必要时 pinned | 否 | `event_offset` |

显示面规则：

| 显示面 | 允许内容 | 禁止内容 |
| --- | --- | --- |
| Assistant 正文 | 模型正文 delta/final、明确阻塞说明 | 工具 raw output、runtime path、控制回执、terminal final_answer |
| 工具轨迹 | 工具 requested/permission/started/completed、安全摘要、失败摘要 | 工具结果拼接进正文、重复 start/return 行 |
| 控制/状态 | steer accepted、pause/stop/continue receipt、safe boundary wait | 作为 assistant 正文回答 |
| Commit/Terminal | commit ack/failed/skipped、turn_completed、task_bridge_terminal | 合成正文、覆盖已有正文 |
| Debug/Trace | event id、source offset、diagnostic | 进入主聊天正文 |

## 3. 主时序表

| 时序 | 阶段 | 输入/触发 | 产生的公开事件 | Event family / channel | UI 行为 | 必须满足 |
| --- | --- | --- | --- | --- | --- | --- |
| T0 | 用户消息入站 | 用户在主会话发送消息 | 无公开 projection frame；用户消息先入 session | 无 | 显示用户气泡 | 不生成 runtime public item |
| T1 | turn anchor 建立 | chat turn 创建 | `chat_turn_bound` | `turn_anchor_terminal / terminal` | trace 记录锚点 | 必须先于 task bridge 和正文/工具投影 |
| T2 | active work 边界裁决 | 当前存在 active turn/task | `active_task_steer_accepted` 或 control receipt 事件 | `status_trace / status` | 控制/状态面显示接收结果 | steer/control 不进入正文，不创建 replacement TaskRun |
| T3 | TaskRun bridge 绑定 | 后台任务接管本 turn | `task_bridge_started` | `turn_anchor_terminal / terminal` | trace 记录 bridge | 后续 task event 必须使用同一 public anchor |
| T4 | 模型输出正文 delta | 模型返回 text delta | `assistant_text_delta` | `assistant_body / body` | 追加 assistant 正文块 | delta 按 `event_offset` 实时显示 |
| T5 | 模型请求工具 | 模型返回 tool_call | `tool_call_requested` | `tool_control / control` | 出现工具轨迹运行项 | `tool_call_id` 必须来自模型 tool_call.id；`request_id` 只做 trace |
| T6 | 权限裁决 | admission/permission 完成 | `tool_permission_decided` | `tool_control / control` | 允许则 trace，需审批/阻塞则 pinned | offset 必须晚于 T5；`permission_decision_id` 不能替代 `tool_call_id` |
| T7 | 工具开始执行 | executor 开始工具调用 | `tool_item_started` | `tool_control / control` | 更新同一工具生命周期 | offset 必须晚于 T6；必须绑定允许的 permission |
| T8 | 工具完成/失败 | executor 收到 observation | `tool_item_completed` | `tool_control / control` | 同一工具项 terminalize 为 done/error | offset 必须晚于 T7；工具 observation 不写正文 |
| T9 | 任务期间用户补充 | active task 输入框发送补充 | `active_task_steer_accepted`，后续可有 steer consumed status | `status_trace / status` | 显示补充已纳入当前任务 | 不退回普通模型 turn，不触发新 task replacement |
| T10 | runtime 私有产物生成 | dynamic context replacement、tool result store | 无普通 public item；必要时 debug trace | 无或 `status_trace / status` | 默认不显示内部路径 | 普通 search/read/list/stat/write/edit 不可访问 runtime private path |
| T11 | 模型基于工具结果继续 | 工具 observation 回到模型上下文 | `assistant_text_delta` 或后续工具事件 | `assistant_body / body` 或 `tool_control / control` | 按 offset 与工具轨迹交错显示 | 模型正文才可进入 body；工具结果自身不可进 body |
| T12 | 模型正文定稿 | 模型 final answer 完成 | `assistant_text_final` | `assistant_body / body` | 正文 finalized | 必须早于 commit checked/ack 和 turn terminal |
| T13 | commit 检查 | session output commit gate 检查正文 | `session_output_commit_checked` | `runtime_commit / commit` | commit trace/status | 不能生成正文，只记录 commit gate 状态 |
| T14 | commit 收口 | commit 成功/失败/跳过 | `session_output_commit_ack` / `failed` / `skipped` | `runtime_commit / commit` | 成功则正文 committed；失败则 pinned/trace | failed 不能静默吞掉 |
| T15 | task bridge 结束 | TaskRun terminal 或 bridge snapshot terminal | `task_bridge_terminal` | `turn_anchor_terminal / terminal` | trace 关闭 task bridge | 如果 terminal 携带 final_answer 但缺 assistant output/commit，只生成 diagnostic，不合成正文 |
| T16 | turn 结束 | chat stream 关闭 | `turn_completed` | `turn_anchor_terminal / terminal` | turn terminal；清理 transient 状态 | 只能关闭 turn，不能补正文 |
| T17 | replay/hydrate | SSE 重连、刷新会话 | 重放 lossless frames | 原事件 family/channel | 前端按 frame id 和 event_offset 合并 | 不重复渲染，不用到达时间重排 |

## 4. 工具生命周期时序

工具轨迹必须是一条稳定生命周期，不允许 start/return 拆成两条互不相干的 UI 记录。

| 顺序 | 事件 | 必备身份字段 | UI 操作 | 错误处理 |
| --- | --- | --- | --- | --- |
| 1 | `tool_call_requested` | `tool_call_id`、`request_id`、`tool_name` | 创建工具运行项 | 缺 `tool_call_id` 进入 protocol diagnostic |
| 2 | `tool_permission_decided` | `tool_call_id`、`permission_decision_id` | 更新权限状态；需审批则 pinned | 未绑定 request 或 offset 过早则 diagnostic |
| 3 | `tool_item_started` | `tool_call_id`、`permission_decision_id`、`tool_lifecycle_id` | 标记同一工具项 running | 未绑定 allowed permission 则 diagnostic |
| 4 | `tool_item_completed` | `tool_call_id`、`permission_decision_id`、`tool_lifecycle_id` | 同一工具项 done/error；成功可 retire transient | 未绑定 started lifecycle 则 diagnostic |

身份分权：

| 字段 | 权威来源 | 用途 | 禁止 |
| --- | --- | --- | --- |
| `tool_call_id` | 模型 `tool_call.id`，缺失时 runtime 生成一次 | 工具生命周期主身份 | 与 `request_id` 随意互换 |
| `request_id` | 模型 action request | trace/action ref | 冒充工具主身份 |
| `permission_decision_id` | permission/admission 层 | 授权证明 | 替代 `tool_call_id` |
| `tool_lifecycle_id` | executor 单次执行实例 | 刷新/hydrate 合并执行项 | 改变模型工具请求身份 |

## 5. Active Steer / Control 时序

| 时序 | 场景 | 边界裁决 | 公开投影 | UI 表现 | 禁止 |
| --- | --- | --- | --- | --- | --- |
| S1 | active task 运行中，用户发送普通补充 | `USER_STEER_SIGNAL` | `active_task_steer_accepted`，必要时 steer consumed status | 控制/任务状态显示“补充已纳入” | 新建 TaskRun replacement |
| S2 | active task 运行中，用户发送 stop/pause/continue | `CONTROL_SIGNAL` | control receipt / runtime control status | 控制面显示等待安全边界或已执行 | 进入普通模型回答 |
| S3 | expected active turn mismatch | blocked receipt | `status_trace` diagnostic 或 pinned block | 告知控制目标失效 | append 到错误任务 |
| S4 | active task 已 terminal/stopped | blocked receipt | `status_trace` diagnostic 或 pinned block | 不再允许 steer/control | 复活旧任务或重绑最近任务 |

steer/control 的公开投影只能说明控制事实，不能成为 assistant 正文。

## 6. Runtime 私有产物时序

| 阶段 | Runtime 行为 | 普通 workspace 工具 | 专用 rehydration 工具 | 公开投影 |
| --- | --- | --- | --- | --- |
| R1 | 写入 replacement JSON | 不可 search/read/list/stat/exists | 可通过 trusted ref 读取 | 无 |
| R2 | 写入 tool result store | 不可 search/read/list/stat/exists | 可通过 tool result ref 读取 | 无 |
| R3 | 模型需要恢复上下文 | 不能猜路径读取 | 使用 `replacement_id + task_run_id + trusted path` | 最多显示安全状态摘要 |
| R4 | 私有路径误入 observation 文本 | 不作为主防线 | 不适用 | projection 层净化文本，但保留工具活动 |

硬边界：

```text
runtime-private boundary
> presentation scrub
```

也就是说，主要防线必须在 workspace 文件能力边界；投影层只做兜底净化，不负责隐藏整条工具活动。

## 7. Commit / Terminal 收口时序

| 顺序 | 事件 | 允许结果 | 禁止结果 |
| --- | --- | --- | --- |
| 1 | `assistant_text_final` | 正文 finalized | 由 terminal/snapshot 代替正文 |
| 2 | `session_output_commit_checked` | 记录 commit gate 检查 | 生成或改写正文 |
| 3 | `session_output_commit_ack` | 标记正文 committed，清理 transient 状态 | 提前于 final answer |
| 4 | `session_output_commit_failed` | pinned/trace 暴露失败 | 静默隐藏失败 |
| 5 | `task_bridge_terminal` | 关闭后台 task bridge | 用 task terminal final_answer 合成正文 |
| 6 | `turn_completed` | 关闭 turn | 补正文、覆盖正文、从 snapshot 反推正文 |

replacement stop closeout 的规则：

```text
replacement stop closeout
-> task projection terminal/stopped
-> no SESSION_CANONICAL_MESSAGE
-> no assistant body
```

## 8. 前端排序与合并规则

| 数据 | 前端处理 | 排序依据 |
| --- | --- | --- |
| body delta/final | 写入 `ledger.body`，记录 `body.source_offsets` | body block 使用最新 body `event_offset` |
| tool/status/pinned/final item | 写入 `ledger.timeline` 和对应集合 | item `eventOffset` |
| commit | 写入 `ledger.commit`，必要时 retire transient | commit frame `event_offset` |
| terminal | 写入 `ledger.terminal`，trace/pinned | terminal frame `event_offset` |
| replay frame | 通过 frame id / item id upsert | 原始 `event_offset` |

消息块合并规则：

```text
assistant body block
+ public timeline items
-> sort by eventOffset
-> render in one assistant message surface
```

禁止使用：

- SSE 到达时间。
- `Date.now()`。
- task source offset。
- 中文标题或文案。
- terminal snapshot。
- semantic key 作为主排序依据。

## 9. Lossless 要求

以下事件必须 lossless，可 replay，不允许被普通状态进度替代：

```text
assistant_text_delta
assistant_text_final
assistant_stream_repair
tool_call_requested
tool_permission_decided
tool_item_started
tool_item_completed
session_output_commit_checked
session_output_commit_ack
session_output_commit_failed
session_output_commit_skipped
chat_turn_bound
task_bridge_started
task_bridge_terminal
turn_completed
protocol_diagnostic
```

以下事件可 best-effort，但不能影响正文、工具完成、commit、terminal 的最终一致性：

```text
runtime_status
runtime_step_summary
ordinary progress note
non-terminal trace-only diagnostic
tool stdout/stderr preview
```

## 10. 验收检查表

| 检查项 | 通过标准 |
| --- | --- |
| 正文实时投影 | `assistant_text_delta` 到达后正文块立即追加，且按 `event_offset` 排序 |
| 正文最终定稿 | `assistant_text_final` 早于 commit checked/ack/terminal |
| 工具轨迹可见 | requested/permission/started/completed 都进入工具控制轨迹或 trace/pinned |
| 工具轨迹同一生命周期 | completed 更新同一 `tool_call_id/tool_lifecycle_id`，不生成重复返回项 |
| 乱序工具事件 | 产生 protocol diagnostic，不进入正常工具轨迹 |
| 用户 steer | active task 补充进入当前任务，不创建 replacement |
| control 指令 | pause/stop/continue 进入控制面，不成为 assistant 正文 |
| runtime 私有路径 | 普通 workspace 工具搜不到、读不到、列不出、写不了 |
| terminal final_answer | 缺正文/commit 时只出 diagnostic，不合成正文 |
| SSE replay | lossless frames 重放后不丢正文 final、工具 completed、commit、terminal |
| 前端排序 | body block 与工具/status item 严格按 `eventOffset` 混排 |
| 无双权威 | 前端不再用 raw stream、terminal snapshot 或到达时间重判正文/工具时序 |

---

## 11. 一句话基准

新路线的时序基准是：

```text
用户意图先过 active boundary；
模型正文与工具控制进入同一个 public event clock；
正文只由 assistant body 事件产生；
工具只由 lifecycle 事件呈现；
commit/terminal 只负责收口；
runtime 私有产物永远不成为普通 workspace 或用户正文。
```
