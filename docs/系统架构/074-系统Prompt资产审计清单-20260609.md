# 系统 Prompt 资产审计清单

日期：2026-06-09

## 结论

本轮审计覆盖当前项目中会被模型看到、会生成模型可见文本、或会影响 prompt 装配的主要 prompt 来源：

- `backend/prompt_library/*`
- `storage/prompt_library/prompt_resources.json`
- `backend/context_system/compaction/*`
- `backend/memory_system/maintenance.py`
- `backend/memory_system/durable.py`
- `backend/runtime/context_management/*`
- `backend/capability_system/skills/builtin/*/SKILL.md`
- `backend/capability_system/mcp/server/server.py`
- `backend/context_system/policy/runtime_models.py`
- `backend/prompting/*`
- 散落 utility prompt：协议修复、RAG finalizer、搜索证据提炼、只读计划员、只读验收员、标题生成、历史摘要。

总体判断：当前 prompt 文本质量明显优于旧式“这是 runtime 节点，用于执行某流程”的写法。系统层、agent work role、生命周期、worker、工具说明、写作图节点大多已经是面向 agent 的直接职责 prompt。

主要问题不是“所有 prompt 都要重写”，而是：

1. prompt 来源仍分散在多个系统，缺少唯一 `PromptCompositionGraph` 装配主权。
2. lifecycle、skill、utility、graph node、recovery boundary 的 slot 分类还没有固定。
3. 少数 graph node role 太短，尤其 `prompt.task_graph.general.main_conversation.graph_node.role` 不符合 agent prompt 标准。
4. 语义压缩 prompt 存在多处重复：`agent_prompts`、`packs`、`ContextCompactor._semantic_compaction_instructions()`、`ModelRuntime.summarize_history()`。
5. `backend/prompting/*` 仍保留旧 prompt builder / professional profiles，测试和任务合同仍引用其中一部分，后续必须迁移或删除旧链路。
6. 若 graph node role 缺失，当前仍可能生成弱 fallback prompt；目标架构应暴露配置错误，而不是继续执行弱 prompt。

## 审计标准

一个合格的 agent prompt 至少需要通过以下检查：

| 检查项 | 标准 |
|---|---|
| 身份 | 是否直接告诉 agent “你是谁”，而不是描述代码节点用途。 |
| 职责 | 是否说明只负责什么、要完成什么。 |
| 边界 | 是否说明不负责什么、不能做什么、不可扩大哪些范围。 |
| 输入 | 是否说明输入来源、授权范围、哪些材料只能作为线索。 |
| 输出 | 是否说明输出格式、必填字段、是否必须 JSON、是否要给裁决。 |
| 质量标准 | 是否给出可检查标准，而不是空泛“高质量”。 |
| 失败处理 | 是否说明证据不足、输入冲突、权限缺失、无法完成时怎么处理。 |
| 权威层级 | 是否避免让旧摘要、记忆、工具内容覆盖用户最新请求和上级规则。 |
| 生命周期 | 是否可归入稳定 slot、动态 slot、utility prompt 或 recovery boundary。 |
| 缓存稳定性 | 是否避免把 run_id、turn_id、时间戳、权限瞬态、token 压力写进 stable prompt。 |

## 高优先级问题

| 优先级 | 问题 | 证据 | 建议 |
|---|---|---|---|
| P1 | 主会话通用 graph node role 不合格 | `prompt.task_graph.general.main_conversation.graph_node.role` 只有一句“主会话默认承接通用任务...” | 重写为完整角色 prompt：身份、职责、何时接管、何时转专用链、输出/阻塞标准。 |
| P1 | 语义压缩 prompt 多权威重复 | `agent.context_compactor_agent.semantic_compaction.work_role`、`runtime.semantic_compaction`、`ContextCompactor._semantic_compaction_instructions()`、`ModelRuntime.summarize_history()` | 统一为一个 `context_compactor` prompt 资源，运行时只引用 prompt ref。 |
| P1 | 旧 `backend/prompting/*` 仍可影响任务合同 | `task_requirement_contracts.py` 引用 `strategy_prototypes`，测试引用 `build_static_prompt*` | 将 professional profiles 迁移到 prompt library 或任务 profile registry；删除旧 builder/cache 主链路。 |
| P1 | graph node prompt 缺失 fallback 过弱 | `context_materializer._agent_instruction()` fallback 为“请根据你的角色职责完成当前节点任务...” | 缺失 role prompt 时应 fail fast 或要求补齐配置，不能弱提示继续跑。 |
| P2 | lifecycle prompt 分类为 environment prompt | `environment.general.lifecycle.*` 目前全部 `cache_scope=static_environment` | 在新装配系统中改入 `lifecycle_stable` slot，避免和 environment base 混淆。 |
| P2 | skill registry snapshot 抽空 fenced 示例 | `SKILLS_SNAPSHOT.md` 的部分“适合的主 Agent 指令风格”为空 | snapshot 只做 admin 展示可以接受；若模型使用 registry prompt view，需要保留示例或从原始 `SKILL.md` 展开。 |
| P2 | utility prompt 散落调用点 | action repair、title、RAG finalizer、distiller、readonly planner/verifier | 新增 `utility_prompt` / `repair_prompt` 资源类型，纳入 manifest 和 accounting。 |

## 推荐升级顺序

1. 建立 prompt asset 分类：
   `system_foundation`、`runtime_protocol`、`runtime_rule`、`agent_work_role`、`worker_role`、`lifecycle_prompt`、`environment_base`、`resource_overlay`、`tool_guidance`、`skill_candidate`、`selected_skill_body`、`graph_node_role`、`utility_prompt`、`repair_prompt`、`recovery_boundary_text`。
2. 新增 prompt lint：
   身份、职责、边界、输入、输出、失败处理、禁止旧摘要越权、禁止 runtime volatile 字段进入 stable prompt。
3. 先修 P1：
   重写 `general.main_conversation` graph prompt；合并语义压缩 prompt；禁止 graph node role 缺失 fallback；迁移旧 `professional_profiles`。
4. 再修 P2：
   graph node 路由/提交/封存/汇编类 prompt 补输出合同；lifecycle prompt 改 slot；utility prompt、durable memory recall selector、MCP prompt endpoint 入库。
5. 最后清理旧链路：
   `backend/prompting/*` 不再参与主 runtime；旧测试改为验证 `PromptCompositionGraph` 和 `ProviderPayloadManifest`。
