# 协议修复状态误显示回归修复记录

日期：2026-06-15

## 现象

前端主消息中仍显示：

```text
当前步骤输出格式不完整，正在自动修正后继续。
```

## 根因

`146-任务协议修复反馈与等待占位投影链路修复记录` 曾把 `runtime.protocol_repair` 从 trace-only 改为 `visible_live` 状态，覆盖了 `142-协议修复提示误投影修复记录` 的正确边界。

这条信号属于系统给模型的协议修复 observation，不是模型正文、不是工具窗口、也不是用户需要看到的公开进度。

## 修复

- `backend/harness/runtime/projection/projector.py`
  - `runtime.protocol_repair` 回到 `_TRACE_ONLY_RUNTIME_STEP_SOURCES`。
  - 删除 `protocol_repair_status` 可见状态分支。
  - trace frame 的 `title/text/detail` 对 protocol repair 置空，防止未来误渲染 hidden trace 时泄露系统修复文案。
- `backend/tests/public_projection_contract_test.py`
  - 删除“protocol repair 可见”的错误保护。
  - 改为验证 `runtime.protocol_repair` 必须是 `slot=trace`、`main_visibility=hidden`、`retention=trace`，且 frame 中不包含“当前步骤输出格式不完整”和 repair instruction。

## 正确线路

```text
模型 action JSON 不合法
-> runtime 生成 protocol repair observation
-> observation 回喂模型
-> public_projection_frame 只保留 hidden trace refs
-> 用户侧等待模型重新给出合法 public_progress_note / public_action_state
```

等待占位 `model_wait_placeholder` 可以显示；协议修复原因不能显示。
