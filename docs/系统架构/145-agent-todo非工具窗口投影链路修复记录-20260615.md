# agent_todo 非工具窗口投影链路修复记录

日期：2026-06-15

## 结论

`agent_todo` 不应该做成常规工具窗口。

它在 runtime 内部可以作为模型维护任务清单的工具单元存在，但在公共投影里不是“agent 调动系统执行命令”的控制轨迹。公共工具窗口只承载 shell、读写文件、搜索、浏览器、补丁等系统执行动作。

## 目标线路

| 输入信号 | 后端分流 | 投影帧 | 前端结果 |
| --- | --- | --- | --- |
| 模型正文 | 保持 `body_append/body_finalize` | `slot=body` | 正文显示 |
| 普通工具请求 | 保持工具生命周期 | `event_family=tool_control`，携带 `tool_call_id/tool_name` | 工具窗口，可折叠 |
| `agent_todo` 请求/权限/开始 | 从公开工具生命周期剔除 | hidden trace，`event_family=status_trace`，不带 `tool_call_id/tool_name` | 不显示工具窗口 |
| `agent_todo` 完成观察 | 折成计划状态摘要 | `status_kind=todo_plan_status`，`slot=status` 或失败 pinned | 计划状态，不显示命令窗口 |
| `agent_todo operation=view` | 不作为工具轨迹展示 | 无公开工具事件 | 不显示“查看任务清单”窗口 |

## 代码落点

- `backend/api/chat.py`
  - `_tool_call_requested_items()` 直接跳过 `agent_todo`，不再产出 `tool_call_requested/tool_permission_decided` 公开工具事件。
  - `_safe_tool_observation_text()` 对 `agent_todo` 只提取干净计划摘要，禁止 JSON/operation/tool id 泄露到用户界面。

- `backend/harness/runtime/projection/projector.py`
  - `ProjectionLifecycleState` 在工具生命周期入口识别 `agent_todo`，不让它进入普通工具生命周期账本。
  - `agent_todo` request/permission/started 只进入 hidden trace。
  - `agent_todo` completed 只生成 `todo_plan_status`，显式覆盖为 `event_family=status_trace`、`channel=status`，并且不携带 `tool_call_id/tool_name`。

- `backend/harness/runtime/projection/items.py`
  - 移除 `agent_todo -> plan` 的工具动作分类残留，避免它重新被当作工具动作标签。

- `frontend/src/components/chat/ChatMessage.tsx`
  - 保留 `agent_todo` 防御过滤，防止历史帧或旧缓存误入工具窗口。

- `frontend/src/components/chat/PublicTimelineActivity.tsx`
  - 保留 `agent_todo` 防御过滤，作为历史兼容保护，不作为主权威。

## 结构探针结果

```text
agent_todo_request_public_events = []

todo completion frame:
  event_family = status_trace
  channel = status
  slot = status
  main_visibility = visible_live
  status_kind = todo_plan_status
  tool_call_id = None
  tool_name = None

todo request frame:
  event_family = status_trace
  channel = status
  slot = trace
  main_visibility = hidden
  status_kind = todo_plan_trace
  tool_call_id = None
  tool_name = None
```

## 保留边界

`agent_todo` 仍然可以存在于 runtime 内部工具系统中，因为模型需要用它维护执行计划。修复的是公共投影身份：它不能占用工具窗口、工具计数、工具折叠组或工具命令行 UI。
