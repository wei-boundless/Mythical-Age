# 089-图任务前台运行控制台重做计划书

日期：2026-06-09

## 1. 结论

现有图任务前台不合格，原因不是控件少，而是前台没有被当成独立生产系统设计。

当前页面仍然复用了 `GraphTaskWorkspace` / 图编辑工作区外壳，前台只是把 `GraphTaskInstanceWorkbench` 塞进旧编辑页结构里。这样会导致：

- 图任务前台启动时混入任务系统配置加载，首屏重；
- 页面滚动链路受旧编辑页容器约束，出现内容不能顺畅拖动的问题；
- 没有中心运行画布，用户看不到任务图正在如何推进；
- 节点对话、节点输出、项目库、项目管理没有形成前台主工作流；
- 图任务的通用性没有表现出来，缺少“图定义加载 / 项目实例加载”的两级管理；
- 文件中心和会话列表是附属面板，不是项目生产资产管理。

因此下一步不应继续修补旧 `GraphTaskInstanceWorkbench`，而应重做图任务前台。

## 2. 前台目标

图任务前台的定位是：多 agent 图任务运行控制台。

它不是图定义编辑器，也不是任务系统配置页。它只负责：

1. 选择图任务定义；
2. 管理图任务项目实例；
3. 启动、监控、恢复运行；
4. 以中心画布展示节点运行状态；
5. 显示每个节点的对话和输出；
6. 管理项目正式库、运行产物和输入资料；
7. 让用户知道当前任务跑到哪里、哪里卡住、下一步该做什么。

## 3. 信息架构

新前台分为四个固定区域。

### 3.1 左侧：图任务与项目管理

两级系统管理必须明确：

- 图任务定义层：
  - 图名称、图类型、发布状态；
  - 可搜索、可筛选；
  - 选择图后加载该图的项目实例；
  - 不加载图编辑器，不加载拓扑配置器。

- 项目实例层：
  - 一个图任务可有多个项目实例；
  - 项目实例是运行容器；
  - 支持创建、选择、刷新；
  - 显示运行状态、最近更新时间、当前运行 id；
  - 状态筛选：全部、运行中、需处理、待启动、完成。

### 3.2 中心：运行监控画布

这是前台重心。

中心画布不是编辑拓扑，而是运行拓扑视图：

- 节点卡片展示：
  - 节点名称 / node id；
  - 节点角色；
  - 当前状态：ready、running、completed、failed、blocked；
  - 最近输出摘要；
  - 所属会话入口；
  - 产物数量；
  - 错误或阻塞原因。

- 边只表现运行关系，不允许编辑：
  - 已完成传递；
  - 正在等待；
  - 阻塞；
  - 可追踪从哪个节点交给哪个节点。

- 画布默认适配通用图任务：
  - 有拓扑时按拓扑展示；
  - 暂无完整拓扑时按运行节点状态分组展示；
  - 不把写作任务写死在 UI 里。

#### 3.2.1 拓扑展示策略

前台默认采用“运行焦点自动跟随”方案，而不是默认完全展开整张图。

理由：

- 图任务前台的主要用户目标是监控、接管、审查当前运行，而不是编辑图结构；
- 大图完全展开会让写作、研究、审校这类连续任务变成结构墙，用户很难判断当前要看哪里；
- 子图、阶段、节点组天然是运行时关注单位，前台应随 active subgraph / active node group 自动切换；
- 完全展开更适合作为调试、总览、设计审查模式，不能作为生产前台默认视图。

因此本轮前台只交付默认跟随模式：

- 中心区域显示当前活跃子图或活跃节点组；
- 节点卡优先显示运行中、阻塞、失败节点；
- 节点输出面板随选中节点切换；
- 项目正式库独立在运行区下方，不挤压主画布。

后续如果后端提供稳定的完整拓扑运行投影、子图层级、边状态和布局坐标，再增加“完整展开 / 调试总览”显式模式切换。

### 3.3 右侧：节点对话与输出

用户必须能看到每个节点到底在干什么。

右侧面板按当前选中节点显示：

- 节点会话；
- 最新 assistant 输出；
- 当前 public timeline；
- 工具调用摘要；
- 节点产物；
- 进入完整会话按钮。

没有选中节点时显示当前运行的主会话和最近活跃节点。

### 3.4 底部或独立页签：项目正式库

项目库不是附属文件框，而是项目生产资产区。

最小可用版本包括：

- 输入资料：
  - brief；
  - requirements；
  - 用户上传或手写材料。

- 正式库：
  - world / characters / outline / chapters / review 等通用目录；
  - 目录和文件搜索；
  - 编辑 / 预览；
  - 保存。

- 运行产物：
  - 节点产物列表；
  - 可载入编辑器；
  - 可归入正式库。

暂不伪造后端没有的能力：

- 版本历史；
- diff；
- 删除；
- 重命名；
- 归档审批。

这些后续要做后端支持。

## 4. 组件改造方案

新增独立前台组件，不继续把旧实例工作台当主页面。

建议文件：

- `frontend/src/components/workspace/views/task-graph-foreground/GraphTaskForegroundView.tsx`
- `frontend/src/components/workspace/views/task-graph-foreground/GraphTaskProjectRail.tsx`
- `frontend/src/components/workspace/views/task-graph-foreground/GraphTaskRunCanvas.tsx`
- `frontend/src/components/workspace/views/task-graph-foreground/GraphTaskNodeInspector.tsx`
- `frontend/src/components/workspace/views/task-graph-foreground/GraphTaskProjectLibrary.tsx`

保留 `GraphTaskWorkspace` 作为任务系统配置 / 图定义编辑入口。

前台入口改为：

- `GraphTaskSystemView` 直接渲染新的 `GraphTaskForegroundView`；
- 不再通过 `GraphTaskWorkspace(surface="operations")` 进入旧编辑壳；
- 任务系统页仍然使用 `GraphTaskWorkspace(surface="configuration")`。

## 5. 数据加载策略

前台必须快。

加载顺序：

1. 加载图任务定义列表；
2. 默认选择推荐图；
3. 只加载当前图的项目实例；
4. 选择项目后加载该项目 monitor；
5. monitor 加载完成后再加载文件树；
6. 节点会话和产物从 monitor 渲染；
7. 跨图实例队列只做后台补充，不参与首屏阻塞。

禁止：

- 进入前台时加载图编辑器；
- 进入前台时加载任务系统完整配置；
- 进入前台时同步加载所有图实例；
- 空状态显示大面积同步中卡片。

## 6. 页面滚动与布局规则

新前台必须有自己的滚动系统：

- 页面根容器：固定高度内 `overflow: auto`；
- 左侧项目栏：内部滚动；
- 中心画布：独立滚动或自适应；
- 右侧节点面板：内部滚动；
- 项目库：独立区域，可滚动；
- 不依赖 `task-graph-editor-page` 的旧编辑器滚动规则；
- 桌面首屏必须看到：
  - 图定义 / 项目实例；
  - 当前运行画布；
  - 节点输出面板入口；
  - 项目库入口。

移动端：

- 单列；
- 顺序为项目管理、运行画布、节点输出、项目库；
- 不允许横向溢出；
- 允许纵向滚动。

## 7. 实施步骤

### 阶段 1：建立新前台骨架

- 新建 `task-graph-foreground` 目录；
- 实现独立 `GraphTaskForegroundView`；
- 接入 `listGraphTasks`、`listGraphTaskInstances`、`getGraphTaskInstanceMonitor`、`getGraphTaskInstanceFileTree`；
- `GraphTaskSystemView` 改为直接使用新前台；
- 保留旧 `GraphTaskInstanceWorkbench`，但不再作为主前台入口。

### 阶段 2：实现中心运行画布

- 从 monitor 解析节点状态；
- 优先使用 `active_node_runtime_views`、`graph_monitor`、`summary`、`node_sessions`；
- 渲染节点运行卡；
- 支持点击节点后右侧检查器切换；
- 无拓扑时按状态分组展示。

### 阶段 3：实现节点输出检查器

- 展示当前节点会话；
- 展示最近输出和更新时间；
- 展示节点产物；
- 支持打开节点会话。

### 阶段 4：实现项目正式库

- 文件树、搜索、编辑、预览、保存；
- 运行产物搜索和载入；
- 常用项目路径模板；
- 保持所有写入走现有项目文件 API。

### 阶段 5：验证和清理

- 删除前台对旧 operations surface 的依赖；
- 清理不再使用的旧 CSS 分支；
- 保留任务系统配置页的图编辑能力；
- 浏览器验证：
  - 桌面可滚动；
  - 移动无横向溢出；
  - 首屏有运行画布；
  - 节点输出可切换；
  - 项目库可编辑。

## 8. 验收标准

新前台必须满足：

- 用户一进图任务，就知道自己在管理哪个图、哪个项目；
- 用户能在中心画布看到节点运行状态；
- 用户能点击节点查看该节点会话和输出；
- 用户能管理项目正式库；
- 用户能启动项目运行；
- 页面能正常拖动；
- 前台不加载图编辑器；
- 前台不绑定环境；
- 空状态不阻塞首屏；
- 桌面和移动端都没有横向溢出。

## 9. 不在本次伪造的能力

以下能力如果后端未提供，本次不假装实现：

- 节点级暂停、重跑、跳过；
- 文件版本历史；
- 文件 diff；
- 产物正式归档审批流；
- 节点真实拓扑坐标编辑；
- 多运行并发控制策略。

这些能力后续应作为后端运行控制和项目库模型升级处理。
