# Agent 显示流程、信号与前端投影规则

日期：2026-06-14

本文只回答三件事：

1. agent 运行时到底有哪些显示流程。
2. 每个流程对应哪些后端/public projection 信号。
3. 前端应该怎么显示、怎么投影、禁止显示什么。

## 0. 先定义显示表面

| 表面 | 作用 | 用户看到什么 | 不该看到什么 |
| --- | --- | --- | --- |
| 用户消息 | 用户输入 | 用户原文 | runtime ID、工具事件 |
| 正文区 | 大模型对用户说的话 | 自然语言反馈、阶段说明、最终回答 | 工具调用号、offset、完整工具参数、重复路径 |
| 工具窗口 | agent 调动系统的控制轨迹 | 一个可折叠控制台窗口：动作、目标、状态、命令行、返回摘要 | 模型正文、生命周期四行、重复反馈、无意义内部 ID |
| 状态栏 | 会话连接/运行状态 | 在线、断线、重连、等待、运行中 | 模型正文、工具输出 |
| 调试/trace | 追溯用 | 完整事件、ID、offset、调用链 | 默认主聊天界面不展示 |

## 1. 完整显示流程总表

| 顺序 | 流程 | 后端原始信号 | public projection 信号 | 前端显示 | 投影归位 | 禁止 |
| --- | --- | --- | --- | --- | --- | --- |
| 1 | 用户输入入场 | user message / turn 创建 | anchor fields | 用户消息气泡 | `turn_id` 归属当前轮 | 不显示 runtime 字段 |
| 2 | 建立运行锚点 | `single_agent_turn_started`, `harness_run_started` | trace/hidden 或状态锚点 | 通常不显示，必要时状态栏显示“运行中” | `turn_id + stream_run_id + turn_run_id` | 不当正文 |
| 3 | 模型阶段反馈 | `step_summary_recorded`, `runtime_step_summary` | `runtime_step_summary` -> `status_trace` | 正文区，作为模型自然语言反馈 | 按 `event_offset` 插入正文流 | 不进工具窗口 |
| 4 | 模型正文流 | `assistant_text_delta`, `assistant_text_final`, `assistant_stream_repair` | `body_append`, `body_finalize` | 正文区 | `bodyBlocks[firstOffset/lastOffset]` | 不和工具轨迹混成一块 |
| 5 | 模型请求工具 | `model_action_admission` | `tool_call_requested` | 工具窗口出现 | `tool_call_id/tool_lifecycle_id` | 不当正文反馈 |
| 6 | 工具权限/准入 | `model_action_admission_checked` | `tool_permission_decided` | 更新同一工具窗口状态 | 同一 `tool_call_id` | 不另开窗口 |
| 7 | 工具开始执行 | `tool_item_started` | `tool_item_started` | 同一工具窗口变为运行中 | 同一 `tool_lifecycle_id` | 不重复显示路径 |
| 8 | 工具返回观察 | `tool_observation`, `turn_tool_observation_recorded`, `task_tool_observation_recorded` | `tool_item_completed` | 同一工具窗口变为完成/失败 | 同一 `tool_call_id/tool_lifecycle_id` | 不按工具名/标题新建 |
| 9 | 模型根据观察继续说话 | `step_summary_recorded` 或正文流 | `runtime_step_summary` / body frame | 正文区 | 按 offset 排在工具后 | 不吞掉反馈 |
| 10 | turn 终止 | `done`, `error`, `stopped` | `turn_completed` | 失败/停止可有状态提示 | turn anchor | 不伪造成模型正文 |
| 11 | 最终提交 | commit checked/ack/failed/skipped | `commit_ack`, `commit_failed` | 成功后主消息只显示最终正文 | commit key + body state | 工具轨迹不抢主位 |

## 2. 正文信号必须怎么投

### 2.1 正文信号清单

| 信号 | 字段 | 含义 | 前端显示 |
| --- | --- | --- | --- |
| `assistant_text_delta` | `content` | 模型正文增量 | 追加到正文区 |
| `assistant_text_final` | `content` | 模型最终正文 | finalize 正文 |
| `assistant_stream_repair` | `replacement_content` | 正文修复 | 替换/修复正文 |
| `runtime_step_summary` | `public_progress_note` | 模型给用户的阶段反馈 | 正文区 |
| `runtime_step_summary` | `current_judgment` | 模型当前公开判断 | 正文区 |
| `runtime_step_summary` | `next_action` | 模型下一步公开说明 | 正文区，可作为下一段 |

### 2.2 正文投影规则

| 规则 | 说明 |
| --- | --- |
| 主权威 | 正文主权威是 model-visible natural language，不是工具事件。 |
| 排序 | 按 public `event_offset` / body block offset 排序。 |
| 合并 | 连续正文可合并为同一 body block。 |
| 去重 | 同一 `compactText` 的反馈只显示一次。 |
| 终态 | `bodyState=finalized/committed` 后，主消息最终态只显示正文。 |

### 2.3 正文禁止项

- 禁止显示 `tool_call_id`。
- 禁止显示 `tool_lifecycle_id`。
- 禁止显示 `permission_decision_id`。
- 禁止显示 public frame offset。
- 禁止把工具路径、命令参数、调用范围当成正文。
- 禁止为了去重直接砍掉模型反馈来源。

## 3. 工具信号必须怎么投

### 3.1 工具生命周期

```text
tool_call_requested
-> tool_permission_decided
-> tool_item_started
-> tool_item_completed
```

### 3.2 工具信号表

| 信号 | 关键字段 | 前端应该做什么 | 禁止 |
| --- | --- | --- | --- |
| `tool_call_requested` | `tool_call_id`, `tool_name`, `target` | 创建或更新一个工具窗口 | 不能创建正文 |
| `tool_permission_decided` | `permission_decision`, `permission_decision_id` | 更新同一窗口状态 | 不能另开窗口 |
| `tool_item_started` | `tool_lifecycle_id`, `tool_call_id` | 同一窗口显示运行中 | 不能重复显示同一路径 |
| `tool_item_completed` | `tool_lifecycle_id`, `tool_call_id`, `observation/error` | 同一窗口显示完成/失败 | 不能按 title 新建窗口 |

### 3.3 工具窗口显示规则

折叠态只显示：

```text
<工具动作> <目标> <状态>
```

例子：

```text
读取 mario.html 运行中
搜索 project-mario-full-fix-plan.md 已完成
更新 frontend/src/components/chat/PublicTimelineActivity.tsx 已完成
```

展开态只显示一个控制台窗口：

```text
$ <tool_name> <target/args>
<系统返回摘要或错误摘要>
```

展开态允许：

| 内容 | 位置 | 说明 |
| --- | --- | --- |
| 系统调用命令行 | 控制台第一行 | 例如 `read_file frontend/src/lib/api.ts`、`search_text "timeline" frontend/src/lib/api.ts` |
| 系统返回摘要 | 控制台后续行 | 成功/失败摘要，不展示大段原始内容 |
| 完整输出 | trace/debug | 不进主聊天默认视图 |
| 调用 ID | trace/debug | 不给普通用户看 |
| offset | trace/debug | 不给普通用户看 |

展开态禁止再拆成 `目标/参数/观察` 表格。那是内部字段，不是用户理解 agent 调动系统的方式。

### 3.4 工具窗口合并规则

| 场景 | 合并键 | 应显示 |
| --- | --- | --- |
| request -> started -> completed | 主键 `tool_call_id`，缺失时才用 `tool_lifecycle_id` | 一个窗口状态更新 |
| 同一工具连续读取同一文件但不同调用 | 不合并为一个生命周期 | 可显示多次，但必须简洁 |
| 同一调用重复收到 replay frame | `frame_id/source_event_id` 去重 | 不重复显示 |
| retry/attempt | `tool_call_id` 下的不同 `tool_lifecycle_id` | 同窗口 attempt 或独立 attempt，必须清楚 |
| completed 缺 request | lifecycle diagnostic | 不进主窗口，进入诊断/trace |

### 3.5 工具生命周期到 UI 的更新规则

后端可以有四个 lifecycle 信号，但前端不能把四个信号投成四行。前端必须按同一个可见窗口更新：

| 后端信号 | 前端行为 | 是否新建可见窗口 |
| --- | --- | --- |
| `tool_call_requested` | 没有窗口时创建工具窗口，显示动作、目标、运行中 | 是 |
| `tool_permission_decided` | 只更新同一窗口的等待/阻塞状态；允许通过时默认不显示权限行 | 否 |
| `tool_item_started` | 只更新同一窗口为运行中，可补充控制台命令行 | 否 |
| `tool_item_completed` | 只更新同一窗口为已完成/失败，可补充控制台输出摘要 | 否 |

可见窗口的标题只允许表达：

```text
<工具动作> <目标> <状态>
```

正常主视图禁止显示：

- `tool_call_id`
- `tool_lifecycle_id`
- `permission_decision_id`
- `event_offset`
- `source_event_type`
- `请求 #11 -> #14` 这类内部时序串

## 4. 正文和工具同时返回时怎么排

agent 一轮里正文和工具可以同时返回。正确排序不是二选一，而是同一时间线上分表面显示。

后端信号时间线示例：

| offset | 信号 | 前端显示 |
| --- | --- | --- |
| 10 | `runtime_step_summary` | 进入正文表面 |
| 11 | `tool_call_requested` | 创建工具窗口 A |
| 12 | `tool_permission_decided` | 更新工具窗口 A，不新建行 |
| 13 | `tool_item_started` | 更新工具窗口 A，不新建行 |
| 14 | `tool_item_completed` | 更新工具窗口 A，不新建行 |
| 15 | `tool_call_requested` | 创建工具窗口 B |
| 16 | `tool_permission_decided` | 更新工具窗口 B，不新建行 |
| 17 | `tool_item_started` | 更新工具窗口 B，不新建行 |
| 18 | `tool_item_completed` | 更新工具窗口 B，不新建行 |
| 19 | `runtime_step_summary` | 进入正文表面 |
| 30 | `assistant_text_final` | 进入正文表面并收口 |

前端可见结果应该是：

```text
正文：我先检查修复计划和 mario.html。
工具窗口：读取 project-mario-full-fix-plan.md 已完成
工具窗口：读取 mario.html 已完成
正文：我已经拿到计划和当前代码，接下来对照缺项。
正文：最终答复。
```

禁止显示成：

```text
读取文件 d:\...\mario.html 读取文件 运行中 d:\...\mario.html 开始 #6 -> #10
读取文件 d:\...\mario.html 读取文件 运行中 d:\...\mario.html 开始 #14 -> #18
```

原因：

- 这把内部轨迹噪声暴露给用户。
- 状态没有被 completion 更新。
- 同一窗口缺生命周期收口。
- 正文反馈缺失。

关键点：上面的 `tool_call_requested -> tool_permission_decided -> tool_item_started -> tool_item_completed` 是后端生命周期，不是四条前端消息。前端只显示同一个窗口的状态变化。

## 5. 前端投影算法要求

### 5.1 输入

前端 reducer 接收 `PublicProjectionFrame`。

必须读取：

- `authority`
- `frame_id`
- `source_event_id`
- `source_event_type`
- `event_offset`
- `event_family`
- `channel`
- `op`
- `slot`
- `source_authority`
- `tool_call_id`
- `tool_lifecycle_id`
- `title`
- `text`
- `detail`
- `state`
- `target`
- `arguments_preview`

### 5.2 分类

| 条件 | 分类 | 表面 |
| --- | --- | --- |
| `slot=body` 或 `event_family=assistant_body` | 正文 | 正文区 |
| `source_authority=model` 且 `event_family=status_trace` | 模型反馈 | 正文区 |
| 有 `tool_call_id/tool_lifecycle_id/tool_name` | 工具生命周期 | 工具窗口 |
| `commit_ack/commit_failed` | 收口 | 正文/状态 |
| hidden/trace_only | trace | 默认不显示 |

### 5.3 排序

```text
先按 event_offset
同 offset 下正文优先于工具请求
工具 request/start/completed 归并后按第一次出现位置显示
```

### 5.4 去重

正文去重：

```text
compactText(title + detail)
```

工具去重：

```text
tool_call_id || tool_lifecycle_id
```

frame 去重：

```text
frame_id || source_event_id
```

禁止：

- 不能因为反馈重复就删除整个反馈生产链。
- 不能用工具标题去重工具生命周期。
- 不能用路径当唯一合并键。

## 6. 当前代码应对照的文件

| 文件 | 责任 |
| --- | --- |
| `backend/api/chat.py` | 把运行事件桥接成 public event |
| `backend/harness/runtime/projection/projector.py` | 把 public event 转成 `PublicProjectionFrame` |
| `frontend/src/lib/projection/reducer.ts` | 将 frame 入账为 `MessagePublicProjection` |
| `frontend/src/components/chat/ChatMessage.tsx` | 将正文块和工具轨迹按时序分发表面 |
| `frontend/src/components/chat/PublicTimelineActivity.tsx` | 渲染工具窗口 |
| `frontend/src/lib/api.ts` | 前后端投影类型合同 |

## 7. 审查清单

每次改投影 UI，必须逐项回答：

| 问题 | 合格答案 |
| --- | --- |
| 模型反馈从哪里来？ | `runtime_step_summary` 或正文流 |
| 模型反馈显示在哪里？ | 正文区 |
| 工具请求从哪里来？ | `tool_call_requested` |
| 工具完成怎么回连？ | 同一 `tool_call_id/tool_lifecycle_id` |
| 折叠态显示什么？ | 工具动作、目标、状态 |
| 是否显示调用号/offset？ | 默认不显示 |
| final 后显示什么？ | 最终正文 |
| trace 在哪里？ | 调试/历史追溯，不抢主消息 |

## 8. 结论

正确的 agent 显示不是“正文或工具二选一”，而是：

```text
模型正文和工具调用可以同一轮同时返回。
前端必须按同一时序投影，但分别落到正文区和工具窗口。
工具窗口只告诉用户系统在做什么。
正文只告诉用户模型在说什么。
```

如果出现以下任一现象，就是投影错误：

- 工具调用把正文反馈挤没了。
- 正文里出现工具调用号、offset、重复路径。
- 一个工具调用拆成多张重复窗口。
- 工具完成后仍显示运行中。
- 为了去重砍掉模型反馈。
