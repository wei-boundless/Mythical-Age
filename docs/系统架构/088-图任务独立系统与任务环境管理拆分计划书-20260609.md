# 图任务前后台分层与任务环境管理拆分计划书

日期：2026-06-09
状态：执行中

## 1. 最新修正结论

图任务系统需要分成两个用户表面：

```text
任务系统后台
  负责图任务配置：图定义、拓扑、节点、边契约、编译报告、发布配置

图任务前台
  负责实例运营：项目实例、启动、续跑、暂停、监控、节点会话、文件和产物

任务环境管理
  只负责 agent / 节点运行时装配所需的环境资产
```

关键语义：

1. 图任务定义不绑定环境。
2. 环境只用于给节点 agent 装配运行上下文。
3. 图任务配置可以放在任务系统后台。
4. 图任务前台不负责编辑拓扑或契约，只负责管理、启动和监控实例。
5. 图任务实例是项目，是多 agent 会话集合。
6. 图任务没有“环节”这个主概念；环节属于节点自身。

这次修正不是否定“图任务作为独立系统”，而是把系统拆成更成熟的 control plane / operations plane：

```text
control plane: 任务系统里的图定义配置
operations plane: 主导航里的图任务项目控制台
```

## 2. 当前问题

旧页面路径会出现：

```text
任务系统 / 环境管理 / 图任务工作区 / Vibe 编码工作区
```

它同时制造三种误解：

- 图任务属于某个任务环境。
- 图任务运行需要环境作为上层容器。
- 图定义编辑、实例项目和运行监控应该混在同一张页面。

这些都不符合当前架构。正确模型是：

```text
任务系统
  图定义配置后台
  契约库
  节点配置
  环境管理
  队列/记录后台

图任务
  实例项目列表
  创建/启动/续跑
  实时监控
  节点会话
  项目文件和产物
```

## 3. 系统导航目标

主导航建议保持：

```text
工作台
图任务
记忆
任务系统
编排
能力
健康
配置
```

| 入口 | 主对象 | 职责 |
| --- | --- | --- |
| 图任务 | GraphTaskInstance / GraphRun | 管理项目实例、启动、续跑、监控、节点会话、文件和产物 |
| 任务系统 | GraphTaskDefinition / Contract / NodeConfig / TaskEnvironment | 配置图定义、契约、节点、环境和后台运行资产 |
| 任务环境 | TaskEnvironment 子域 | 管理 agent 可装配的资料、提示、资源、默认能力 |

前台“图任务”入口要求：

1. 作为主导航固定项，和主会话同级。
2. 第一屏就是生产控制台，不是说明页。
3. 只展示实例项目、运行状态、节点会话和产物管理。
4. 不出现“图定义编辑”“编辑图定义”这类配置入口。
5. 主会话页里的“图任务层”按钮直接切换到图任务前台。

后台“任务系统”入口要求：

1. 有独立“图定义”配置域。
2. 图定义域负责拓扑、节点、边契约、编译报告和发布。
3. 环境管理域不再提供图任务工作区。
4. 任务系统可以从图任务前台或旧入口接收 `mode: "editor"` 跳转。

## 4. 图任务前台边界

图任务前台包含：

```text
实例项目
  一个图定义下的多个项目实例

运行控制台
  GraphRun、work order、节点状态、失败恢复

节点会话
  每个节点 agent 的独立 session

项目文件
  input / working / artifacts / memory / logs / runs

产物中心
  按节点、契约、运行、版本、状态管理产物
```

图任务前台不包含：

```text
图定义编辑
拓扑编辑
边契约配置
任务环境装配
环境资料库管理
环境提示词配置
图级阶段/环节
```

## 5. 任务系统后台边界

任务系统后台中的“图定义”配置域包含：

```text
图定义
  图 ID、标题、领域、发布状态

拓扑
  节点、边、入口、出口、资源节点

节点配置
  节点角色、执行者引用、契约绑定、装配预览

边契约
  handoff、artifact flow、review gate、A2A 等协议实例

编译报告
  缺失项、阻塞项、警告、可发布裁决

发布配置
  draft / published / run_bound / archived
```

任务系统后台可以配置图任务，但不承担实例项目运营。实例项目运营回到前台图任务页。

## 6. 任务环境管理边界

任务环境管理保留：

```text
环境类型
资源装载
环境说明
环境内任务归属
```

它可以被节点配置引用，但不直接管理图任务。

环境管理页面可以展示“哪些节点配置会引用这个环境”，但不提供：

```text
图任务工作区
图定义编辑器
图运行监控
图任务项目管理
```

## 7. 环节语义收口

图任务不展示“图级环节”。

正确模型：

```text
图任务 = 拓扑 + 节点 + 边契约 + 实例运行
节点 = agent 工作单元，可拥有自己的环节、步骤、内部执行链
边 = 节点之间的交接协议
```

UI 展示规则：

1. 图任务前台只展示节点状态：ready / running / completed / failed / blocked。
2. 任务系统图定义编辑只展示节点、边、契约、资源引用和编译报告。
3. 节点详情里才展示节点内部环节，例如：规划、执行、审查、提交。
4. 如果后端保留 `phase_id`，它只能作为节点生命周期坐标或兼容字段，不应作为用户主概念。

## 8. 前端落地计划

### 阶段一：图任务前台运营面

修改：

```text
frontend/src/components/workspace/views/GraphTaskSystemView.tsx
frontend/src/components/workspace/views/task-graph-workbench/GraphTaskWorkspace.tsx
frontend/src/components/workspace/views/task-graph-workbench/GraphTaskInstanceWorkbench.tsx
```

目标：

- `GraphTaskSystemView` 只渲染实例项目运营面。
- `GraphTaskWorkspace` 支持明确 surface：`operations` / `configuration` / `combined`。
- `operations` surface 强制实例项目模式，隐藏图定义编辑入口。
- 实例页保留创建实例、启动运行、监控、节点会话、文件和产物。

### 阶段二：任务系统图定义配置域

修改：

```text
frontend/src/components/workspace/views/TaskSystemView.tsx
```

目标：

- 新增“图定义”配置域。
- 该域渲染 `GraphTaskWorkspace surface="configuration"`。
- `configuration` surface 强制图定义编辑模式，不展示实例项目运营入口。
- 任务系统主导航文案保留为“任务系统”，不再缩窄成“任务环境”。

### 阶段三：移除环境管理旧入口

修改：

```text
frontend/src/components/workspace/views/task-system/environment/TaskEnvironmentManagementWorkbench.tsx
frontend/src/components/workspace/views/task-system/environment/EnvironmentPages.tsx
```

删除：

- 环境管理下的 `graphs` 子页。
- `EnvironmentGraphInventoryPage`。
- 环境页中的图运行监控和图定义编辑器。

### 阶段四：跳转语义收口

修改：

```text
frontend/src/lib/store/runtime.ts
frontend/src/lib/run-monitor/controller.ts
```

目标：

- `openTaskGraphWorkspace()` 默认进入图任务前台运营面。
- `openTaskGraphWorkspace({ mode: "editor" })` 进入任务系统图定义配置域。
- run monitor graph signal 进入图任务前台。
- 前台和后台都不携带图级 `task_environment_id`。

## 9. 后端边界检查

后端实例 API 方向保持：

```text
backend/api/graph_task_instances.py
backend/task_system/graph_instances/
```

需要持续保证：

1. 图任务实例 API 不读取环境作为上层容器。
2. `start_graph_task_instance_run()` 只传实例 scope。
3. runtime diagnostics 中 `graph_task_instance_id` 是实例归属。
4. 节点会话继承实例 scope，而不是环境 scope。
5. 产物路径进入 `storage/graph_task_instances/<instance>/...`。

## 10. 验收标准

完成后用户应看到：

1. 点击主导航“图任务”直接进入图任务前台控制台。
2. 图任务前台不出现“图定义编辑”切换或“编辑图定义”按钮。
3. 图任务前台可以创建实例、启动运行、续跑、暂停、监控。
4. 节点会话可以在实例内查看。
5. 项目文件和产物按实例管理。
6. 点击主导航“任务系统”可以进入“图定义”配置域。
7. 图定义配置域可以编辑拓扑、节点、边契约、发布状态和编译报告。
8. 环境管理里没有图任务工作区主入口。
9. 页面路径不再出现“环境管理 / 图任务工作区 / Vibe 编码工作区”。
10. 图任务创建实例和启动运行不需要选择环境。
11. 图任务 UI 不出现“图级环节/阶段”作为主概念。
