# 图任务独立系统与任务环境管理拆分计划书

日期：2026-06-09
状态：待确认执行

## 1. 修正结论

当前把“图任务工作区”挂在“环境管理”下面是错误的信息架构。

新的边界应调整为：

```text
任务环境管理系统
  只负责 agent / 节点运行时装配所需的环境资产

图任务系统
  独立负责任务图定义、编译、实例项目、运行监控、节点会话和产物管理
```

关键语义：

1. 图任务不属于任务环境。
2. 图任务定义不绑定环境。
3. 环境只用于给节点 agent 装配运行上下文。
4. 图任务实例是项目，是多 agent 会话集合。
5. 图任务没有“环节”这个主概念；环节是节点自身的执行结构。

## 2. 当前问题

当前页面路径会出现：

```text
任务系统 / 环境管理 / 图任务工作区 / Vibe 编码工作区
```

这会让用户误解为：

- 图任务属于某个任务环境。
- 图任务运行需要环境作为上层容器。
- 图任务实例项目和环境项目是同一种东西。

这些都不符合已经确认的架构。

## 3. 目标导航

建议把图任务提升为主页面级入口，而不是二级配置页。

系统级导航调整为：

```text
工作台
图任务
记忆
任务环境
编排
能力
健康
配置
```

其中：

| 系统 | 主对象 | 职责 |
| --- | --- | --- |
| 图任务 | GraphTaskDefinition / GraphTaskInstance | 定义拓扑、编译配置、创建实例项目、运行多 agent 图任务 |
| 任务环境 | TaskEnvironment | 管理 agent 可装配的资料、提示、资源、默认能力 |
| 编排 | Agent / Node / Capability assembly | 管理 agent、节点配置、能力、权限和运行装配 |

第一阶段可以先不重命名所有内部代码，但用户可见入口必须拆开。

图任务入口要求：

1. 作为主导航固定项，用户可以顺手点击进入。
2. 不依赖先进入任务环境、任务系统或配置页。
3. 入口文案直接叫“图任务”，不要叫“环境图任务”“任务图配置”。
4. 打开后第一屏就是生产工作台，不是说明页。
5. 第一屏必须能直接继续运行、创建实例项目或打开最近实例。

## 4. 图任务系统边界

图任务系统包含：

```text
图定义
  拓扑、节点、边、边契约、图级编译规则

实例项目
  一个图定义下的多个项目实例

运行控制台
  GraphRun、work order、节点状态、失败恢复

节点会话
  每个节点 agent 的独立 session

项目文件
  input / working / artifacts / memory / logs / runs

产物中心
  按节点、契约、运行、版本、状态管理产物
```

图任务系统不包含：

```text
任务环境装配
环境资料库管理
环境提示词配置
图级阶段/环节
```

## 5. 任务环境管理边界

任务环境管理保留：

```text
环境类型
资源装载
环境说明
环境内任务
```

它可以为节点提供环境候选，但不直接管理图任务。

也就是说，环境管理页面可以显示“哪些节点配置会引用这个环境”，但不应提供“图任务工作区”主入口。

## 6. 环节语义收口

图任务不展示“图级环节”。

正确模型：

```text
图任务 = 拓扑 + 节点 + 边契约 + 实例运行
节点 = agent 工作单元，可拥有自己的环节、步骤、内部执行链
边 = 节点之间的交接协议
```

UI 展示规则：

1. 图任务概览只展示节点状态：ready / running / completed / failed / blocked。
2. 图任务编辑只展示节点、边、契约、资源引用和编译报告。
3. 节点详情里才展示节点内部环节，例如：规划、执行、审查、提交。
4. 如果后端保留 `phase_id`，它只能作为节点生命周期坐标或兼容字段，不应作为用户主概念。

## 7. 前端拆分计划

### 阶段一：新增独立图任务系统入口

新增或改造：

```text
frontend/src/components/workspace/views/GraphTaskSystemView.tsx
```

职责：

- 直接渲染 `GraphTaskWorkspace`。
- 不接收 `taskEnvironmentId`。
- 不显示环境选择器。
- 使用 `/api/orchestration/graph-tasks` 作为图定义来源。
- 作为主页面级入口挂到系统导航。
- 默认展示图任务生产首页。

图任务生产首页第一屏应包含：

```text
运行中实例
最近图任务项目
创建实例项目
常用图定义
问题/阻塞提示
```

首页不是营销页，也不是说明页。它应该让用户直接完成：

```text
继续一个运行中实例
从常用图定义创建新项目
打开某个实例的节点会话
进入实例项目文件
查看阻塞和失败节点
```

### 阶段二：导航拆分

修改：

```text
frontend/src/app/page.tsx
frontend/src/lib/store/types.ts
frontend/src/lib/store/runtime.ts
```

目标：

- 增加或明确 `graph-task-system` 视图。
- 顶部/侧边系统导航中的“图任务”进入独立图任务系统。
- “任务”或“任务系统”改成“任务环境”。
- 图任务在主导航中保持高优先级位置，靠近“工作台”。
- 原 `creative` 入口如果继续保留，必须明确是旧创作项目页或迁移到图任务系统，不得和新图任务系统并列造成歧义。

### 阶段三：从环境管理中移除图任务工作区

修改：

```text
frontend/src/components/workspace/views/TaskSystemView.tsx
frontend/src/components/workspace/views/task-system/environment/EnvironmentPages.tsx
frontend/src/components/workspace/views/task-system/environment/TaskEnvironmentManagementWorkbench.tsx
```

删除或降级：

- 环境管理下的 `graphs` 子页。
- `EnvironmentGraphInventoryPage` 作为图任务主入口。
- 页面路径里的 `环境管理 / 图任务工作区 / <环境名>`。

如需保留引用视图，只能叫：

```text
环境引用
```

并只展示“哪些节点或任务可能使用这个环境”，不能启动或管理图任务。

### 阶段四：图任务工作区去环境参数

修改：

```text
frontend/src/components/workspace/views/task-graph-workbench/GraphTaskWorkspace.tsx
frontend/src/components/workspace/views/task-graph-workbench/GraphTaskInstanceWorkbench.tsx
```

目标：

- 移除 `taskEnvironmentId` 作为图任务工作区 props。
- 图任务定义列表不再依赖 `environment_graph_inventory`。
- 创建实例、启动运行、节点会话、文件产物全部使用图任务实例 scope：

```json
{
  "workspace_view": "graph_task",
  "task_environment_id": "",
  "project_id": "<graph_task_instance_id>"
}
```

### 阶段五：运行管理归位

当前 `RunManagementWorkbench` 中如果有“图任务项目”，应迁移到图任务系统。

任务环境管理里的运行页面只展示环境相关任务运行，不做图任务项目管理。

## 8. 后端边界检查

后端已经新增的实例 API 方向是正确的：

```text
backend/api/graph_task_instances.py
backend/task_system/graph_instances/
```

还需要检查：

1. 图任务实例 API 不读取环境作为上层容器。
2. `start_graph_task_instance_run()` 只传实例 scope。
3. runtime diagnostics 中 `graph_task_instance_id` 是实例归属。
4. 节点会话继承实例 scope，而不是环境 scope。
5. 产物路径进入 `storage/graph_task_instances/<instance>/...`。

## 9. 旧入口处理

不保留长期双入口。

处理策略：

1. 独立图任务系统上线后，环境管理里的图任务子页删除。
2. 旧的环境图清单只允许作为迁移期间的临时入口，后续清理。
3. 主会话页不显示图任务实例和节点会话。
4. 旧 `TaskGraphRunControlPanel` 可以作为浮窗/最近运行提示，但不是主入口。

## 10. 验收标准

完成后用户应看到：

1. 点击主导航“图任务”直接进入图任务系统。
2. 图任务系统首页就是可操作生产工作台。
3. 页面路径不再出现“环境管理 / 图任务工作区 / Vibe 编码工作区”。
4. 任务环境管理里没有图任务工作区主入口。
5. 图任务创建实例和启动运行不需要选择环境。
6. 节点会话可以在实例内查看。
7. 项目文件和产物按实例管理。
8. 图任务 UI 不出现“图级环节/阶段”作为主概念。
9. 节点详情可以展示节点自己的环节或内部执行链。
10. 用户无需理解 graph_run_id 或环境配置，就能从首页继续或启动图任务。

## 11. 建议执行顺序

1. 新增 `GraphTaskSystemView` 独立主入口。
2. 导航加入高优先级“图任务”，任务系统改名为任务环境。
3. 从环境管理删除图任务子页。
4. `GraphTaskWorkspace` 去掉环境 props 和环境 inventory 依赖。
5. 把现有实例项目工作台迁到独立图任务系统。
6. 增加图任务生产首页：运行中实例、最近项目、创建实例、常用图定义。
7. 跑 TypeScript、后端实例 API 回归、固定端口浏览器验证。
