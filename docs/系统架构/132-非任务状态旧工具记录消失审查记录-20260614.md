# 非任务状态旧工具记录消失审查记录

日期：2026-06-14

本记录接续 `129-agent显示流程与投影规则说明-20260614.md` 与 `131-投影重复根因审查记录-20260614.md`，只审查“旧工具记录为什么在非任务状态下消失”。

## 1. 现象

截图中，旧消息只剩模型反馈正文：

```text
我在上一轮还没开始系统审查文件就被截断了...
继续读取更多关键部分：
Let me read some of the key helper functions deeper in the file:
```

但对应运行中真实存在多条工具调用，例如：

```text
read_file backend/harness/loop/single_agent_turn.py start_line=99 line_count=200
read_file backend/harness/loop/single_agent_turn.py start_line=280 line_count=200
```

因此问题不是“工具从未产生”，而是“离开运行态后工具轨迹没有被显示恢复”。

## 2. 已确认断点

### C1. 前端在非流式历史正文存在时直接清空工具时间线

`frontend/src/components/chat/ChatMessage.tsx` 当前逻辑：

```text
hasCommittedAssistantText = !isUser && !streamingContent && assistantContentText.trim()
publicTimelineItems = isUser || hasCommittedAssistantText ? [] : publicTimelineItemsFromProjection(publicProjection)
```

这意味着：

| 状态 | 结果 |
| --- | --- |
| 正在流式运行 | 允许显示 `publicProjection` 工具窗口 |
| 非流式、历史正文存在 | 直接隐藏所有工具窗口 |

这正好对应“非任务状态下旧工具记录消失”。

### C2. 前端在正文 finalized/committed 后也直接清空工具时间线

`frontend/src/components/chat/ChatMessage.tsx` 的 `publicTimelineItemsFromProjection()` 内部还有：

```text
if (projectionHasClosedBody(projection)) return []
```

`projectionHasClosedBody()` 在 `bodyState=finalized/committed` 且有正文时返回 true。

这会造成即使 `publicProjection.timeline` 里有工具窗口，最终态也不显示。

### C3. 历史加载接口没有返回可重建工具窗口的 projection frames

前端加载会话详情走：

```text
getSessionTimeline(sessionId).catch(getSessionHistory)
-> toUiMessages(history.messages)
```

`backend/api/sessions.py` 的 `/sessions/{session_id}/timeline` 调用：

```text
build_session_runtime_timeline(...)
```

`backend/harness/runtime/session_timeline.py` 当前返回：

```text
history messages
runtime_attachments
```

其中 `runtime_attachments` 只有：

```text
run_id
anchor_turn_id
anchor_message_id
status
latest_event_type
event_count
trace_available
debug_trace_ref
```

它不返回：

```text
public_projection_frame[]
projectionLedger
publicProjection
tool timeline items
```

所以刷新页面、切换会话、运行结束后重新加载历史时，前端没有可重放的工具投影数据源。

### C4. `toUiMessages()` 只把历史消息转为正文消息，不会恢复投影

`frontend/src/lib/store/utils.ts` 的 `toUiMessages()` 只写入：

```text
content
toolCalls
retrievals
sourceTurnId
sourceTaskRunId
answer metadata
```

它不会写入：

```text
projectionLedger
publicProjection
runtime projection timeline
```

因此非任务状态的历史消息天然只剩正文。

## 3. 线路审查表

| Line | 期望线路 | 当前代码证据 | 状态 |
| --- | --- | --- | --- |
| L1 | live SSE `public_projection_frame` -> reducer -> `publicProjection.timeline` -> 工具窗口 | `frontend/src/lib/store/events.ts` 调 `applyPublicProjectionFrame()`；`ChatMessage.tsx` 可渲染 `publicTimelineItemsFromProjection()` | 运行态可连接 |
| L2 | finalized/committed 后仍保留旧工具窗口为历史轨迹 | `ChatMessage.tsx` 的 `projectionHasClosedBody()` 返回 true 后直接 `return []` | 断开 |
| L3 | 非流式历史正文存在时仍显示旧工具轨迹 | `hasCommittedAssistantText` 为 true 时 `publicTimelineItems=[]` | 断开 |
| L4 | `/sessions/{id}/timeline` 返回可重放 projection frames | `session_timeline.py` 只返回 `runtime_attachments` 摘要 | 断开 |
| L5 | `toUiMessages()` 将历史 projection 恢复成 `publicProjection` | 当前只转换正文和 metadata | 断开 |

## 4. 根因

这是两个层面的缺线叠加：

1. **显示层主动隐藏**：前端认为“非流式 + 有正文”或“正文 finalized/committed”就应该隐藏所有工具窗口。
2. **历史层没有回放数据**：后端历史 timeline 没有把旧运行的 `public_projection_frame` 或已归并的 `publicProjection` 返回给前端。

所以它不是单纯 CSS 隐藏，也不是工具记录没产生，而是 live projection 和 history projection 之间没有闭环。

## 5. 目标行为

结合当前要求，目标应改为：

| 状态 | 正文 | 工具窗口 |
| --- | --- | --- |
| 运行中 | 按时序显示模型反馈/正文 | 显示当前工具窗口 |
| 工具完成但模型未收口 | 正文继续按 offset 显示 | 工具窗口显示已完成 |
| 正文 finalized/committed | 主视觉以正文为主 | 旧工具窗口仍作为折叠历史轨迹保留 |
| 刷新/切会话/非任务状态 | 显示历史正文 | 从历史 projection replay 恢复旧工具窗口 |

重点：正文收口后可以降低工具轨迹的视觉权重，但不能让旧工具记录消失。

## 6. 修复方向

### 后端

在 `/sessions/{session_id}/timeline` 中为每个 `turn_run/task_run` attachment 增加可重放投影数据：

```text
public_projection_frames[]
```

只返回主聊天需要的公开 frame：

```text
slot=body
slot=current_action
event_family=tool_control
event_family=status_trace 且 source_authority=model
turn_anchor_terminal/pinned
runtime_commit
```

禁止返回调试-only 大 payload、完整工具输出和内部模型消息。

### 前端

1. `refreshSessionDetails()` 在 `toUiMessages(history.messages)` 后，把 `history.runtime_attachments[].public_projection_frames` replay 到对应 assistant message。
2. `ChatMessage.tsx` 删除“非流式正文存在就清空 `publicTimelineItems`”的逻辑。
3. `projectionHasClosedBody()` 不再让工具窗口整体消失；只用于降低运行态噪声或隐藏非工具状态。
4. 历史工具窗口默认折叠，显示动作、目标、状态；展开后显示命令行和摘要。

## 7. 注意点

- 不能用 `tool_calls` 历史字段替代 projection。`tool_calls` 是模型/协议历史，不等于用户可见工具窗口。
- 不能只在前端从正文猜工具窗口。工具窗口必须来自后端公开投影 frame 或已归并的 `publicProjection`。
- 不能把完整 trace 直接塞进主聊天。历史恢复只恢复公开窗口，不恢复内部 ID、offset、大段 raw output。
- 不能按路径合并不同工具调用，仍然必须按 `tool_call_id || tool_lifecycle_id` 合并。
