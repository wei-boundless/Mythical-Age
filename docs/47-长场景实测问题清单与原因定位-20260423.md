## 2. 问题清单

## 2.1 P1: PDF 跟读链路实质失效

### 现象

在研究场景和运营场景中，PDF 相关问题多次没有形成可靠答案：

1. “现在打开 knowledge/AI Knowledge/2025年AI治理报告：回归现实主义.pdf，给我一个全文总览。”
2. “第三页具体讲了什么？”
3. “第四页如果要给业务负责人看，应该重点看哪几句？”
4. “把这份 PDF 的核心结论压成三条行动建议。”
5. “回到刚才那份 PDF，第二部分强调的约束是什么？”

用户可见退化表现：

- 返回“已定位到相关页面，但尚未形成可靠摘要”
- 直接暴露乱码文本
- 命中的页面明显偏向参考文献页、尾页，而不是正文核心页

典型输出：

- `已定位到与问题最相关的页面：P72、P69、P71。当前工具返回的是原始检索片段，尚未形成可靠摘要。`
- `回归现实主义2025年AI治理报告 隠㚵蔠裮䅳熱閔 腾讯研究院`

### 影响

- PDF 工具当前只能算“找得到页”，不能算“答得出内容”。
- 这会直接影响后续 PDF 子 agent 的可信度，因为主链已经无法稳定提供可复用摘要。
- follow-up 虽然保留了 PDF 绑定，但绑定的是退化结果，不是稳定语义摘要。

### 根因链

根因 1：`全文总览/总结/行动建议` 等问法默认走的是 `browse`，不是 `deep_read`。

- “全文总览”“核心结论”“行动建议”当前没有被提升到 `deep_read`。

## 2.2 P1: 结构化数据显式切换入口失效

### 现象

以下首答都退化成了 fallback：

1. “切到 knowledge/E-commerce Data/inventory.xlsx，先看哪些仓库缺货。”
2. “现在换成 knowledge/E-commerce Data/employees.xlsx，找出薪资前五的人。”
3. “再回到 inventory.xlsx，哪一个仓库最该优先补货？”

但这些数据集的 follow-up 问题又能正常出结果：

- “按仓库汇总前五。”
- “按部门汇总这些高薪员工。”

### 影响

- 结构化数据引擎本身并没有坏，坏的是“显式切换数据集”的入口。
- 用户首次点名切换到某个 Excel 时，系统可能先把路径切坏，后续答案自然退化。
- 这会导致用户误判为“表格工具不稳定”，实际是绑定路径入口不稳定。

根因 1：显式数据集路径提取正则不允许空格，导致完整路径被截断。

- 文件：
  - [task_understanding.py](/D:/AI应用/langchain-agent/backend/understanding/task_understanding.py)

- 位置：

  - [task_understanding.py:438](/D:/AI应用/langchain-agent/backend/understanding/task_understanding.py#L438)
  - [binding_resolver.py:78](/D:/AI应用/langchain-agent/backend/query/binding_resolver.py#L78)
  - [binding_resolver.py:83](/D:/AI应用/langchain-agent/backend/query/binding_resolver.py#L83)

- `knowledge/E-commerce Data/inventory.xlsx` 被提取成了 `Data/inventory.xlsx`

根因 2：坏路径一旦进入 `understanding.tool_input`，后续不会被正确 binding 覆盖。

- 文件：

## 2.4 P2: 记忆写入链路被 compound 切分污染

### 现象

问题：

- “记住：以后复杂问题先给结论，再展开解释。”

- `plan_route = compound`
- `subquery_count = 2`
- 第二个子任务被拆成 “展开解释”
- runtime 最终落成了 `rag`

## 3. 与后续修正的关系

结合当前问题，后续修正建议按以下顺序推进：

1. 先把 `PDF 子 agent` 做实。
   - 明确 browse / deep_read / page_read 的职责边界
   - 增强正文页排序和坏页识别
   - 让 PDF 输出先形成稳定 canonical summary，再交给主链分类器
2. 再修 `显式资源路径解析`。
   - 先把带空格路径的提取与 binding 修正稳定
3. 再修 `memory / compound` 边界。
   - 避免偏好写入被顺序切分污染
4. 最后收紧 `long scenario` 验收标准。
   - 把 fallback、乱码、错路由纳入失败条件

## 4. 当前判断

- `RAG 主链整体可运行`

- `当前最值得优先收敛的是 PDF 子 agent，而不是继续在表层报告上追求 passed`
