# 编排系统与任务系统实用化重做计划

## 目标

把编排系统和任务系统从“说明型卡片工作台”改成“配置型数据库界面”。系统页进入后独立占满页面，不再放进主会话三段式工作台；任务图编辑器作为任务系统下的专用编辑页，不能再被外层任务系统标题栏、对象入口和编辑器自身工作台重复挤压。

## 当前问题

1. 任务系统入口使用卡片表达对象层级，适合展示，不适合配置。用户需要的是快速定位对象、切换记录、查看当前状态和进入配置。
2. 主对象与支撑对象以卡片分区后，右侧支撑列会拉高整个入口区域，压缩真正的配置区。
3. 任务图编辑器外层还套着任务系统 Shell，内部又有 TaskGraph TopBar、工作集、图层导航、画布、检查器和执行包追踪，形成两套工作台叠加。
4. 图编辑器当前把图层导航、模板/执行节点、画布、右侧检查器、底部追踪全塞在一屏，空间层级不清。
5. 编排系统和任务系统已经脱离主会话工作台，但内部仍需要从“卡片说明”进一步收束为对象管理和直接配置。

## 目标结构

### 系统级承载

1. 编排系统和任务系统进入后直接进入独立系统页。
2. 系统页只保留窄导航和主内容区。
3. 主会话、任务系统、编排系统是同级系统入口，不混在一页三栏内。

### 任务系统管理页

1. 任务系统管理页采用数据库式对象目录，不再使用大卡片。
2. 对象目录行按对象类型展示：
   - 主对象：任务域、任务定义、任务图
   - 支撑对象：契约库、资源权威、专业运行、编排资源、运行管理
3. 每行表达字段：对象类型、当前记录/数量、配置用途、状态。
4. 点击行切换右侧配置页面。
5. 顶部只保留当前任务域、关键计数、刷新/新建域动作。
6. 指标和支撑信息采用状态行、字段组、表格，不使用大卡片承载普通配置项。

### 任务图编辑器

1. 任务图编辑器从 TaskSystemShell 的 editor 包装中分离出来，作为独立的 `task-graph-editor-page`。
2. 编辑器只保留一套顶部：TaskGraph 自己的 TopBar 和工作集选择条。
3. 图编辑器内部布局改成：
   - 顶部：图身份、保存、发布、任务域/图草稿选择。
   - 左侧：图层导航，改为紧凑数据库式列表，不再使用大卡片组。
   - 中央：当前编辑页，拓扑页优先给画布最大空间。
   - 右侧：当前对象检查器。
   - 底部：执行包追踪，保持窄条，不抢画布高度。
4. 拓扑编辑页收束辅助栏宽度，画布成为最大区域。
5. 图编辑器内部少用卡片：导航用行表，工具用工具条，检查器用字段表，只有真正重复对象或需要框定边界的复杂块才使用卡片。

### 编排系统

1. 编排系统保留独立系统页。
2. Agent 目录和权限配置优先采用数据库式表格、状态行、字段组。
3. 能力/工具授权不再使用卡片墙，改成权限矩阵：
   - 能力类型
   - 能力名称
   - 来源
   - operation 映射数量
   - 风险
   - 当前状态
   - 允许/阻断动作
4. 右侧只显示当前选中能力的详情、风险和元数据，不再重复展示所有能力。

## 实施步骤

1. 更新计划文档，明确从卡片式改为数据库式。
2. 修改任务系统管理入口：用对象目录表替换对象卡片组。
3. 调整任务系统管理页 CSS，让对象目录占固定紧凑高度，配置区尽早出现。
4. 拆分任务图编辑器承载：editor 模式直接返回专用编辑页，不再套 TaskSystemShell。
5. 压缩 TaskGraph 图层导航，从分组卡片改成数据库式列表。
6. 优化拓扑页布局，减少左右栏宽度，把画布作为主区域。
7. 降低卡片密度，把普通指标、工具和选择详情改为状态行/字段组。
8. 重做编排系统权限页，把能力授权卡片墙改为权限矩阵。
9. 运行构建、相关测试，并用 Edge 截图验证任务系统管理页、任务图编辑器和编排权限页。

## 不做

1. 不改后端 API。
2. 不改任务图运行逻辑。
3. 不改任务图保存、发布、续跑 ID 逻辑。
4. 不碰监控系统。
5. 不用假数据应付测试。
