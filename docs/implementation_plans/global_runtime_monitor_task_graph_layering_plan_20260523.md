# 全局运行监控分层修正计划

## 背景

当前全局监控 Dock 直接展示后端 `globalRuntimeMonitor.task_runs` 原始列表。该列表同时包含用户启动的顶层任务图、任务图模块导入的子图运行、节点内部 Agent 任务，导致前端入口看起来像是启动了一堆子任务，破坏“顶层只监控任务图，具体子任务在详情里看”的产品分层。

## 目标

- 全局监控列表只展示用户可感知的顶层任务图运行。
- 任务图详情继续展示内部节点、子图、Agent 进程和产物状态。
- 选择态不能自动落到内部 Agent 任务上。
- 空状态文案改为任务图语义，避免把内部执行单元称为全局任务入口。

## 实施步骤

1. 在运行时 store 中增加统一的顶层任务图判断函数。
2. `applyGlobalRuntimeMonitorSnapshot` 选择默认项时只从顶层任务图中选择。
3. 事件流携带内部子任务事件时，不切换全局选中项。
4. `TaskMonitorDock` 使用同一规则过滤列表，并基于过滤后的列表计算统计。
5. `RuntimeMonitorDetailView` 使用同一规则定位可选任务，避免详情页回退到内部子任务。
6. 增加 store 单测，覆盖“内部子任务不抢占全局选中项”。

## 验证

- 运行前端测试。
- 用页面打开监控 Dock，确认列表只显示顶层任务图，详情里仍能看到任务图节点进度。
