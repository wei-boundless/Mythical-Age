# 能力系统全宽系统页实施计划

## 背景

能力系统当前仍包在通用工作台壳层里，因此页面左右两侧会出现：

- 左侧会话 / 文件管理面板。
- 右侧监控 / 网页 / 文件辅助面板。

这些面板对能力系统管理页没有直接价值，还压缩了中间内容，导致字体和信息块显得拥挤。

## 目标

- 能力系统改用系统页壳层，和任务系统、编排系统保持一致。
- 进入能力系统时不再显示左右两侧管理 / 监控面板。
- 保留左侧极窄系统导航栏，用于在会话、任务、编排、能力之间切换。
- 保留能力系统内部现有层级入口和管理功能。
- 保留前一轮对能力系统字体、行高、间距的清晰化调整。

## 实施步骤

1. 修改 `frontend/src/app/page.tsx`：
   - 将 `activeWorkspaceView === "capability-system"` 从 `WorkbenchShell` 分支移出。
   - 像任务系统、编排系统一样放入 `SystemPageShell`。
   - 保留 `TaskGraphRunInteractionDock`，避免运行交互能力丢失。

2. 修改 `frontend/src/app/globals.css`：
   - 让系统页壳层也继承 crisp console 变量和能力系统清晰排版规则。
   - 确认能力系统主内容在系统页内占满可用宽高。

3. 验证：
   - 运行前端构建。
   - 用本地 Edge 打开能力系统，确认左右管理 / 监控面板消失。
   - 检查能力系统概览和服务接入页仍清晰、无横向拥挤。
