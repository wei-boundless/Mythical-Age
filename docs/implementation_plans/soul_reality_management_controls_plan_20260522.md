# 现实世界契约与投影管理控件修复计划

## 背景

现实世界页面已经独立于洪荒世界，但仍存在两个问题：

1. 返回按钮在深色背景上对比不足，视觉上接近黑色。
2. 共同契约和执行投影库只展示不可管理，无法满足现实世界工作台的实际使用需求。

## 目标

1. 修复现实世界返回按钮的可读性和交互状态。
2. 共同契约支持选择、查看、编辑和保存。
3. 执行投影库支持列表选择、设为当前投影、删除非系统投影，并提供新建入口。
4. 不暴露 `prompt`、`runtime`、`catalog` 等开发语句到页面可见文案。
5. 不影响洪荒世界五个灵魂页面和原故事文案。

## 实施步骤

1. 后端接口
   - 在 soul API 中增加共同契约保存接口。
   - 使用现有 `SoulCatalogStore` 保存 `common_contracts` 桶，保持资源目录结构一致。
   - 保存后返回完整 soul catalog，方便前端刷新。

2. 前端 API
   - 增加共同契约保存函数。
   - 复用已有投影选择、新建、删除接口。

3. 现实世界前端交互
   - 增加共同契约选择状态、编辑草稿、保存状态。
   - 增加投影选择状态和管理状态。
   - 投影列表从只读条目改为可选择列表，右侧展示当前投影详情与管理操作。

4. 视觉修复
   - 强化现实世界返回按钮文字和边框对比。
   - 管理控件使用轻量列表、分段操作和文本区，避免回到卡片堆叠。

5. 验证
   - `npm run build`
   - soul 相关后端回归脚本
   - 本地 Edge 验证：按钮可见、共同契约可编辑保存、投影可选择、页面无噪声文案、无横向溢出。
