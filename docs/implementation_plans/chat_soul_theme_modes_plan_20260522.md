# 主会话重构前灵魂外观模式计划

## 背景

用户明确要找的是最近一轮 `重构` 之前的灵魂系统 UI，而不是更早的远古页面。对照 git 历史后，目标来源锁定为 `22faa8b 重构` 之前的 `2cc2fbf` 附近：那一版仍保留 `workspace-shell`、当前灵魂背景图、`data-soul` 五套氛围变量、柔和面板和消息质感。

## 目标

1. 主会话外观模式从“默认/现实”扩展为“默认/现实/河伯/句芒/四岳/祝融/玄女”。
2. 五套灵魂外观复用最近重构前的核心视觉资产：当前灵魂背景图、五套 `data-soul` 变量、光感纹理、柔和圆角、半透明面板。
3. 模式选择持久化，刷新后保持。
4. 模式切换只改变主会话视觉，不改变聊天、任务、流式、模型选择和灵魂系统业务逻辑。
5. 页面只出现用户能理解的短标签，不暴露开发式说明。

## 实施步骤

1. 类型与状态
   - 新增 `ChatVisualMode` 类型，加入五个灵魂 key。
   - localStorage 读取时校验合法模式，避免旧值或脏值污染 UI。
   - 灵魂模式时在 `documentElement.dataset.soul` 写入对应灵魂 key，退出时清理。

2. 入口控件
   - 将 ChatPanel 的两段式按钮扩展为横向流动的模式选择条。
   - 使用短标签：默认、现实、河伯、句芒、四岳、祝融、玄女。

3. 样式迁移
   - 新增 `.practical-workspace--soul-chat`，不复用现有 `.practical-workspace--soul` 的单列页面语义。
   - 为主会话、侧栏、消息、输入区、监控停靠区和背景图添加灵魂主题覆盖。
   - 使用 `frontend/public/souls/backgrounds/*-bg.png` 和五个灵魂已有立绘，不生成新图。

4. 验证
   - `npm run build`
   - Edge 验证：七个模式都能切换，刷新保持；桌面和移动无横向溢出；控制台无错误。
