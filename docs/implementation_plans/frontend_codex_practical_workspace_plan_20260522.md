# 前端 Codex 式实用工作台重构计划

## 目标

把当前前端从“氛围型灵魂背景 + 多入口页面”重做为实用工作台：

- 左侧：固定导航与会话入口，信息密度高、可快速切换。
- 中央：主工作区只保留两层核心入口：主会话页面、图任务层。
- 图任务层：直接承载任务图管理与编辑器，不再把编辑器藏在额外页面之外。
- 右侧：可折叠任务监控栏，在 Agent 开始任务或绑定任务图运行时用于观察运行状态、节点、产物、风险。
- 灵魂系统：不再作为界面背景；本轮先移除背景式呈现，后续可在右侧或会话中以投影/助手模块进入。

## 界面分类

- 产品类型：本地 Agent 工作台。
- 用户角色：任务编排者、Agent 操作者、调试者。
- 主对象：会话、任务图、任务图运行监控。
- 页面层级：
  - Shell 层：左导航、中央工作区、右监控栏。
  - 主工作区层：主会话 / 图任务。
  - 图任务层：任务域、任务、任务图、契约、运行、编辑器。
  - 监控层：只读观察与人工干预入口，不混入主会话内容。

## 设计系统决策

- 风格：Codex 式简洁实用，浅色、低装饰、清晰分隔、稳定布局。
- 色彩：近白背景、灰阶文字、黑色主操作、少量状态色；不使用灵魂大背景、渐变氛围或大面积发光。
- 组件：按钮、卡片、面板都保持 6-8px 圆角；用图标表达操作，不用装饰性图形。
- 布局：中央内容优先，左栏窄而可扫读，右监控栏可折叠，不抢占主任务。
- 信息原则：监控信息放右侧，图任务配置放图任务层，主会话只保留对话与输入。

## 实施步骤

1. 新建实用工作台骨架：在 `frontend/src/app/page.tsx` 中替换旧背景式 Shell，加入左栏、中央双层 Tabs、右侧监控 Dock。
2. 改造左侧导航：保留会话列表和新会话入口，加入“主会话 / 图任务层”两个主层级卡片按钮，移除灵魂立绘背景式入口。
3. 改造聊天面板：去掉内部“任务图监控”页签，监控统一移到右侧 Dock。
4. 新建右侧监控 Dock：复用 `TaskGraphRunMonitorPanel`，空状态时显示可折叠监控说明，有运行信号时展示任务状态。
5. CSS 覆盖为 Codex 式浅色工作台：在全局样式追加 scoped overrides，避免影响任务图内部已有编辑器语义。
6. 验证：运行 lint/test/build 中可行项，启动 Next dev server，用本地 Edge/Playwright 检查桌面与窄屏布局。

## 成功标准

- 首屏结构符合“左导航 / 中央主工作区 / 右侧可折叠监控”。
- 中央只有“主会话”和“图任务层”两个主层级入口。
- 图任务层内能打开并嵌入任务图编辑器。
- 主会话不再内嵌监控页。
- 右监控栏可以折叠，且有任务运行时能承载现有监控面板。
- 灵魂背景不再渲染在主界面底层。
- 页面在 1440px 桌面和移动宽度下不出现主要文本重叠或水平溢出。
