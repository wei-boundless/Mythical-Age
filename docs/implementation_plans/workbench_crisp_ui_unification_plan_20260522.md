# Workbench Crisp UI Unification Plan 2026-05-22

## Problem

当前主工作台和多个系统页面虽然已经具备三栏结构，但视觉上仍然发糊：

- 大量半透明背景、渐变、blur 和柔和阴影叠在一起，边界不清。
- 旧的暗色/氛围化样式仍会泄漏到任务系统、编排、能力系统和监控页。
- 文本灰度偏浅，辅助信息与正文差距不够明确，读起来像一层雾。
- 页面内卡片、面板和按钮的圆角与边框尺度不统一。
- 系统页之间使用不同视觉语言，用户从主会话切到图任务、能力或监控时感到断裂。

## Target

把项目 UI 收束成一个清晰、专业、适合 Agent 工作台的控制台：

- 近白底、深色正文、明确边框，优先保证文字和结构可读。
- 取消工作台和系统页里的 blur、玻璃感、漂浮阴影和低透明度叠层。
- 统一面板、按钮、输入框、表单、列表、状态徽标、监控行的尺寸和边界。
- 保持现有三栏架构和系统页层级，不把不同层级混回一页。
- 保留真实功能入口，不用说明性副标题撑 UI。

## Design Rules

1. 全局工作台基调：`#f3f5f8` 页面底，`#ffffff` 主面板，`#cbd3df` 结构线，`#0f172a` 正文。
2. 主动作蓝色只用于选中、主要按钮和焦点，不铺满页面。
3. 状态色使用蓝、绿、琥珀、红，并以边框/左侧信号为主。
4. 卡片和面板半径不超过 8px，减少视觉泡泡感。
5. 取消 `backdrop-filter` 和模糊纹理；需要分区时使用实线、轻底色和紧凑间距。
6. 字体权重提高到 500-700，辅助文字不得低到不可读。
7. 系统页面继承同一套 light console tokens，避免旧暗色变量继续支配页面。

## Implementation Steps

1. 在全局 CSS 末尾新增一层 `Crisp Console Unification` 覆盖，统一工作台变量和旧系统页变量。
2. 覆盖主壳层、左右面板、toolbar、tab、列表、搜索、文件编辑器、聊天输入的清晰样式。
3. 覆盖任务系统、编排系统、能力系统、运行监控等共用系统页类，去掉模糊/阴影/暗背景。
4. 覆盖系统框架独立页面，使它不再使用暗色氛围背景。
5. 清理容易继续制造模糊感的工作台 blur 样式。
6. 运行前端测试/构建，启动本地页面并用 Edge 进行视觉检查。

## Acceptance

- 主工作台、图任务、编排、能力、监控详情和系统框架页面都呈现同一套清晰控制台风格。
- 页面不再依赖玻璃模糊和低透明渐变来区分层级。
- 截图中的主区域、左右栏和右侧监控文字明显更清楚。
- `npm run test` 和 `npm run build` 不因样式调整失败。
