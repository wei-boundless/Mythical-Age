# 任务系统简洁数据库式 UI 与图编辑器重修计划 - 2026-05-23

## 目标

1. 任务系统整体改成简洁、实用、低装饰的工作台，不再用大面积展示卡片占据操作空间。
2. 管理与配置页采用数据库式结构：左侧对象目录像表索引，右侧内容像记录详情/属性表，强化扫描、选择、编辑和保存。
3. 图编辑器采用真正编辑器布局：顶部状态与操作栏、左侧窄图层/工具栏、中央主画布、右侧 Inspector，不再让多组说明卡围住画布。
4. 拓扑编辑页优先给画布空间，新增节点、资源节点、任务模板、边过滤放进紧凑工具区，选择详情放右侧检查器。
5. 保留现有数据流、API 和图编辑能力，只重修结构和视觉表达，不改运行语义。

## 实施步骤

1. 调整 `TaskSystemShell`：管理模式使用数据库式页面头、对象索引和工作区结构；编辑模式保持可返回但压缩头部。
2. 调整 `TaskGraphStudioShell` 与 `TaskGraphLayerNav`：图层导航变成窄编辑器导航，当前层说明压缩到顶部状态栏。
3. 调整 `TaskGraphTopologyPage`：重排为左侧工具列 + 中央画布 + 右侧 Inspector，移除画布周围大卡片式占位。
4. 更新 `globals.css` 中任务系统相关样式：统一简洁数据库风，减少圆角、阴影、渐变和嵌套卡片感。
5. 跑前端构建；如构建过慢则至少跑 TypeScript/Next 编译命令并记录结果。
