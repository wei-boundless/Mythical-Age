# 能力系统管理页统一与分群优化方案

## 1. 目标

把能力系统管理页改成低噪声、实用优先的工作台首页：先按“能力群”分卡，再进入单独管理页编辑具体内容。页面不再沿用旧的 `units / skills / tools / endpoints` 四面板展示逻辑，而是根据后端统一目录的真实结构，把 Skills、工具、MCP provider、能力端点和能力单元组织到同一套能力治理视图里。

## 2. 后端事实

当前后端已经不是“两套系统并排”的状态，而是：

- `capability_units` 是统一投影层，承载 `tool / skill / mcp / operation` 的治理视图。
- `mcp_management` 是统一 MCP provider 管理面，本地和外部 MCP 只是在 provider_kind / transport 上不同。
- `capability_endpoints` 是从 MCP 目录投影出来的端点视图，不应被前端当成第二套能力源。
- `skills` 和 `tools` 仍然是唯一可编辑对象，前端只负责编辑它们的真实内容和管理备注。

因此，前端首页必须按真实用途分群，而不是按旧系统分层。

## 3. 首页结构

### 3.1 分群卡片

首页只展示四个按具体作用划分的能力群卡片，每张卡片描述一个可管理的能力群：

- 工作方法：管理 Skills、模型可见提示和 SKILL.md。
- 执行工具：维护本地工具的类型、边界、风险和备注。
- 服务接入：本地 MCP、外部 MCP 和端点投影统一管理。
- 授权治理：查看统一能力投影、权限、健康和依赖。

卡片的职责只是告诉用户“这类能力是什么、当前状态如何、可以进入哪里管理”，不直接展开大量 JSON，也不把 backend 字段名伪装成信息架构。

### 3.2 进入方式

- 点击卡片进入单独管理页。
- 单独管理页只处理这一类对象，不把别的层级混进来。
- 页面内部仍然可以左右分栏，但该分栏只属于该单独对象，不再承载首页聚合职责。

## 4. 管理页结构

### 4.1 工作方法管理页

- 左侧列表只保留标题、路由、激活状态和任务范围。
- 右侧保留 `prompt_view`、`SKILL.md`、依赖 operation、校验结果。
- `SKILL.md` 默认折叠，编辑态再展开。
- 新建 skill 保持轻量，减少噪声字段。

### 4.2 执行工具管理页

- 左侧列表只保留工具名、operation、风险和边界。
- 右侧保留工具类型、LLM 说明、治理备注、可见性和授权视图。
- 原始契约与诊断 JSON 不默认占据主视图。

### 4.3 服务接入管理页

- 本地 MCP 和外部 MCP 统一到一个管理群里，不再拆两套管理层。
- Provider、server、tool、endpoint 使用同一套列表和筛选逻辑。
- 默认展示状态、transport、operation、模型可见性和授权预览。
- 只有在 inspect 或调试时才展开原始 diagnostics / schema。

### 4.4 授权治理视图

- 把 `capability_units` 作为统一治理视图的默认入口之一。
- 列表按 kind / source / risk 分组。
- 右侧显示 execution state、operation 依赖、permission view、health、source_ref。
- `display_facets` 只做辅助标签，不抢主视图权重。

## 5. 设计原则

- 低噪声优先，少解释，多事实。
- 不把展示字段伪装成执行事实。
- 不保留旧 MCP / endpoint 双壳入口。
- 不把同一信息在首页和详情页重复三遍。
- 原始 JSON、schema、diagnostics 统一放进折叠区。
- 页面标题和分类按钮要直接说清对象，不写开发说明式副标题。

## 6. 实施步骤

1. 重写能力系统首页，让它变成按作用划分的分群卡片入口。
2. 把工作方法、执行工具、服务接入、授权治理拆成独立管理页结构。
3. 删除旧的四面板首页逻辑和冗余噪声展示。
4. 清理旧 MCP 独立管理层入口、旧样式和冲突计划文档。
5. 实际浏览器打开验收，检查布局、对齐和响应式表现。

## 7. 验收标准

- 首页只显示清晰的能力群卡片，不再铺满旧面板。
- MCP 和能力端点在管理层面被统一解释，不再分两套入口。
- 服务接入页以 `mcp_management` 为真实管理面，以 `capability_endpoints` 为投影预览。
- Skills 和 Tools 的编辑页只保留真正有用的信息。
- 默认视图低噪声、可扫描、可编辑。
- 页面在浏览器中真实可用，没有错位、重叠或卡片堆叠噪声。
