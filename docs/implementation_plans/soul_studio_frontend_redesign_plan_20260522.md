# 灵魂系统 Soul Studio 前端重设计计划

## 问题判断

当前灵魂系统虽然完成了“先世界观，再灵魂”的技术结构，但视觉和页面架构仍然像后台表格：卡片、边框、三栏切割、重复盒子太多。用户明确要求不要借鉴旧前端风格，因此本次不在旧壳上继续修补，改为重写灵魂系统页面层级和视觉语言。

## 设计目标

1. 灵魂系统作为独立的 Soul Studio，不再像普通设置页。
2. 页面层级改为：
   - 顶部世界场域：用沉浸式世界选择和当前世界状态建立入口。
   - 中心灵魂舞台：当前灵魂、立绘、故事摘要和召唤状态成为主视觉。
   - 底部工作层：灵魂设定、投影、共同契约作为工作台，不抢主视觉。
3. 世界观仍是第一层，灵魂仍是第二层，投影/契约仍是第三层。
4. 洪荒视觉只在洪荒世界观下出现；默认世界保持无背景工作状态。
5. 减少硬边界、卡片和盒子感，使用流动背景、分层留白、曲线/光带、舞台式布局和列表式选择。

## 实施步骤

1. 重写 `PlaygroundView.tsx` 的渲染结构：
   - 保留现有数据加载、保存、投影编辑函数。
   - 替换 return 内的页面结构。
   - 新增世界切换轨、灵魂环形/横向选择、舞台区、工作层区域。
2. 重写灵魂系统专用 CSS：
   - 用 `.soul-studio` 命名新结构，避免继续依赖旧 `.soul-system-console` 样式。
   - 旧样式不再作为视觉依据，仅保留兼容无害。
3. 响应式：
   - 桌面：世界场域 + 舞台 + 工作层。
   - 移动：世界选择横向滚动，舞台上下堆叠，工作层单列。
4. 验证：
   - `npm run build`
   - 灵魂目录相关后端回归测试
   - Edge 桌面和移动截图验证：无横向溢出、世界切换可点击、洪荒内容只在洪荒世界出现。

## 完成标准

- 页面不再呈现为旧后台卡片/盒子堆叠。
- 默认世界没有洪荒舞台。
- 洪荒世界能显示五个灵魂和立绘舞台。
- 灵魂设定、投影、共同契约仍可切换和编辑。
- 桌面和移动都可用。
