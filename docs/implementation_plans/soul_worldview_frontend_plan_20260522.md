# 灵魂系统世界观入口与浅蓝工作台前端重构计划

## 目标

把灵魂系统从“直接选择灵魂”的旧入口，重构为“先选择世界观，再选择灵魂”的两级结构。同时调整主工作台为浅蓝、清爽、实用的控制台风格，并保留洪荒风格面板，但只在洪荒世界观组合下启用。

## 当前结构判断

- 主工作台入口在 `frontend/src/app/page.tsx`，目前只显式开放“主会话”和“图任务层”。
- 灵魂系统组件在 `frontend/src/components/workspace/views/PlaygroundView.tsx`，但没有接入当前主导航。
- 后端已提供正式资源目录：`SoulSystemCatalog.resource_catalog`，包含 `worlds`、`stories`、`cards`、`work_prompts`、`common_contracts`、`modes` 等资源层。
- 当前资源数据里只有默认无背景世界，但内置灵魂故事和投影仍带有洪荒设定，因此需要把洪荒作为正式世界观资源补入，而不是把洪荒样式当成全局皮肤。

## 目标信息架构

1. 工作台主层级：
   - 主会话
   - 图任务层
   - 灵魂系统
2. 灵魂系统内部层级：
   - 世界观选择：先选择 `默认无背景世界`、`洪荒时代` 等世界观。
   - 灵魂选择：只展示当前世界观下绑定的灵魂卡片。
   - 管理模式：灵魂设定、投影、共同契约。
   - 工作日志：在灵魂视角下展示最近参与过的任务摘要，不复制记忆库内容。
3. 风格规则：
   - 主工作台统一浅蓝底、白色面板、低阴影。
   - 灵魂系统默认使用同一浅蓝控制台风格。
   - 洪荒面板、立绘舞台、故事氛围只在所选世界观为洪荒时出现。

## 实施步骤

1. 接通入口：
   - 在 `frontend/src/app/page.tsx` 引入并渲染 `PlaygroundView`。
   - 在 `frontend/src/components/layout/Sidebar.tsx` 和顶部层级切换中加入“灵魂系统”。
   - 允许 `?view=playground` 进入灵魂系统。
2. 补齐资源数据：
   - 在 `backend/soul/worlds/catalog.json` 增加 `world.honghuang`。
   - 将内置灵魂故事和卡片绑定到 `world.honghuang`，保留 `world.default` 给无背景/纯工作组合。
3. 重构灵魂页结构：
   - 新增世界观选择状态。
   - 从 `resource_catalog` 的 `worlds/cards/stories` 推导当前世界观的灵魂列表。
   - 页面顶部改成世界观选择卡片；选中洪荒世界观时展示洪荒故事与立绘舞台。
   - 灵魂设定和投影列表只展示当前世界观下的灵魂。
4. 统一视觉：
   - 追加 scoped CSS 覆盖主工作台、聊天输入和灵魂系统。
   - 发送按钮增强为明确的蓝色主按钮。
   - 洪荒风格 scoped 到 `.soul-system-console--honghuang`。
5. 验证：
   - 运行前端类型检查或构建中可行项。
   - 启动 Next dev server。
   - 用本地 Edge 浏览器检查主会话、灵魂系统、桌面和窄屏布局。

## 完成标准

- 主导航能进入灵魂系统。
- 灵魂系统首屏先选择世界观。
- 洪荒视觉只在洪荒世界观下出现。
- 默认无背景世界不会展示洪荒面板。
- 发送按钮在主页面清楚可见。
- 页面在桌面和窄屏下无明显重叠、横向溢出或不可操作区域。

## 追加改版：去卡片化视觉

用户反馈当前页面“卡片机制”过重，视觉重复且不好看。保留既定的信息架构，但更换表达方式：

1. 世界观入口从卡片网格改成目录索引式选择：用行列表、编号、状态和活动指示线表达层级，不再用大块卡片。
2. 灵魂选择从卡片改成谱系名单：轻边界、行选择、状态在右侧，不再一项一个厚盒子。
3. 模式切换从卡片改成分段导航：只保留当前段高亮和简短说明。
4. 设定内容区弱化卡片边框，改为文档分节/编辑台风格。
5. 洪荒风格只保留在洪荒世界观的舞台和起源条中，默认世界保持浅蓝工作台。
