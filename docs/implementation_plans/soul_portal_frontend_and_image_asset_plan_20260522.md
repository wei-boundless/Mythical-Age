# 灵魂系统门户与视觉资产重构计划

## 目标

把 `PlaygroundView` 从后台式编辑页重构为“灵魂系统门户”：用户先进入总界面，选择世界观，再经过短过场进入世界页面；世界页面中选择灵魂，灵魂内部承载背景故事、灵魂本体、工作投影和最近工作日志。默认世界保持淡蓝纯工作气质，洪荒风格只在洪荒世界内生效。

## 页面层级

1. 总界面 `SoulHome`
   - 显示当前灵魂/世界的氛围背景。
   - 以带背景图的世界门扉选择世界观。
   - 不直接暴露投影编辑器。
2. 过场 `WorldTransition`
   - 点击世界后播放 0.8-1.2 秒穿越时空动画。
   - 动画结束进入世界页面。
3. 世界页面 `WorldScene`
   - 展示世界背景图、世界介绍、世界内灵魂。
   - 洪荒世界使用洪荒视觉，默认世界使用淡蓝工作风格。
4. 灵魂页面 `SoulPresence`
   - 无边框展示灵魂立绘与背景故事。
   - 灵魂内部切换：背景与本体、工作投影、共同契约。
   - 轻量展示最近工作日志。

## 后端与资产

1. 配置 OpenAI 兼容生图供应商：
   - `SOUL_IMAGE_API_BASE_URL`
   - `SOUL_IMAGE_API_KEY`
   - `SOUL_IMAGE_MODEL`
2. 新增后端视觉资产服务：
   - 读取服务端环境变量，前端不接触密钥。
   - 调用 `/images/generations`。
   - 保存生成图到 `frontend/public/souls/generated`。
   - 暴露生成接口供后续前端管理使用。
3. 给世界观与灵魂具象资源补充图片元数据：
   - 世界门扉图。
   - 世界场景图。
   - 灵魂背景图。

## 前端实施

1. 保留现有数据加载、灵魂切换、投影保存、共同契约保存逻辑。
2. 新增门户状态：
   - `home`
   - `transition`
   - `world`
3. 重写 JSX 结构：
   - 总界面世界门扉。
   - 穿越动画。
   - 世界页介绍和灵魂召唤列。
   - 灵魂详情与工作层。
4. 重写灵魂系统 CSS：
   - 删除旧 Soul Studio 页面样式。
   - 用背景图、遮罩、留白、线性层级替代盒子堆叠。
   - 保证移动端无横向溢出。

## 验证

1. 后端回归：
   - 灵魂资源目录测试。
   - 灵魂模式/活动日志相关测试。
2. 前端：
   - `npm run build`
   - 本地 Edge 打开 `http://localhost:3000/?view=playground`
   - 桌面与移动截图检查：总界面、洪荒世界页、灵魂详情、投影层。

## 完成标准

- 进入灵魂系统先看到总界面和世界门扉。
- 点击世界后有穿越动画再进入世界。
- 世界页面能展示世界背景介绍和灵魂选择。
- 灵魂展示无边框、简洁、具象化。
- 灵魂内部能查看背景、本体、投影、工作日志。
- 密钥只在后端 `.env` 中使用，不进入前端打包。
