# 前端工作台视觉优化计划（2026-05-24）

## 目标

将当前前端统一调整为浅色、克制、密集但清晰的工具型工作台风格，重点优化颜色、线条、字体、边界、表单和面板层级，使其接近成熟 Agent / IDE 控制台的视觉质感。

## 设计系统决策

- 产品类型：Agent 本地工作台 / 多面板控制台。
- 用户角色：开发者、任务编排者、运行监控者。
- 主对象：会话、任务图、运行监控、能力与配置对象。
- 视觉方向：浅色数据工作台，不使用营销式大面积装饰，不使用深色灵魂主题影响控制台主界面。
- 基础色：近白背景、白色面板、浅灰分割线、近黑正文、蓝色作为选中/运行强调。
- 字体：优先使用 Segoe UI / Microsoft YaHei UI / PingFang SC，保持中文界面清晰紧凑；等宽区域使用 Cascadia Code / Consolas。
- 形状：面板和控件半径控制在 6-8px，避免大圆角和厚重卡片。

## 实施步骤

1. 新增全局控制台 token，统一 body、工作台、系统页的浅色背景、文本、边框和 focus ring。
2. 重写工作台 Shell 的关键外观：左栏、右栏、顶部工具条、resize handle、会话列表、项目上下文和中心区域。
3. 统一系统导航 rail：细线边界、紧凑 icon button、清晰 active 状态。
4. 覆盖通用表单、按钮、输入框、空状态、滚动条、代码块，使它们符合工作台风格。
5. 调整移动端和窄屏表现，避免面板错位或文字溢出。
6. 使用真实前端构建和本地浏览器检查视觉结果。

## 第二轮扩展范围

用户确认需要将除灵魂页以外的其它页面也统一为通用 UI，因此追加以下实施项：

1. 给 `SystemPageShell` 增加页面级 class，区分 `chat`、`memory`、`task-system`、`orchestration`、`capability-system`、`system-config`、`system-framework`、`soul-system`。
2. 将系统框架页也纳入系统页 Shell，使其共享左侧系统导航和通用工作台背景。
3. CSS 覆盖层限定为 `:not(.system-page-shell--soul-system)`，避免改动灵魂页视觉。
4. 统一 Memory、Task、Orchestration、Capability、System Config、System Framework 的页面头部、导航面板、列表行、数据卡片、表单、按钮、代码块和状态提示。
5. 对代表性页面逐一截图验证：主会话、记忆、任务、编排、能力、配置、系统框架；灵魂页只检查未被通用覆盖破坏。

## 验收标准

- 页面整体是统一的浅色工作台，而不是深色主题和浅色覆盖混杂。
- 线条、字体、按钮、输入框风格在 Shell 和系统页之间一致。
- active / hover / focus 状态明确且不造成布局位移。
- 登录表单、普通表单和工作台面板都使用同一套边界、阴影和按钮语言。
- `npm run build` 或等价验证通过；若受既有问题阻塞，需要记录具体错误。
