# 灵魂系统现实世界归属与页面重构计划

## 背景

当前灵魂系统入口已经具备世界选择和穿越流程，但默认世界仍以“无背景世界”呈现，并且现实工作资产没有形成独立世界页面。用户明确要求：

1. “无背景世界”改为“现实世界”。
2. 共同契约归入现实世界。
3. 现有无角色投影库，也就是之前河伯投影库里用于纯工作/无角色执行的 prompts，归入现实世界页面。
4. 现实世界页面不能复用洪荒世界页面视觉和结构。
5. 清理前端中的开发语句和无意义噪声信息。

## 目标

把灵魂系统拆成两个真实不同的世界层级：

1. 现实世界
   - 面向真实任务、工作契约、无角色投影和执行入口。
   - 使用清澈、现代、轻量、淡蓝的现实工作界面。
   - 不使用洪荒召唤、神话灵魂本体叙事作为主结构。
2. 洪荒时代
   - 面向洪荒世界观组合和五个灵魂本体。
   - 保留门扉、灵魂火焰、背景故事、投影等沉浸叙事。

## 实施步骤

1. 数据归属梳理
   - 搜索后端世界目录、灵魂资源卡、投影 prompt 和共同契约来源。
   - 确认默认世界 `world.default` 是否可以保持 id 不变，仅改变展示名称与内容，避免破坏现有引用。
   - 找出无角色投影 prompt 的前端/后端字段，归入现实世界页面展示。

2. 世界信息重命名
   - 将 `world.default` 的展示标题从“默认无背景世界”改为“现实世界”。
   - 将入口文案从“无背景/纯工作”改为“现实任务/共同契约/工作投影”。
   - 保留内部 id，减少后端和已有引用的破坏面。

3. 现实世界页面独立设计
   - 在 `PlaygroundView` 中根据世界类型分流：现实世界使用 `renderRealityWorldScene`，洪荒世界使用 `renderMythWorldScene`。
   - 现实世界页面不展示洪荒灵魂召唤列表作为主结构。
   - 现实世界页面主层级为：世界说明、共同契约、无角色投影库、最近工作。
   - 可保留当前灵魂作为辅助状态，但不让它变成页面主角。

4. 噪声清理
   - 前端可见文本禁止出现 `prompt`、内部 id、fallback、authority、runtime、catalog 等开发语句。
   - 工作资产名称改成用户可理解的“工作指令”“共同契约”“执行投影”“现实任务”。

5. 验证
   - `npm run build`
   - 重启默认 3000 前端。
   - 用本地 Edge 验证：
     - 入口显示“现实世界”和“洪荒时代”。
     - 现实世界页面不复用洪荒召唤页结构。
     - 洪荒页面仍保留灵魂选择与灵魂本体。
     - 页面无横向溢出，移动端可用。
