# 会话运行状态实用化计划

## 目标

- 会话底部固定输入区显示真实的“正在干什么”，用户不需要下拉到最后一条消息才能知道状态。
- 状态来自 SSE 事件、工具调用、任务图运行监控和审批状态，不做装饰性假进度。
- 文案保持低噪声：只显示当前阶段、最近事件、任务图/审批关键提醒和少量指标。

## 实施范围

1. Store 增加 `sessionActivity`：
   - 当前阶段标题。
   - 详细说明。
   - 状态级别：`idle | running | waiting | success | error | stopped`。
   - 最近事件、工具名、更新时间。
2. 流式会话事件写入活动状态：
   - 发送前：接收请求。
   - SSE 事件：理解任务、整理上下文、读取记忆、调用工具、生成回答、写入状态。
   - 停止/错误/完成：给出明确收口。
3. 任务图监控补充活动状态：
   - waiting approval 时显示“等待审批”。
   - 有运行中任务图时显示当前 task run / coordination run。
4. 输入框上方新增 `SessionActivityBar`：
   - 固定在会话底部区域。
   - 不使用卡片式大块装饰。
   - 不替代右侧监控，只做会话当前状态摘要。

## 验收标准

- 用户发送消息后，不滚动页面也能看到当前正在做什么。
- 工具调用时能看到工具名。
- 等待审批时状态条明确提示。
- 完成、停止、错误时状态收口，不一直显示“正在思考”。
