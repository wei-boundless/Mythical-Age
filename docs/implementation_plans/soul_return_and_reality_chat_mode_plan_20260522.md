# 灵魂系统返回工作台与主会话现实风格模式计划

## 背景

当前通过 `?view=playground` 进入灵魂系统时，页面会进入强制 Playground 状态，灵魂系统内部没有稳定出口回到主工作台。现实世界页面的视觉语言已经更符合用户预期，可以迁移为主会话的可切换外观模式。

## 目标

1. 灵魂系统新增“回到工作台/主会话”出口。
2. 从 URL 强制进入灵魂系统时，返回操作必须清掉强制状态和 query 参数。
3. 主会话增加可切换外观模式：默认模式 / 现实模式。
4. 现实模式只改变主会话视觉层级，不破坏现有聊天输入、消息、任务选择和流式逻辑。
5. 现实模式沿用现实世界的清澈、现代、执行台气质，但保留主会话的实用密度。

## 实施步骤

1. 路由与状态
   - 在 `page.tsx` 中为 PlaygroundView 传入返回回调。
   - 回调设置工作区为 `chat`，关闭 `forcedPlayground`，并移除 URL 中的 `view=playground`。

2. 灵魂系统出口
   - 在 PlaygroundView 的首页和世界页增加“回到工作台”按钮。
   - 该按钮调用上层回调，不只改变内部 portal stage。

3. 主会话模式
   - 在 `page.tsx` 增加 `chatVisualMode` 状态并持久化到 localStorage。
   - 仅主会话启用现实模式 class。
   - 在 ChatPanel 头部加入默认/现实模式切换。

4. 视觉实现
   - 为现实模式增加背景、主内容、消息区、输入区、消息气泡、侧边栏的轻量覆盖样式。
   - 避免硬卡片堆叠，使用线性光带、透明层、左边界和空间感。

5. 验证
   - `npm run build`
   - Edge 验证：灵魂系统可返回主会话；主会话模式可切换；刷新后保持选择；无横向溢出。
