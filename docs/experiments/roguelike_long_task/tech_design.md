# 技术设计文档

## 架构概览
游戏采用单页 HTML 文件内嵌 JavaScript 和 CSS 的架构，所有资源放在 `docs/experiments/roguelike_long_task/` 下，便于在浏览器直接打开或通过简单 HTTP 服务器运行。

## 模块划分

### 1. 游戏引擎 (Game Engine)
- **Game 类**：主控制器，管理游戏循环、状态切换（菜单、游戏运行、暂停、结束）。
- **Canvas 渲染器**：使用 Canvas 2D API 绘制游戏画面，支持多层渲染（背景、实体、UI）。
- **输入管理**：监听键盘事件（WASD/方向键移动，空格键攻击等）和鼠标事件（点击攻击）。
- **资源加载器**：加载图像、音效（可选）等资源，使用 Promise 异步加载。

### 2. 世界与房间 (World & Rooms)
- **DungeonGenerator 类**：程序化生成每层地下塔的房间布局。
- **Room 类**：管理单个房间内的敌人、门、物品等，响应玩家进入/离开事件。
- **TileMap 类**：管理二维瓦片地图数据，支持碰撞检测、门开关等。

### 3. 实体系统 (Entities)
- **Entity 基类**：位置、尺寸、速度、生命值等通用属性，更新与渲染虚方法。
- **Player 类**：继承 Entity，处理玩家移动、攻击、经验、升级等逻辑。
- **Enemy 类**：继承 Entity，包含 AI 状态机（巡逻、追击、攻击），不同类型子类。
- **Projectile 类**：继承 Entity，直线飞行的子弹/箭矢，碰撞检测。
- **Boss 类**：继承 Enemy，特殊攻击模式和阶段切换。

### 4. AI 系统
- **状态机**：每个敌人有 Idle/Patrol/Chase/Attack 状态，状态间转换由条件触发。
- **路径寻找**：简单 A* 寻路（或基于方向的移动），避开障碍物。
- **行为树（可选）**：用于复杂的 Boss AI。

### 5. 战斗系统
- **伤害计算**：攻击力 - 防御力（或简单减法），暴击概率。
- **碰撞检测**：使用矩形碰撞检测（AABB），实现攻击命中判定。
- **效果系统**：增益/减益效果，如燃烧、减速、护盾等，含持续时间和触发逻辑。

### 6. UI/HUD
- **HUD 类**：管理生命条、经验条、技能图标等动态 UI，使用 Canvas 绘制或 CSS 覆盖。
- **菜单系统**：弹出升级选择、游戏结束界面、通关界面。

### 7. 数据管理
- **GameData 对象**：存储全局游戏状态（玩家属性、当前层数、统计等）。
- **Save/Load（可选）**：暂不实现，未来可增加 Web Storage 支持。

## 技术选型
- **渲染**：HTML5 Canvas 2D，无需 WebGL。
- **音频**：暂不实现音效（超出 MVP 范围），可使用 Web Audio API 按需添加。
- **图像资源**：PNG 位图，由 AI 图像生成工具生成并保存在本地。
- **框架**：无，纯 JavaScript 实现，便于控制和迭代。

## 数据流
1. 用户输入 → 输入管理器 → Player 状态更新。
2. Game Loop 每帧调用所有 Entity 的更新方法，包括 Enemy AI 决策和 Projectile 移动。
3. 碰撞检测系统检测所有实体间的碰撞，触发伤害/效果。
4. 世界模块更新房间状态（门开关、敌人死亡等）。
5. 渲染器清空画布，按层绘制背景、实体、UI。

## 性能优化
- 只渲染视野内的实体（视锥剔除）。
- 使用 off-screen canvas 预处理静态背景。
- 限制每帧实体更新数量（对大量投射物可进行对象池管理）。

## 错误处理与调试
- 使用 `try-catch` 包围主循环，捕获异常并显示错误提示。
- 提供调试模式（通过 URL 参数 `?debug` 开启），绘制碰撞框、AI 状态、FPS 等。

## 部署与运行
- 直接打开 `index.html` 文件（部分浏览器可能因跨域限制无法加载本地图像，建议使用 Live Server 或 `http-server`）。
- 兼容 Chrome、Firefox、Edge 等现代浏览器。
