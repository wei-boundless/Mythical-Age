# MVP 实现说明

## 概述
本文档描述 2D 俯视角 Roguelike 游戏垂直切片的实现方案，涵盖核心系统、游戏流程和边界条件。

## 入口文件
- `index.html`：单文件包含所有 HTML、CSS 和 JavaScript 游戏逻辑。
- 图像资产：玩家位图从 `frontend/public/souls/generated/` 加载（若加载失败使用 Canvas 绘制）。
- 其他敌人类、环境资产均使用 Canvas 程序化绘制，保证 MVP 可运行。

## 核心系统实现

### 1. 游戏循环与状态管理
- 使用 `requestAnimationFrame` 驱动主循环。
- GameState 枚举：MENU、PLAYING、UPGRADE_SELECT、GAMEOVER、VICTORY。
- Game 类管理状态切换、实体更新、渲染调度。

### 2. 输入处理
- 键盘事件：WASD/方向键控制移动方向，空格键攻击。
- 连续按键处理：记录当前按下的键，每帧读取。
- 升级选择：使用鼠标点击对应选项。

### 3. 实体系统
- Entity 基类：位置 (x, y)、宽度 (w, h)、速度 (vx, vy)、生命值 (hp, maxHp)、攻击力 (atk)、颜色/图像。
- Player 子类：继承 Entity，添加经验值 (exp)、等级 (lvl)、升级所需经验 (expToNext)、技能增强属性。移动速度由键盘输入控制，攻击冷却 (attackCooldown) 限制攻击频率。
- Enemy 子类：继承 Entity，添加 AI 状态机、巡逻路径、检测范围、攻击间隙。
- Projectile 子类：继承 Entity，表示子弹/箭矢，有伤害和寿命。
- Boss 子类：继承 Enemy，增加阶段（phase）、特殊攻击冷却、技能模式。

### 4. 世界与房间
- Room 类：定义矩形区域（x, y, w, h），存储门位置、敌人列表、类型（普通/Boss/起始）。房间背景颜色区分。
- Dungeon 类：管理当前层的房间集合、当前房间索引。生成房间布局：随机位置，确保连通性（门对齐）。
- 门机制：进入房间后门关闭，清除所有敌人后门打开。Boss 房间击败 Boss 后出现通往下一层楼梯。

### 5. 敌人 AI
- 状态机：Idle → Patrol → Chase → Attack
  - Idle：在随机点停顿。
  - Patrol：沿预设路径点移动。
  - Chase：玩家进入检测范围后追击。
  - Attack：在攻击范围内时发动攻击，冷却后再次攻击。
- 不同类型敌人行为差异：
  - 骷髅兵：近战，移速中，攻击力低。
  - 骷髅弓箭手：远程，与玩家保持距离，发射 Projectile。
  - 暗影刺客：高移速，周期性隐身（闪烁效果），突袭。
  - Boss：生命低于 50% 进入狂暴，增加速度/攻击模式。

### 6. 战斗系统
- 攻击判定：玩家攻击生成一个攻击矩形（前方扇形简化为宽矩形），与敌人碰撞检测。
- 伤害计算：攻击力直接扣除 HP（无防御）。暴击：10% 概率 1.5 倍伤害（可升级）。
- 投射物：敌人发射的箭矢，碰撞玩家造成伤害。
- 死亡：HP ≤ 0，玩家进入死亡状态；敌人移除并给予经验。

### 7. 升级系统
- 经验获取：击杀骷髅兵 20 EXP，弓箭手 30 EXP，刺客 50 EXP，Boss 100 EXP。
- 升级阈值：每级所需经验 = level * 50 + 50。
- 升级时暂停游戏，弹出三个随机选项：
  1. 攻击力 +5
  2. 最大生命值 +20（同时恢复等量）
  3. 移动速度 +10%
  4. 生命偷取（击杀回复 10% 最大生命）
  5. 闪电链（攻击有 20% 概率对附近敌人造成 50% 伤害）
  6. 护盾（每层开始获得 20 点护盾）
- 实现：从预定义列表中随机抽取 3 个不重复选项，应用效果并更新玩家属性。

### 8. UI/HUD
- Canvas 上层绘制：
  - 左上角：生命条（红色背景，绿色填充），经验条（蓝色），等级数字。
  - 右上角：当前层数（如 "Floor 1/5"）。
  - 底部：技能图标（仅显示已激活技能）。
  - 菜单界面：游戏标题、开始按钮。
  - 升级选择界面：半透明遮罩，三个按钮显示选项效果。
  - 死亡画面：统计击杀数、层数、时间，重试按钮。
  - 胜利画面：通关文字。

### 9. 程序化内容
- 房间生成：每层 5-8 个房间，使用 BSP 或简单随机放置。
- 敌人分布：每个房间根据类型生成 1-4 个敌人（最后一间为 Boss）。
- 道具：无宝箱，仅升级选项。

## 游戏流程
1. 启动：显示标题画面，点击“Start”开始。
2. 第一层：生成房间，玩家从起始房间开始，依次击败各房间敌人，进入 Boss 房间。
3. 击败 Boss 后，显示“进入下一层”提示，点击后重置房间布局并增加敌人强度（生命/攻击系数）。
4. 第五层 Boss 击败后显示胜利画面。
5. 玩家死亡则显示死亡统计，点击重试重新开始。

## 边界与限制
- 无音效（MVP 范围外）。
- 无存档功能（可后续增加）。
- 敌人 AI 不使用 A* 寻路，采用方向移动（避开墙壁）。
- 墙壁碰撞：所有实体不能穿越墙壁，移动前检测。
- 地图大小：约 800x600 像素的 Canvas 视口，世界大小与 Canvas 相同，无需滚动（后续可扩展）。
