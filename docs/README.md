## 三、文档治理规则

后续新增文档统一按下面规则执行：

1. `设计原则` 只记录外部参考、方法论和可迁移设计原则，不直接承担本项目实施蓝图。
2. `系统规划` 只记录本项目正式边界、源码对照、实施蓝图和补充设计，不再混入泛泛外部框架综述。
3. 一个问题如果已经进入“本项目正式决策”，就应该在 `系统规划` 落盘，不应只停留在 `设计原则`。
4. 文档文件名统一保留编号，编号表达阅读顺序和依赖关系。
5. 文档一旦成为正式依据，应尽量通过新增补充文档推进，不随意回滚历史结论。
