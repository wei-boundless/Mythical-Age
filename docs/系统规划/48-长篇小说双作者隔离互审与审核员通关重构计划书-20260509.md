# 48-长篇小说双创作者单裁判生产架构重构计划书-20260509

## 背景

长篇小说百万字任务不能依赖多个 Agent 反复共享完整讨论上下文。过厚、过杂的上下文会污染写作方向，让角色在解释流程、讨论任务、互相揣测中偏离正文生产。

新的结构原则是：Agent 不共享原始会话上下文，只共享经过授权的写作资产、稿件和审核单。

同时，长篇小说不是一次性生成任务，而是长期生产系统。它需要先建立稳定的世界观和结构大纲，再进入正文生产；进入正文生产后，世界观和大纲不能随意反复重写，只能在既成小说事实基础上做偏差修正和有限创新。

## 重构目标

1. 把长篇小说任务拆成基础定稿层、正文生产层、偏差修正层和完结收束层。
2. 把世界观设计和结构大纲划分分开处理。
3. 世界观和大纲初步定稿后，分别进入世界观记忆库和大纲记忆库，成为后续写作参照。
4. 把章节生产改为双创作者隔离互审流程。
5. 增加独立审核员 Agent，作为章节、设定修正和大纲修正的通关裁判。
6. 审核员以小说专家角度进行严格、公正、可执行的通关评分。
7. 审核员关注两位创作者互审方向是否偏离原设定、既成正文、人物动机、世界观边界和长篇走向。
8. 只有通过审核的章节终稿、新设定修正稿或大纲修正稿才能进入对应记忆库。

## 总体架构

```text
用户硬设定
  -> 世界观初设
  -> 结构大纲细则
  -> 基础审核通关
  -> 世界观记忆库 / 大纲记忆库
  -> 章节循环
  -> 偏差修正循环
  -> 完结收束
```

### 基础定稿层

基础定稿层负责把用户给出的少量设定转化为可长期执行的创作参照。

世界观设计负责：

1. 建立世界运行规则、神灵关系、地理空间、力量边界和历史记忆。
2. 只扩展用户给出的硬设定，不推翻用户给出的基础。
3. 输出可供正文调用的世界规则，而不是散乱名词表。

结构大纲负责：

1. 建立全书主线、分卷结构、阶段目标和章节推进逻辑。
2. 把大纲写成细则，明确每个阶段的钩子、冲突、人物变化和信息释放。
3. 为写手提供可执行引导，而不是空泛方向。

基础定稿通过后，世界观进入世界观记忆库，大纲进入大纲记忆库。进入正文生产后，二者默认不再随意重写。

## 目标章节流程

```text
章节大纲
  -> 作者A初稿
  -> 作者B审读意见
  -> 作者A综合修订稿
  -> 作者B终稿处理
  -> 审核员评分通关
  -> 创作资产整理
  -> 下一章
```

## 偏差修正流程

正文生产过程中，如果终稿暴露出原世界观、大纲或人物规划的偏差，不允许直接回头重写基础资产，而是进入偏差修正流程。

```text
既成正文事实
  -> 设定修正提议A
  -> 设定修正提议B
  -> 审核员仲裁
  -> 通过：写入世界观记忆库或大纲记忆库
  -> 不通过：保持原设定，返回章节修订
```

偏差修正不是“重新创作世界观”，而是在既成小说事实基础上修正原设定与正文之间的偏差。

## 上下文授权原则

1. 作者A只读取用户硬设定、已采用项目资产、当前章大纲、必要前情摘要和上一轮审核单。
2. 作者B只读取用户硬设定、已采用项目资产、当前章大纲、作者A交付稿件和必要前情摘要。
3. 审核员只读取用户硬设定、已采用项目资产、当前章大纲、作者A稿件、作者B意见与终稿、评分规则。
4. 记忆整理员只读取通过审核的终稿、审核单和本章新增事实。
5. 不把任何角色的原始思考过程、内部提示词、未授权上下文或全量会话历史传给其他角色。

补充原则：

1. 写手需要尊重原设定，但允许在正文中做局部创造，只要不破坏硬设定和已定主线。
2. 审核员必须更忠实于原设定、已定大纲和既成小说事实。
3. 审核员不需要长期记忆，不负责储存事实；它每轮只根据授权输入做仲裁。
4. 世界观设计师进入正文生产后要重新绑定契约和投影，从“初设创造者”转为“偏差修正提议者”。
5. 大纲设计师进入正文生产后也要从“全书规划者”转为“结构偏差修正提议者”。

## 审核员契约方向

审核员不是普通挑错器，也不是第三作者。它是章节、设定修正和结构修正的通关裁判。

审核员必须：

1. 以长篇小说专家角度评审章节。
2. 保持公正、严格、稳定，不偏袒作者A或作者B。
3. 判断章节是否真正推动故事，而不是只堆设定或气氛。
4. 关注两位作者互审方向是否让剧情偏离用户硬设定、项目主线、人物动机和世界观边界。
5. 输出评分、通关结论、必须修复项和下一轮修改目标。
6. 不直接重写正文，不替代作者完成创作。
7. 在设定修正环节，只仲裁两边提议是否合理，不亲自创造第三套设定。
8. 在结构修正环节，判断修正是否服务既定长篇走向，而不是为了当前章节方便牺牲全书结构。
9. 对过松和过紧都保持警惕：过松会让作品漂移，过紧会让正文停滞。

审核员的判断基准：

1. 用户硬设定优先。
2. 已通过审核的世界观记忆和大纲记忆优先。
3. 已成文且通过审核的正文事实优先。
4. 写手局部创造可以被保留，但必须能解释为原设定的自然延展。
5. 不能因为某个提议更华丽、更方便或更刺激，就放弃原设定和已成事实。

## 评分建议

满分 100，通过线 85。

1. 剧情推进：20
2. 人物动机与行为可信度：20
3. 世界观一致性与边界感：15
4. 章节结构完整度：15
5. 语言质感与可读性：10
6. 长篇伏笔和后续延展：10
7. 双作者互审方向控制：10

低于 85 分时，章节不得进入记忆整理。审核员只给可执行的返修目标，不写正文。

## 双创作者单裁判机制的推广

双创作者单裁判机制不只用于章节正文，也用于所有会改变作品事实的环节。

适用环节：

1. 世界观初设。
2. 全书结构和分卷大纲。
3. 单章正文生产。
4. 世界观偏差修正。
5. 大纲偏差修正。
6. 人物线偏差修正。

不适用环节：

1. 记忆整理。
2. 产物落盘。
3. 字数统计。
4. 文件清单汇总。

这些环节只需要忠实记录，不需要创作性互审。

## 成员组定义

### 1. 总协调组

#### 总制片 Agent

建议 Agent ID：`agent:novel_showrunner`

职责：

1. 接收用户硬设定和任务目标。
2. 启动基础定稿、章节循环、偏差修正和最终收束。
3. 判断当前阶段应该调用哪个创作组。
4. 判断审核失败后应返回哪一组继续修订。
5. 控制循环是否继续，直到作品达到计划章节数或目标字数。

不能做：

1. 不直接写正文。
2. 不直接修改世界观记忆库、大纲记忆库、人物记忆库或正文事实库。
3. 不跳过审核员将产物送入记忆库。

读取：

1. 用户硬设定。
2. 所有已通过审核的记忆库。
3. 当前阶段运行状态。
4. 审核员评分单和通关结论。

写入：

1. 阶段调度指令。
2. 循环控制状态。
3. 偏差修正触发记录。

### 2. 世界观创作组

#### 世界观设计师 A

建议 Agent ID：`agent:novel_world_designer_a`

职责：

1. 在基础定稿阶段提出世界观方案 A。
2. 在偏差修正阶段根据既成正文事实提出世界观修正方案 A。
3. 重点关注世界运行规则、神灵关系、地理秩序、力量边界和历史记忆。

不能做：

1. 不直接写正文。
2. 不直接写入世界观记忆库。
3. 不推翻用户硬设定。

#### 世界观设计师 B

建议 Agent ID：`agent:novel_world_designer_b`

职责：

1. 在基础定稿阶段提出世界观方案 B，或对方案 A 做替代性修订。
2. 在偏差修正阶段根据既成正文事实提出世界观修正方案 B。
3. 重点检查方案 A 是否过于松散、过度设定化或不利于正文推进。

不能做：

1. 不直接写正文。
2. 不直接写入世界观记忆库。
3. 不把方便当前章节的设定改动包装成世界观升级。

共同读取：

1. 用户硬设定。
2. 已通过审核的世界观记忆库。
3. 已通过审核的正文事实库。
4. 审核记录库中的相关返修意见。

共同写入：

1. 世界观候选方案。
2. 世界观修正提议。

### 3. 大纲结构组

#### 大纲结构师 A

建议 Agent ID：`agent:novel_outline_designer_a`

职责：

1. 在基础定稿阶段提出全书结构、分卷计划、阶段目标和章节钩子方案 A。
2. 在偏差修正阶段提出结构修正方案 A。
3. 重点保证百万字长篇有持续推进路线。

不能做：

1. 不直接写正文。
2. 不直接写入大纲记忆库。
3. 不把大纲写成抽象口号。

#### 大纲结构师 B

建议 Agent ID：`agent:novel_outline_designer_b`

职责：

1. 在基础定稿阶段提出结构方案 B，或对方案 A 做替代性修订。
2. 在偏差修正阶段提出结构修正方案 B。
3. 重点检查章节钩子、人物变化、信息释放和分卷节奏是否足够细。

不能做：

1. 不直接写正文。
2. 不直接写入大纲记忆库。
3. 不为了当前章节便利牺牲全书结构。

共同读取：

1. 用户硬设定。
2. 已通过审核的大纲记忆库。
3. 已通过审核的正文事实库。
4. 审核记录库中的相关返修意见。

共同写入：

1. 大纲候选方案。
2. 大纲修正提议。
3. 章节细则候选。

### 4. 人物设定组

#### 人物设计师 A

建议 Agent ID：`agent:novel_character_designer_a`

职责：

1. 提出主角、关键人物、关系网、秘密、能力和人物弧方案 A。
2. 在正文暴露人物偏差时提出人物修正方案 A。

#### 人物设计师 B

建议 Agent ID：`agent:novel_character_designer_b`

职责：

1. 提出人物方案 B，或对方案 A 做替代性修订。
2. 检查人物动机、关系变化和成长弧是否可信。

共同读取：

1. 用户硬设定。
2. 人物记忆库。
3. 世界观记忆库。
4. 大纲记忆库。
5. 正文事实库。

共同写入：

1. 人物候选方案。
2. 人物偏差修正提议。

### 5. 正文写作组

#### 写手 A

建议 Agent ID：`agent:novel_writer_a`

职责：

1. 根据章节细纲写初稿。
2. 根据写手 B 的意见写综合修订稿。
3. 在审核未通过时根据审核员返修要求继续修订。

特点：

1. 更偏创造、场景推进、人物行动和叙事展开。
2. 可以在原设定边界内进行局部自由创造。

不能做：

1. 不直接修改世界观、大纲或人物稳定记忆。
2. 不把未审核通过的新设定当成既成事实。

#### 写手 B

建议 Agent ID：`agent:novel_writer_b`

职责：

1. 阅读写手 A 的稿件，提出审读意见。
2. 对修订稿进行二次处理，形成候选终稿。
3. 在审核未通过时根据审核员返修要求继续与写手 A 互改。

特点：

1. 更偏审读、纠偏、节奏控制、终稿处理。
2. 关注初稿是否偏离章节目标、人物动机和大纲钩子。

不能做：

1. 不直接修改稳定记忆库。
2. 不把自己的审读意见写成审核员裁决。

共同读取：

1. 用户硬设定。
2. 世界观记忆库。
3. 大纲记忆库。
4. 人物记忆库。
5. 正文事实库中的必要前情摘要。
6. 当前章节细纲。
7. 上一轮审核员返修要求。

共同写入：

1. 初稿。
2. 审读意见。
3. 综合修订稿。
4. 候选终稿。

### 6. 审核裁判组

#### 小说审核员

建议 Agent ID：`agent:novel_quality_reviewer`

职责：

1. 审核每个创作环节是否可以结束。
2. 仲裁两个创作者的方案是否符合原设定、既成事实、当前目标和小说质量要求。
3. 输出评分单、通关结论、返修要求和偏差类型。
4. 判断是否需要触发世界观、大纲或人物偏差修正。

核心原则：

1. 不保存长期上下文。
2. 不参与创作过程。
3. 不继承两个创作者的内部讨论。
4. 不创造第三套方案。
5. 只根据本轮提交材料和授权记忆进行裁判。

读取：

1. 用户硬设定。
2. 当前任务目标。
3. 对应记忆库中已经通过审核的内容。
4. 创作者 A 的作品。
5. 创作者 B 的作品或修改稿。
6. 最终候选稿。
7. 评分规则。

写入：

1. 审核评分单。
2. 通关或不通关结论。
3. 必须修复项。
4. 可保留项。
5. 偏差类型。

不能做：

1. 不写正文。
2. 不写世界观正文。
3. 不写大纲正文。
4. 不写长期记忆。
5. 不把个人偏好当成裁决依据。

### 7. 记忆整理组

#### 记忆整理员

建议 Agent ID：`agent:memory_steward`

职责：

1. 只整理已经通过审核的内容。
2. 根据审核结论把内容写入对应记忆库。
3. 区分用户硬设定、已通过设定、正文事实、人物状态、候选内容和待定冲突。

读取：

1. 通过审核的最终稿。
2. 审核员评分单。
3. 审核员通关结论。
4. 总协调给出的写入目标。

写入：

1. 世界观记忆库。
2. 大纲记忆库。
3. 人物记忆库。
4. 正文事实库。
5. 审核记录库。

不能做：

1. 不创造新设定。
2. 不补写正文。
3. 不把未通过审核的内容写成稳定事实。

```text

   |

基础定稿
   |
   +--> 世界观记忆库
   +--> 大纲记忆库

   |       |

   |       |
   |       +--> 审核不通过 --> 写手 A/B 继续互改
   |
   +--> 发现偏差
           |
           +--> 世界观偏差修正
           +--> 大纲偏差修正
           +--> 人物偏差修正
```

## 拓扑规则

1. 每个创作节点都必须有审核员作为退出条件。
2. 创作者之间可以互相修改，不需要每一步显式等待审核员。
3. 审核员只在环节试图结束时介入。
4. 审核员未通过，当前环节不得结束。
5. 未通过内容不得写入记忆库。
6. 未通过内容不得成为后续章节依据。
7. 记忆库只接收审核通过后的最终产物。
8. 审核员不存长期上下文，只看本轮授权材料。
9. 写手可以局部自由创造，审核员必须忠实于原设定、既成事实和全书质量。
10. 世界观、大纲、人物的修正必须基于既成正文事实，不能为了当前方便重新发明系统。

## 记忆库划分

长篇小说至少需要以下记忆库：

1. 世界观记忆库：存放已通过审核的世界规则、神灵关系、地理秩序、力量边界和修正记录。
2. 大纲记忆库：存放已通过审核的全书结构、分卷目标、阶段钩子和章节细则。
3. 人物记忆库：存放已通过审核的人物身份、关系、动机、能力、秘密和状态变化。
4. 正文事实库：存放已通过审核章节中已经发生的事件和不可回退事实。
5. 审核记录库：存放评分、通关结论、返修要求和偏差修正理由。

审核员不直接写长期记忆。记忆整理员只整理已经通过审核的内容。

## 停止条件

章节循环持续推进，直到满足以下条件：

1. 已完成章节数达到计划章节数，或总字数达到目标规模。
2. 最后一章通过审核。
3. 世界观记忆库、大纲记忆库、人物记忆库和正文事实库完成最终整理。
4. 最终产物清单真实反映已经完成的正文和资产。
