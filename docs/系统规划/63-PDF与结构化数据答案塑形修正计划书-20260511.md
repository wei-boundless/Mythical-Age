# 63-PDF与结构化数据答案塑形修正计划书

日期：2026-05-11

## 背景

最新六十轮长场景实测为 45/53。路由主链已明显收敛，剩余非长记忆问题集中在能力执行后的答案塑形：

- PDF 能力已正确进入 `mcp_pdf`，但对“压成三条行动建议、每条带行动动词”的请求仍返回文档摘要。
- 结构化数据能力已正确进入 `mcp_structured_data`，但对“哪些仓库完全没有缺口 / 是否存在完全没有缺口的仓库”仍返回库存概览。
- 本地知识库问答实际使用 `template.rag.knowledge_answer`，但长场景合同仍检查旧 `plan.route=rag`。

## 目标

1. PDF 能力层能识别用户要求的输出形态，并在已有证据摘要基础上生成可直接展示的行动建议。
2. 结构化数据能力层能把库存“完全没有缺口的仓库”识别为按仓库聚合后的精确判断。
3. 长场景合同不再用旧 route 标签误判已正常工作的 RAG 工具链。

## 实施原则

- 不在测试中伪造结果。
- 不在长跑 runner 中针对单个轮次写补丁。
- 将修正落在 PDF worker / 结构化数据 planner-engine 这类能力边界内。
- 继续保留长记忆问题为后续专项，不混入本轮收口。

## 实施步骤

### 阶段一：结构化数据库存缺口精确问答

- 在 `StructuredDataPlanner` 中识别“完全没有缺口 / 没有缺口 / 无缺口 + 仓库/地区”等库存状态查询。
- 增加专门分析类型，用于判断每个仓库的总缺口是否为 0。
- 在 `StructuredDataEngine` 中按 `warehouse` 分组计算 `shortage_qty = max(reorder_level - stock_on_hand, 0)`。
- 若没有满足条件的仓库，直接回答“没有完全没有缺口的仓库。”。

### 阶段二：PDF 行动建议输出塑形

- 在 `PDFWorker` 的 canonical answer 生成阶段识别“行动建议 / 三条 / 行动动词 / 压成”等输出请求。
- 基于 canonical summary 与 evidence 生成三条以行动动词开头的建议。
- 不改 PDF 解析和证据定位算法，只修能力结果向用户答案的映射。

### 阶段三：长场景合同更新

- 将本地知识库问答断言从 `plan.route=rag` 调整为当前统一工具通道的真实契约：`plan.tool=search_text` + `event=retrieval`。

### 阶段四：验证

- 增加或扩展能力质量回归测试。
- 运行能力回归、理解/模板回归和长场景复测。
