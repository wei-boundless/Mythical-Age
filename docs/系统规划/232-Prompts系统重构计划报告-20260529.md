# Prompts 系统重构计划报告

日期：2026-05-29

关联文档：

- `docs/系统框架/015-Prompts系统框架设计书-20260529.md`

## 1. 重构目标

建立正式 Prompts 系统，让稳定 prompt 资产按类别统一管理，并让单 agent harness、TaskRun、图任务、特定任务、环境、agent profile、skill、soul 都通过同一套 prompt 装配机制进入 runtime packet。

本次重构不是修 prompt 文案，而是修 prompt 权威边界：

```text
Stable Prompt Assets
-> PromptAssembly
-> RuntimeProjection
-> RuntimeCompiler
-> RuntimeInvocationPacket
```

## 2. 当前问题

### 2.1 RuntimeCompiler 硬编码大段 prompt

文件：

- `backend/harness/runtime/compiler.py`

问题：

- `turn_action`、`task_execution`、`observation_followup` 的核心 prompt 直接硬编码。
- 历史 bug 修复规则逐渐堆进 task_execution prompt。
- prompt 资源不可版本化，不便审计和缓存。

### 2.2 Agent / Environment prompt 内联

文件：

- `storage/orchestration/agent_runtime_profiles.json`
- `backend/agent_system/profiles/runtime_profile_registry.py`
- `backend/task_system/environments/default_environments.py`

问题：

- agent work role prompt 内联在 profile metadata。
- environment prompt 内联在 environment definition。
- 这些稳定 prompt 资产没有统一版本、分类和 manifest。

### 2.3 旧 prompt 库边界不清

文件：

- `backend/prompt_library/*`
- `storage/prompt_library/prompt_resources.json`

问题：

- 有 prompt resource 模型，但类别不完整。
- 主要保存旧图任务 `stage_role`。
- selector 仍使用 `task_goal_type`。
- assembler 仍生成旧任务类型 prompt。

### 2.4 旧 prompting 仍被 API 引用

文件：

- `backend/prompting/*`
- `backend/query/runtime.py`
- `backend/api/sessions.py`
- `backend/api/tokens.py`

问题：

- 旧 static prompt 仍用于 session system_prompt 展示和 token 估算。
- 这不应再代表新 harness 的真实 prompt。
- 后续应改为读取 RuntimePromptManifest / prompt accounting ledger。

## 3. 重构边界

### 3.1 本次允许改动

- Prompts Library 数据模型。
- Prompt pack 定义。
- RuntimeCompiler prompt 来源。
- Agent/environment prompt 引用方式。
- Graph/specific task prompt contract 接入。
- Prompt manifest 和 prompt accounting 接口。
- 旧 prompt selector 的降权或删除。

### 3.2 本次不改变

- TaskRun 状态机语义。
- 工具执行器。
- Operation authorization 的决策权。
- Graph execution scheduler。
- 用户消息提交和 session history 基本结构。

## 4. 目标文件结构

建议保留 `backend/prompt_library` 目录名，先改 authority 和模型，避免一次性改大量 import。后续可再迁移为 `backend/prompts`。

目标模块：

```text
backend/prompt_library/
  models.py
  registry.py
  packs.py
  assembly.py
  manifest.py
  migration.py
  validation.py
  runtime_sections.py
```

存储：

```text
storage/prompt_library/
  prompt_resources.json
  prompt_packs.json
```

长期目标：

```text
storage/prompts/
  resources.json
  packs.json
  manifests/
```

## 5. 分阶段实施

## Phase 1：建立正式 PromptResource / PromptPack 模型

### 目标

让 prompts 库支持正式分类、版本、状态、缓存边界和 prompt pack。

### 文件

- `backend/prompt_library/models.py`
- `backend/prompt_library/registry.py`
- 新增 `backend/prompt_library/packs.py`
- 新增 `backend/prompt_library/manifest.py`
- `backend/tests/prompt_library_*`

### 具体改动

1. 扩展 `PromptResource`：

```text
prompt_id
category
subtype
version
owner_layer
cache_scope
allowed_invocation_kinds
allowed_runtime_modes
allowed_agent_refs
allowed_environment_refs
status
```

2. 保留旧字段只用于迁移读取：

```text
resource_id -> prompt_id
resource_type -> category/subtype 映射
applies_to_task_goal_types -> deprecated metadata
```

3. 新增 `PromptPack`：

```text
pack_id
invocation_kind
ordered_prompt_refs
cache_scope
status
```

4. Registry 支持：

- list active resources
- get prompt by id
- get pack by id
- reject deprecated resource for new runtime assembly

### 验收

- 新旧 prompt_resources.json 可读取。
- 新资源可以按 category/subtype 查询。
- pack 可以解析为 ordered prompt refs。
- deprecated 资源不会进入新主链。

### 禁止

- 不在 Phase 1 改 RuntimeCompiler 行为。
- 不删除旧资源文件。

## Phase 2：迁移 RuntimeCompiler 硬编码 prompt 到 runtime prompt pack

### 目标

RuntimeCompiler 不再直接维护大段稳定 prompt 文案。

### 文件

- `backend/harness/runtime/compiler.py`
- `backend/prompt_library/packs.py`
- `backend/prompt_library/assembly.py`
- `storage/prompt_library/prompt_resources.json`
- `storage/prompt_library/prompt_packs.json`

### 具体改动

1. 建立 runtime prompt resources：

- `runtime.turn_action.v1`
- `runtime.task_execution.v1`
- `runtime.observation_followup.v1`
- `runtime.action_schema.v1`
- `runtime.failure_recovery.v1`
- `runtime.completion_self_review.v1`

2. 建立 runtime packs：

- `runtime.pack.turn_action.v1`
- `runtime.pack.task_execution.v1`
- `runtime.pack.observation_followup.v1`

3. RuntimeCompiler 输入变为：

```text
PromptAssemblyResult
RuntimeProjection
StablePayload
VolatilePayload
```

4. 保留 `_runtime_projection_instruction()` 或迁移到 projection builder，但它不进入 Prompts Library。

### 验收

- `compile_turn_action_packet()` 仍输出合法 schema。
- `compile_task_execution_packet()` 仍把 stable contract 放在 volatile observations 之前。
- runtime prompt 文案来自 prompt resource。
- packet manifest 记录 prompt refs。
- 现有 focused tests 通过。

### 禁止

- 不把 observations 写入 prompt resource。
- 不把 mode instruction 放回 prompt resource。
- 不把工具授权逻辑写入 prompt 文案。

## Phase 3：迁移 agent / environment / soul prompt

### 目标

把 agent role prompt、environment prompt、soul role prompt 接入 Prompts Library。

### 文件

- `backend/agent_system/profiles/runtime_profile_registry.py`
- `storage/orchestration/agent_runtime_profiles.json`
- `backend/task_system/environments/default_environments.py`
- `backend/task_system/environments/models.py`
- `backend/soul/assembly_service.py`
- `backend/prompt_library/registry.py`

### 具体改动

1. Agent profile：

```text
metadata.work_role_prompt -> prompt_ref: agent.main.work_role.v1
```

2. Environment：

```text
EnvironmentPrompt.content -> prompt_ref: environment.development.sandbox.v1
```

3. Soul：

```text
soul role prompt -> category=soul, subtype=role_persona
```

4. RuntimeAssembly 收集：

```text
agent_prompt_refs
environment_prompt_refs
soul_prompt_ref
```

### 验收

- role mode 可以装配 soul prompt。
- standard/professional 不装配 soul prompt。
- environment prompt 仍进入 system prompt。
- agent work role prompt 仍进入 system prompt。
- PromptManifest 可显示 agent/environment/soul prompt refs。

### 禁止

- Environment 不装配 skills。
- Soul 不改变工具、任务、权限。
- Agent profile 不内联大段 prompt。

## Phase 4：统一 task / graph_node / specific task prompt contract

### 目标

图任务、特定任务、单节点合同使用同一 prompt contract。

### 文件

- `backend/harness/graph/context_materializer.py`
- `backend/task_system/compiler/graph_harness_config_publisher.py`
- `backend/task_system/engagement/*`
- `backend/task_system/tasks/*`
- `storage/prompt_library/prompt_resources.json`
- `storage/tasks/graph_harness_configs.json`

### 具体改动

1. 标准 prompt contract：

```text
role_prompt
task_instruction
output_instruction
forbidden_behavior
definition_of_done
```

2. 旧 stage_role 迁移：

```text
stage_role -> graph_node.role
```

3. 单特定任务使用同一结构：

```text
specific_task.prompt_contract -> task prompt sections
```

4. PromptAssemblyService 按 contract fields 生成 sections，不通过普通 turn task_goal_type 猜测。

### 验收

- Graph node prompt 能进入 TaskRunContract。
- Specific task prompt 能单独启动并进入 TaskRunContract。
- 单节点和图节点 prompt 数据结构一致。
- 没有“这是 runtime 节点”类开发说明进入 agent prompt。

### 禁止

- 不从 graph title 猜 task_goal_type。
- 不用 task_goal_type 选择普通 turn prompt。
- 不把 prompt resource migration 当运行时 fallback。

## Phase 5：替换旧 prompting 展示和 token 估算

### 目标

旧 static prompt 不再被当作 session 真实 system prompt。

### 文件

- `backend/query/runtime.py`
- `backend/api/sessions.py`
- `backend/api/tokens.py`
- `backend/runtime/prompt_accounting/*`
- `backend/harness/runtime/session_timeline.py`

### 具体改动

1. `build_system_prompt_for_session()` 改为读取最近一次 RuntimePromptManifest。
2. `/sessions/{id}/messages` 返回：

```text
messages
latest_prompt_manifest_summary
```

不再返回旧 static `system_prompt` 作为真实 prompt。

3. `/tokens/session/{id}` 使用 prompt accounting ledger。

4. 旧 `backend/prompting/*` 降级为迁移源或删除。

### 验收

- 前端 session prompt 展示与真实 runtime packet 一致。
- token 估算来自实际 packet / ledger。
- 新主链不 import `prompting.build_static_prompt`。

### 禁止

- 不保留旧 static prompt 作为 fallback。
- 不让旧 API 输出误导用户以为那是真实 harness prompt。

## 6. Cutover 规则

### 6.1 Shadow 阶段

新 PromptAssemblyResult 与旧 RuntimeCompiler prompt 并行生成，但只有旧路径发给模型。

验收：

- manifest 与旧 packet 内容对齐。
- token 估算正确。
- prompt refs 可追踪。

### 6.2 Cutover 阶段

RuntimeCompiler 改用 PromptAssemblyResult。

验收：

- CLI 普通对话通过。
- CLI TaskRun 通过。
- 图节点 TaskRun 通过。
- session timeline 仍显示状态。

### 6.3 Cleanup 阶段

删除旧 fallback。

验收：

- 新主链没有旧 `prompting` import。
- prompt_library selector 不做旧 task_goal_type 路由。
- 旧测试已清理。

## 7. 验证矩阵

| 场景 | 验证 |
|---|---|
| 普通闲聊 | 不启动 TaskRun，prompt manifest 存在 |
| 简单只读工具 | tool_call 后 observation followup 正常 |
| 真实交付任务 | agent 请求 TaskRun，TaskRun prompt pack 生效 |
| TaskRun 完成 | artifact evidence / verification 进入 volatile state |
| role mode | soul prompt 装配，TaskRun 禁止 |
| professional mode | TaskRun、todo、自审、证据要求来自 runtime projection |
| development environment | environment prompt 进入 packet，工具可见性由 agent 权限决定 |
| graph node | graph_node prompt contract 进入 TaskRunContract |
| specific task | 与 graph node 使用同一 prompt contract |
| token API | token 来源与实际 packet / ledger 一致 |

## 8. 风险与控制

### 风险 1：Prompt 文案迁移导致 agent 行为退化

控制：

- 每个 runtime pack 先保持原文等价迁移。
- 通过真实 CLI 跑普通 turn、TaskRun、observation followup。

### 风险 2：旧 prompt_library 被误当新权威

控制：

- 新资源必须有 category/subtype/status。
- deprecated 资源不进入新主链。
- selector 禁止 task_goal_type 普通 turn 匹配。

### 风险 3：Prompt cache 边界错乱

控制：

- Manifest 记录 cache_scope。
- tests 验证 stable before volatile。
- observations/tool results 永远 volatile。

### 风险 4：Graph prompt 迁移丢失

控制：

- 迁移脚本生成 source_ref。
- graph node prompt contract 前后对比。
- publishing 后 TaskRunContract.prompt_contract 必须非空。

## 9. 完成定义

Prompts 系统重构完成必须满足：

- Prompts Library 按 runtime/agent/environment/task/graph_node/skill/soul 分类。
- RuntimeCompiler 不再硬编码大段稳定 prompt。
- 每次 RuntimeInvocationPacket 都有 RuntimePromptManifest。
- mode 不作为 prompt 文案来源。
- runtime projection 不进入 Prompts Library。
- environment prompt 不授予工具。
- agent profile 不内联主工作 prompt。
- graph node 和 specific task 使用统一 prompt contract。
- 新主链不再依赖 `backend/prompting/*`。
- 旧 prompt selector 不再通过 task_goal_type 影响普通 turn。
