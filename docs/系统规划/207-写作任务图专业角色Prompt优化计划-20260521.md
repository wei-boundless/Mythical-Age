# 写作任务图专业角色 Prompt 优化计划

## 目标

将模块化长篇写作任务图中的节点 prompt 从简短职责说明升级为面向 agent 的专业角色指令。重点解决世界观链路过于粗略的问题，让世界观设计、审核、提交能够真实支撑中文网络小说商业化长篇创作，并让后续人设、剧情、细纲、章节生产节点按照头部商业网文的共性工艺标准承接。

## 约束

- prompt 必须写给执行 agent，使用专业身份、职责边界、工作方法和交付标准，不写成开发说明或运行时字段解释。
- 专业标准必须对标头部中文商业网文和名家级作品的共性能力，包括题材卖点、叙事节奏、场景张力、人物欲望、爽点兑现、情绪回报、长线悬念、章末牵引和责编式质量判断。
- 允许 prompt 引导 agent 学习名家级作品的共性工艺和商业判断，但不能要求复刻任何具体作者的可识别文风、句式、口癖、人物模板、桥段模板或专属设定。
- 世界观节点必须明确如何构建中文网络小说商业化世界设定，但只能给方向维度，不能预设具体题材资产。核心维度包括世界卖点、空间场域、历史秩序、群体关系、交换体系、资源产出、成长体系、原创机制、禁忌代价、升级路径、爽点来源和长期追读钩子。
- 所有具体设定必须来自用户硬设定、项目题材依据或已建立世界逻辑；没有依据时，不主动加入题材专属元素、套路资产或类型预设。
- 写手节点必须给出正向文风目标：名家级中文商业网文长篇写手式正文，语言自然、有现场感和节奏弹性，场景化推进、人物欲望驱动、动作与对话承载信息、情绪价值明确、章末有追读牵引。
- 审核节点只做裁决和返修定位，不替设计节点补写设定。
- 提交节点只固化审核通过内容，不把候选分歧、推测或未审设定写入基准库。
- 同步修改配置源脚本和已发布运行配置，避免下次重新生成任务图时回退。

## 实施步骤

1. 在 `scripts/configure_writing_modular_novel_graph.py` 中重写模块化写作任务图的专业节点 prompt。
2. 重点增强 `world_design`、`world_review`、`memory_commit_world` 三个世界观链路节点。
3. 同步增强设计初始化链路中的项目启动、人设、剧情、对齐、细纲、审核、基准库初始化 prompt，保证后续节点能承接商业化世界观输出。
4. 补强章节循环和收尾交付节点的专业角色边界，明确头部商业网文节奏、章末钩子、情绪价值、自然文风、爽点兑现、卷级悬念和审核标准，避免正文、审核、记忆提交互相越权。
5. 运行配置脚本，重新生成 `storage/tasks` 运行资产。
6. 增加回归断言，确认工作流 prompt 和任务图节点 `role_prompt` 都包含新的专业世界观构建要求。
7. 运行模块化写作任务图相关回归测试，并检查实际任务图执行包仍可编译通过。

## 验收标准

- `workflow.writing.modular_novel.node.world_design` 的 prompt 明确要求构建商业网文世界设定 Bible，而不是只罗列设定项。
- `workflow.writing.modular_novel.node.world_design` 的 prompt 使用题材中立的方向维度，不使用具体类型资产举例。
- `workflow.writing.modular_novel.node.world_design` 的 prompt 明确使用名家级中文商业网文世界架构师身份，并包含不能复刻具体作者可识别风格或专属设定的边界。
- `workflow.writing.modular_novel.node.world_review` 的 prompt 明确审核维度、裁决类型和返修范围。
- `workflow.writing.modular_novel.node.chapter_draft` 的 prompt 明确要求名家级中文商业网文长篇写手身份、自然中文小说正文、强场景推进、爽点兑现和商业化追读节奏。
- `workflow.writing.modular_novel.node.chapter_review`、`volume_review`、`final_review` 的 prompt 明确使用头部连载或头部商业长篇标准做裁决。
- `graph.writing.modular_novel.design_init` 中 `world_design` 节点 metadata 的 `role_prompt` 与 workflow prompt 同步。
- 模块化写作任务图回归测试通过。
