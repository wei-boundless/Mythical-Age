# 灵魂背景立绘生成规格

日期：2026-05-07

## 目标

为前端灵魂切换机制生成一组适合当作工作台背景的人物立绘。

这些图片不是头像，也不是完整海报，而是服务于产品壳层的背景资产：

- 切换灵魂时，背景人物、主题色、光感同步变化。
- 人物存在感明确，但不能抢任务系统、协调拓扑、会话输入区。
- 画面需要保留足够暗部和留白，方便 UI 面板叠加。

## 输出文件

保存到：

```text
frontend/public/souls/backgrounds/
```

文件名固定为：

```text
goumang-bg.png
hebo-bg.png
siyue-bg.png
zhurong-bg.png
xuannv-bg.png
```

## 统一规格

- 尺寸：`3840x2160`
- 比例：16:9 横图
- 风格：半写实东方神话概念立绘，现代高端产品背景图质感
- 构图：人物位于画面右侧 60% 到 88% 区域，左侧和中央保留暗色留白
- UI 安全区：画面左侧 55% 不要出现高对比细节、文字、脸部、手部、兵器尖端
- 背景：深色、低噪声、可叠加玻璃态/深色工作台面板
- 人物：全身或七分身，姿态稳定，边缘清晰但不过分锐利
- 光感：单一主光 + 弱轮廓光，避免大面积过曝
- 禁止：文字、水印、logo、现代现实服装、赛博霓虹、夸张游戏 UI、过度复杂背景、正中大脸

Composition/framing: 16:9 wide composition, character on the right 60%-88% of the image, left 55% kept as dark negative space for interface panels, no important details in the left and center UI safe area.

## 使用说明

生成后直接替换同名文件：

```text
frontend/public/souls/backgrounds/<key>-bg.png
```

前端已经按 `activeSoul.backgroundPath` 读取背景图，灵魂切换时会自动改变：

- 背景人物图
- 背景位置
- 背景透明度
- 主题色与光感

如果某张背景临时缺失，可以先用 `frontend/public/souls/<key>.png` 复制为 `<key>-bg.png` 作为占位。
