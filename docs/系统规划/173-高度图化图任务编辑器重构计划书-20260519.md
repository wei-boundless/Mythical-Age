# 高度图化图任务编辑器重构计划书

日期：2026-05-19
状态：实施中
适用范围：TaskGraph Studio、任务图编辑器、可组合任务图、节点/边/图节点配置、写作任务模板后续落地

## 0. 核心结论

这次重构的核心不是再增加一个“模块页”，而是把编辑器的对象心智统一成一套自然命名：

```text
Node：节点，是任务图中的时序点 / 执行位。
Task：任务，是节点被触发后产生的执行动作 / 工作单。
TaskGraph：任务图，是节点和边组成的流程结构 / 编排蓝图。
GraphUnit：图节点，是父任务图中引用另一张任务图的节点。
Edge：边，是节点之间的交接结构 / 交接协议。
```

因此编辑器里不再把“节点配置、任务绑定、Agent、契约、时序、记忆交接”拆成彼此割裂的页面。用户选中一个节点、图节点或边后，右侧对象台必须直接出现这个对象的可配置内容；运行时再根据这些配置生成任务动作和任务流程。

## 1. 目标对象模型

### 1.1 GraphObject

编辑器中所有可选对象都先映射成 GraphObject。GraphObject 不等于 Task，它是任务图中的结构对象，是生成任务动作的依据：

| 对象 | 编辑器语义 | 保存源 |
| --- | --- | --- |
| 当前任务图 | 流程结构 / 可组合编排蓝图 | `TaskGraphDraftV2` |
| 普通节点 Unit | 时序点 / 执行位 | `nodes[]` |
| GraphUnit | 指向子任务图的图节点 | `metadata.timeline_blocks[]` + 标准视图 Unit |
| PortEdge | 交接结构 / 交接协议 | `edges[]` 或 `metadata.composable_graph.port_edges[]` |
| TimelineBlock | 阶段拼接点 / 图节点来源 | `metadata.timeline_blocks[]` |
| NestedRuntime | 子图任务流程的运行边界 | 图块 + 标准视图派生 |

### 1.2 节点、任务、任务图的关系

```text
Node = Temporal Point / Execution Slot
Task = Execution Action / Work Order
TaskGraph = Process Blueprint
GraphUnit = TaskGraph Invocation as a Node
Nested Task Flow = Runtime activation of a GraphUnit
```

这套关系的关键是：同构发生在“可组合任务图结构”这一层，而不是把节点和任务直接等同。

- 节点是任务图中的时序点，负责声明这个位置需要什么输入、谁可以执行、输出什么、何时允许继续。
- 任务是执行动作，只有运行时激活节点后才产生。
- 任务图是一组时序点和交接协议组成的流程蓝图。
- 图节点是任务图在父任务图中的节点化引用。
- 运行到图节点时，系统才启动被引用任务图对应的子任务流程。

### 1.3 边的职责

边不是执行端，也不是任务。边是节点之间的交接协议，负责：

- 上游时序点的输出什么时候触发下游；
- 交接载荷使用哪个契约；
- 传正文、摘要、引用还是通知；
- 是否需要确认；
- 失败是否传播；
- 哪些工作记忆可以跨边；
- 交接在时序层如何生效。

运行时执行到边时，边不会自己产生任务动作；它只决定上游动作的结果如何变成下游动作的合法输入。

## 2. 工作台布局

重构后的主工作台仍是三栏，但每一栏职责重新定义：

```text
左侧：图层关系 / 对象树 / 当前层级
中间：任务图画布 / 图化流程结构
右侧：选中对象配置台
底部：预检 / 标准视图 / 运行诊断
```

关键约束：

1. 当前层级只显示当前图内的 Unit 和 PortEdge。
2. GraphUnit 作为父图节点显示，不能在父图画布里混排子图内部节点。
3. 点击 GraphUnit 的“进入子图工作台”才切换到子图编辑上下文。
4. 右侧对象台是主配置入口，高级分层页面只作为深层编辑和全局浏览。

## 3. 右侧对象台必须覆盖的配置

### 3.1 当前图

- 图中文名 / 标题；
- 图入口和出口；
- 图级契约；
- 协作模式、协调者 Agent；
- 新增阶段图块；
- 新增显式端口边。

### 3.2 普通节点

- 中文名 / 标题；
- 节点类型；
- 绑定已有任务；
- Agent；
- Projection；
- 节点契约、输入契约、输出契约；
- 阶段、顺序、执行模式、等待策略、汇合策略；
- 主链、阻塞阶段出口；
- 产物目标。

节点表单必须优先使用下拉选择已有任务、Agent、Projection、契约，而不是逼用户手写 ID。

### 3.3 GraphUnit

- 图块中文名；
- `linked_graph_id`；
- 版本锚点；
- 交接契约；
- 可见性；
- 断开策略；
- 进入子图工作台。

GraphUnit 只编辑父图看到的接口和运行边界，不编辑子图内部节点。

### 3.4 PortEdge

PortEdge 分两类：

1. 旧 `edges[]` 派生边：直接编辑原始交接字段。
2. 显式覆盖边：编辑 `metadata.composable_graph.port_edges[]`，支持连接任意 Unit 端口。

旧边必须可配置：

- 载荷契约；
- 边类型；
- 等待策略；
- 确认策略；
- 失败传播；
- 结果投递；
- 模型可见标签；
- Prompt 使用说明；
- 触发、可见、确认、传播、阶段时序；
- 工作记忆携带策略。

并提供“升级为显式端口边”，让用户把旧节点边转为真正的 Unit/Port 连接。

## 4. 数据写入原则

1. `nodes[] / edges[] / metadata.timeline_blocks[]` 仍是真相源。
2. `metadata.composable_graph` 只作为显式覆盖层，不另起第二真相源。
3. 普通节点配置写 `updateTaskGraphNode`。
4. 旧边配置写 `updateTaskGraphEdge`。
5. 显式端口边配置写 `updateTaskGraphMetadata`。
6. GraphUnit 的父图边界配置写 `metadata.timeline_blocks[]`。
7. 子图内部配置只能进入子图工作台后修改。

## 6. 验证标准

- 选中普通节点后，右侧能直接改任务、Agent、Projection、契约、时序。
- 选中 GraphUnit 后，右侧能直接改 linked graph、版本、交接契约，并能进入子图。
- 选中旧边后，右侧能直接改交接契约、等待/确认/失败/投递/时序/记忆交接。
- 选中显式端口边后，右侧能改 Unit 端口连接。
- 修改后调用现有 `updateTaskGraphNode / updateTaskGraphEdge / updateTaskGraphMetadata`，不做伪状态。
- 测试和构建真实通过。

## 7. 第二轮优化原则

第一轮已经让右侧对象台可以真实写回，但任务图编辑器还必须进一步强化“图化选择”：

1. 左侧不只是摘要导航，而是任务图对象树。
   - 当前任务图；
   - 节点树；
   - 交接边；
   - 图节点来源；
   - 子图入口；
   - 诊断定位。

2. 中间画布不只是展示结构，也要能切换语义分面。
   - 节点；
   - 边；
   - 嵌套运行；
   - 图节点来源。

3. 右侧对象台仍是唯一常用配置入口。
   - 左侧负责选对象；
   - 中间负责看结构和点选；
   - 右侧负责配置协议；
   - 底部负责预检闭环。

这一步的目标是让用户不需要记住“去哪个旧页面改哪个配置”，而是自然地从任务图对象出发：点节点、点边、点图节点，然后直接配置。
