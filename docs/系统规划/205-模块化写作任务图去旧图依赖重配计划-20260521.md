# 模块化写作任务图去旧图依赖重配计划

## 目标

把模块化写作任务图从旧 `graph.writing.simple_novel` 的复制改名模式中彻底拆出来。新配置必须以三张子图和一张总图为原生资产：设计初始化图、章节批次创作图、收尾交付图、模块化总图。

## 约束

- 不再读取旧图作为 source graph。
- 删除旧 `graph.writing.simple_novel` 的任务图注册，避免前端继续把旧图作为可选写作任务图。
- 新图内部不得出现 `writing_simple_novel` 记忆 scope、旧图 source 标记、旧图 task/flow/workflow 依赖。
- 章节要求来自通用契约和运行时循环输入，不开写作专用后端私门。
- 节点提示必须是面向 agent 的职责提示，不能把运行时字段说明伪装成 prompt。

## 实施步骤

1. 重写配置脚本入口，移除 `SOURCE_GRAPH_ID` 与旧图读取。
2. 新增原生节点规格表，显式定义每个节点的角色、输入契约、输出契约、投影、记忆读取、产物目标、可读交接包。
3. 新增原生边规格表，显式定义业务交接、记忆读取、记忆提交、返修边和图单元交接边。
4. 用通用 `contract_bindings` 统一承载 schema、memory、artifact、handoff、runtime、acceptance、governance。
5. 将章节图配置为一卷二十万字、十章一批、五卷百万字，并通过 `runtime_loop_policy` 管控批次和分卷循环。
6. 在配置脚本内删除旧图注册项，只保留新模块化图和必要的通用注册。
7. 运行配置脚本，重新生成任务图资产。
8. 做静态核对：确认旧图被移除、新图不再引用旧图、节点和边都存在、章节循环参数正确。

## 验收标准

- `graph.writing.simple_novel` 不再存在于 `storage/tasks/task_graphs.json`。
- 新图四个 ID 全部存在并发布：`master`、`design_init`、`chapter_cycle`、`finalize`。
- 新图配置中不再出现 `SOURCE_GRAPH_ID`、`source_graph_id: graph.writing.simple_novel`、`writing_simple_novel` scope。
- 章节图配置为 `target_volumes=5`、`chapters_per_volume=100`、`chapters_per_round=10`、`volume_target_words=200000`。
- 创作节点的可读上下文由 `artifact_context_policy.items` 显式声明，边的 `target_input_key` 与节点可读项对齐。
