# 80-PDF页级抽取主数据与页状态语义修复计划-20260514

## 1. 问题背景

当前 PDF 页级抽取问题，不是单纯的摘要能力问题，而是页级主数据、页状态语义、上层消费边界三层结构错位：

1. 转换/抽取结果没有稳定保留每一个物理页的页级状态。
2. 页级 runtime 把多种不同失败形态压扁成少数退化原因。
3. 主 Agent 在消费页级退化结果时，会替底层补做页型推断。

这会导致系统虽然“能回答”，但答案边界不够严格，尤其容易把：

- 过渡页
- 目录页
- 结构化结果缺失页
- 文本损坏页
- 纯图片/OCR失败页

混成同一种“没有稳定可提取的正文”。

## 2. 核心问题

### 2.1 页级主数据不完整

当前转换结果主要保存 blocks，但没有把每一个物理页作为一等对象稳定保存。结果是某些页即使在 PDF 中存在，只要没有可靠 block，就会在页级运行时表现得像“页不存在于抽取世界中”。

### 2.2 页级退化语义过粗

当前 runtime 主要使用：

- `target_page_has_no_stable_text`
- `target_page_text_quality_low`

它们不足以表达：

- 这是标题/过渡页
- 这是目录页
- 这是结构化结果缺失页
- 这是文本损坏页

### 2.3 上层消费边界过宽

主 Agent 会根据“只能识别标题”“没有稳定正文”继续推成“这是过渡页”。这种推断不是完全错误，但不应由主 Agent 自由补完，而应由页级能力先给出结构化页状态。

## 3. 修复理论依据

### 3.1 主数据先于解释

如果页级主数据本身不完整，后续任何摘要、裁决、重写都只能建立在不稳定基础上。应先修正页对象和页状态，再讨论表达。

### 3.2 状态语义必须与失败来源同构

结构缺失、OCR失败、目录页、过渡页，本质上不是同一种失败。只有把失败来源拆开，系统的后续判断、测试和用户解释才会稳定。

### 3.3 判别权应靠近证据层

“这页是什么类型”应优先在 PDF 页级 runtime 内做结构化判定，而不是交给主 Agent 根据少量残余文本自由补全。

### 3.4 页级测试应验证状态，不只验证自然语言

如果测试只断言回答里有没有“目录”“过渡”“没有正文”，系统仍然可能靠上层话术过关。页级回归必须验证结构化页状态和退化原因。

## 4. 修复目标

本次修复要达成以下目标：

1. PDF 页级主数据能够覆盖每一个物理页。
2. 页级 runtime 能区分主要页状态与退化来源。
3. `pdf_worker` 与输出边界层按结构化页状态生成用户可读解释。
4. 主 Agent 不再替底层自由猜页型。
5. 回归测试验证页状态，而不是只验证措辞。

### 5.2 PDF 分析 / runtime 层

- `backend/capability_system/units/mcp/local/pdf/agent/models.py`

- `backend/evidence/pdf_worker.py`
- `backend/output_boundary/classifier.py`

## 6. 设计方案

### 6.1 将页对象提升为一等结构

为 PDF 转换结果增加页级主数据，至少包含：

- `page_number`
- `text_block_count`
- `table_block_count`
- `image_block_count`
- `diagnostic_block_count`
- `has_text`
- `has_usable_text`
- `page_state`
- `state_confidence`

这里的目标不是直接做最终结论，而是保留页级事实。

### 6.2 建立页状态枚举

首批页状态至少包括：

- `body_content`
- `transition_title_only`
- `toc_like`
- `page_structure_missing`
- `text_corrupted`
- `image_or_scan_without_text`
- `reference_like`
- `cover_or_copyright`

### 6.3 runtime 不再把缺页与无正文混说

页级请求处理时：

- 页存在但无结构内容：`page_structure_missing`
- 页存在且只剩标题：`transition_title_only`
- 页存在且像目录：`toc_like`
- 页存在但文本损坏：`text_corrupted`

只有真正无法稳定抽取正文时，才继续落到更泛化的低质量状态。

### 6.4 上层只复述页状态，不替底层补页型

`pdf_worker` 和输出边界层应优先根据 `page_state` 生成人类解释，不再把“无稳定正文”直接推成“过渡页”。

## 7. 实施步骤

### Step 1. 补齐页级主数据结构

- 为转换结果增加页级记录模型
- 写入 conversion cache
- 读取 conversion cache 时兼容旧结构，但新结构成为标准输出

### Step 2. 建立页状态判定

- parser / runtime 生成页状态
- 为缺页、低质量、标题页、目录页、损坏页分别建模

### Step 3. 改造页级回答链路

- `pdf_worker` 映射新的页状态
- `output_boundary` 识别新的退化原因
- 调整回答文案，明确“事实层”和“判定层”

### Step 4. 补齐回归测试

- 测试标题页 / 过渡页
- 测试结构缺失页
- 测试目录页
- 测试文本损坏页
- 测试转换结果页主数据持久化

## 8. 风险与控制

### 风险 1：旧缓存结构仍在

控制方式：

- 读取层兼容旧缓存
- 写入层统一输出新结构
- 新测试覆盖新结构

### 风险 2：过度放宽页级门槛

控制方式：

- 不通过降低阈值解决结构缺失问题
- 优先区分状态，再决定是否可摘要

### 风险 3：回答措辞变得更保守

控制方式：

- 允许上层基于结构化页状态直接给结论
- 但结论必须与页状态一致，不能越权猜测

## 9. 完成标准

满足以下条件才算完成：

1. PDF 转换结果中可稳定看到页级主数据。
2. 页级 runtime 能输出结构化页状态。
3. P3 / P4 类场景可明确区分“标题过渡页”和“结构缺失页”。
4. `pdf_worker` 与输出边界层不再混说不同退化来源。
5. 定向回归测试真实通过。
