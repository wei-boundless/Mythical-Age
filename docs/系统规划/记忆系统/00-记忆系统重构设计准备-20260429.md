# 记忆系统重构设计准备

日期：2026-04-29  
定位：本文件用于启动记忆系统重构。目标不是立刻重写所有 memory 代码，而是先把旧记忆能力从 query / prompt / persistence 中解耦，建立 MemorySystem 的正式边界。

---

## 1. 总原则

记忆系统在新 AgentRuntime 中负责：

```text
带什么上下文。
恢复哪些会话状态。
产生什么记忆写回候选。
哪些记忆候选可以进入 prompt。
哪些写回候选等待 CommitGate。
```

记忆系统不负责：

```text
决定当前任务是什么。
决定工具是否可执行。
决定 RuntimeDirective。
直接写 final answer。
绕过 CommitGate 写 durable memory。
覆盖 TaskSystem 的任务事实。
```

一句话：

```text
MemorySystem 提供上下文事实与写回候选，不拥有执行权和最终答案权。
```

---

## 2. 与其他系统的关系

### 2.1 与任务系统

任务系统给出：

```text
TaskContract
TaskFamily
TaskMode
TaskPromptContract refs
```

记忆系统返回：

```text
MemoryContextCandidate
MemoryRestoreCandidate
MemoryWriteCandidate
```

记忆系统不得把“记忆里曾经做过什么”当成当前任务事实。当前任务事实仍以 TaskContract 为准。

### 2.2 与操作系统

操作系统决定资源边界：

```text
read memory index
read session memory
write durable memory
write session memory
```

记忆系统不能因为 memory intent 就直接获得写权限。写入 durable memory 必须成为候选，并等待 CommitGate / MemoryGate。

### 2.3 与灵魂系统

灵魂系统负责模型可见呈现：

```text
SoulRuntimeView
PromptManifest
Prompt section
```

记忆系统只能提供记忆上下文候选和注入建议。最终哪些记忆进入 prompt，需要 ContextPolicy / SoulSystem 收束，不由 MemorySystem 直接拼 system prompt。

### 2.4 与编排系统

编排系统统一采纳候选：

```text
MemoryContextCandidate -> CandidateSet
MemoryRestoreCandidate -> CandidateSet
MemoryWriteCandidate -> CommitCandidate future
```

编排系统不直接读写 memory 文件。

### 2.5 与 CommitGate / OutputBoundary

记忆写回必须走：

```text
MemoryWriteCandidate
  -> CommitCandidate
  -> CommitGate
  -> MemoryCommitRecord
```

最终答案是否提及记忆写入，由 OutputBoundary 决定。

## 3. 旧代码现状

当前相关目录和模块：

```text
backend/memory/
backend/durable_memory/
backend/session-memory/
backend/structured_memory/
backend/query/runtime_context_state.py
backend/query/runtime_persistence.py
backend/api/memory.py

```

当前混合职责：

```text
durable memory 文件治理。

query turn 上下文打包。
persistent memory prompt block。
memory intent 判断。
memory recall preview。
assistant persistence gate。
memory write acknowledgement。

```

主要风险：

```text
记忆读写和 query runtime 耦合。

session memory / durable memory / structured memory 边界不够清晰。
memory intent 可能被误用成执行决策。
```

## 5. 分层职责

### 5.1 MemoryIntentCandidateLayer

输入：

```text
user message
history summary
UnderstandingCandidateLayer output
```

输出：

```text
MemoryIntentCandidate
```

要求：

```text
candidate_only。
不能触发 memory read / write。
不能决定 skip rag。
不能覆盖 TaskContract。
```

### 5.2 MemoryContextBuilder

负责生成：

```text
MemoryContextCandidate
MemoryContextPackage
```

来源：

```text
durable memory recall
session memory state
active bindings
context slots
compressed context
```

要求：

```text
只读。
可追踪来源。
可解释为什么选中。
不直接拼 prompt。
```

### 5.3 MemoryRestoreCandidateBuilder

负责恢复：

```text
active_pdf
active_dataset
active_object_handle_id
active_result_handle_id
active_subset_handle_id
flow_state
task_state
```

要求：

```text
恢复是候选，不是事实。
由编排系统采纳。
与当前 TaskContract 冲突时必须降权。
```

### 5.4 MemoryPolicy

负责判断：

```text
哪些记忆可读。
哪些记忆可注入。
哪些记忆可成为写回候选。
哪些记忆必须等待用户确认。
哪些记忆必须拒绝。
```

要求：

```text
policy 不执行。
policy 不写文件。
policy 输出可审计 decision。
```

### 5.5 MemoryGate

MemoryGate 是 MemorySystem 内部的最终复核门。

它复核：

```text
durable write 是否允许。
session write 是否允许。
memory merge / archive / delete 是否允许。
manual governance 是否允许。
```

但真正落盘仍应通过 CommitGate。

### 5.6 MemoryWriteCandidateBuilder

负责把模型输出、用户明确记忆请求、执行结果、纠错信号变成：

```text
MemoryWriteCandidate
```

要求：

```text
默认 candidate_only。
默认不落盘。
必须有 source evidence。
必须有 reason。
必须可被用户治理。
```

### 5.7 MemoryStoreAdapter

隔离存储实现：

```text
durable_memory/notes
session-memory/{session_id}
storage/indexes_v2/durable_memory
storage/indexes_v2/session_memory
structured_memory
```

外部系统只能通过 adapter 读写，不直接碰文件结构。

### 5.8 MemoryGovernance

负责：

```text
create
disable
activate
archive
delete
merge
audit log
index refresh
```

要求：

```text
治理 API 是人工操作入口。
agent 自动写回不是治理 API。
治理操作也要留下 MemoryCommitRecord / governance_log。
```

## 6. 新链路草案

### 6.1 读上下文链路

```text
TaskContract
  -> MemoryIntentCandidate
  -> MemoryReadRequest
  -> MemoryPolicy
  -> MemoryContextBuilder
  -> MemoryContextCandidate
  -> CandidateSet
  -> OrchestrationSystem adopts / rejects
  -> ContextPolicy / SoulSystem renders prompt section
```

### 6.2 恢复状态链路

```text
session_id
  -> SessionMemoryStore
  -> MemoryRestoreCandidate
  -> CandidateSet
  -> OrchestrationSystem adopts / rejects
```

### 6.3 写回链路

```text
ResultCandidate / user explicit memory request / correction signal
  -> MemoryWriteCandidate
  -> MemoryPolicy
  -> MemoryGateDecision
  -> CommitCandidate
  -> CommitGate
  -> MemoryStoreAdapter
  -> MemoryCommitRecord
```

---

## 7. query 迁移边界

从 query 迁出：

```text
runtime_context_state.py -> MemoryRestoreCandidateBuilder / MemoryContextBuilder
memory intent direct handling -> MemoryIntentCandidateLayer
build_persistent_memory_block -> MemoryContextBuilder + SoulSystem render
memory write acknowledgement -> OutputBoundary + CommitGate result
assistant persistence memory gate -> CommitGate / OutputBoundary
```

query 暂时保留：

```text
请求入口。
事件流 adapter。
legacy compatibility wrapper。
```

禁止：

```text
query runtime 直接写 durable memory。
query runtime 直接决定 memory injection。
query runtime 直接把 memory intent 当执行路由。
```

## 8. 第一阶段施工计划

Phase M0：冻结现状

```text
列出现有 memory API / facade / store / query 依赖。
补最小回归，确保重构前行为可观察。
```

Phase M1：定义合同

```text
backend/memory_system/contracts.py
MemoryIntentCandidate
MemoryContextCandidate
MemoryRestoreCandidate
MemoryWriteCandidate
MemoryPolicy
MemoryGateDecision
```

Phase M2：候选化

```text
把 understanding.memory_intent 输出接入 MemoryIntentCandidate。
把 RuntimeContextState.load_session_restore_candidates 改为 MemoryRestoreCandidate。
把 durable recall 输出改为 MemoryContextCandidate。
```

Phase M3：prompt 解耦

```text
build_persistent_memory_block 不再直接拼 prompt。
MemoryContextCandidate 交给 SoulSystem / ContextPolicy 渲染。
```

Phase M4：写回封门

```text
memory write ack 不直接写。
MemoryWriteCandidate 进入 CommitGatePreview。
durable write 默认 blocked。
```

Phase M5：治理隔离

```text
api/memory.py 只做 governance adapter。
MemoryGovernance 负责治理动作。
MemoryStoreAdapter 负责落盘。
```

## 9. 验收原则

```text
MemoryIntentCandidate.authority == "candidate_only"
MemoryContextCandidate.preview_only == true
MemoryRestoreCandidate.authority == "candidate_only"
MemoryWriteCandidate.allowed == false by default
durable memory write cannot bypass CommitGate
session memory write cannot bypass CommitGate
prompt memory section must have source refs
memory recall must be explainable
query runtime must not own memory policy
```

---

## 10. 暂不做

```text
不重写全部 durable memory 存储格式。
不删除现有 memory API。
不开放自动 durable memory 写回。
不做多智能体 memory scope。
不做长期记忆自动合并策略。
不把 OutputBoundary 和 CommitGate 混进 MemorySystem。
```
