# 能力系统列表去噪与 Tool 正文名补齐计划书

## 背景

能力系统页面当前在列表卡片里展示了过多治理信息，例如风险、边界、适配器、模块等。用户在列表阶段只需要快速识别并选择能力；风险、来源、契约、暴露策略应放到右侧详情区。

同时，tool 注册层只有内部 `name`，前端被迫展示 `web_search`、`read_file`、`python_repl` 这类工程名，不适合作为正文显示名。

## 目标

1. Tool 注册层补 `display_name`，每个 tool 都有中文正文名。
2. 能力系统 Tool 列表只展示正文名，不再展示摘要噪声。
3. Skill 和 Endpoint 列表同步收敛为“名字优先”，详情留在右侧。
4. Tool 搜索保留内部名、正文名、标签和治理信息，列表不展示但仍可检索。
5. 右侧详情继续展示风险、来源、契约、备注和策略。

## 实施范围

- `backend/capability_system/tool_definitions.py`
  - `ToolDefinition` 增加 `display_name: str`。
  - 所有内置工具补中文正文名。
- `frontend/src/lib/api.ts`
  - Tool 类型增加 `display_name`。
- `frontend/src/components/workspace/views/CapabilitySystemView.tsx`
  - 列表卡片去掉摘要段落。
  - Tool 标题优先使用 `display_name`。
  - 详情页标题展示正文名，内部名作为次级信息。
- `frontend/src/app/globals.css`
  - 收紧能力系统列表卡片尺寸和文本层级。

## 验证

- 后端 `py_compile` 验证 tool 注册文件。
- 前端运行 `npx tsc --noEmit`，若失败需区分是否来自本次改动。
