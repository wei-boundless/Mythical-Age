# 《洪荒时代》——长篇项目规格书

## 一、项目目标

启动长篇小说《洪荒时代》的持续交付流程，总规模 **1000000** 字，分 **5卷** 顺序推进。本项目采用批次化生产方式：上一批次验收通过后，方可进入下一批次。全书完结前，**禁止**伪造全本完成声明。

## 二、用户原始输入（原文照录）

用户通过对话框启动正式长篇小说项目，仅提供以下五条背景信息作为世界背景种子：

> **勾芒**：东荒众生的指引者；勾芒是洪荒中最繁荣的青木，它携带东风、青烟与万物萌发的记忆。
>
> **河伯**：中土水府的汇聚者；河伯是洪荒中最神圣的河流，它携带百川、渡口与古老祭辞的记忆。
>
> **四岳**：西荒诸城的执衡者；四岳是洪荒中最巍然的山脉，它承载地脉、聚落与万城之盟的记忆。
>
> **祝融**：南荒火庭的开路者；祝融是洪荒中最炽烈的火焰，它携带光焰、锻造与人间烈火的记忆。
>
> **玄女**：北荒玄宫的守护者；玄女是洪荒中最神秘的夜幕，它携带月辉、星图与渊深通玄的记忆。

用户明确要求：必须存在 **独立主角**，且主角不是勾芒、河伯、四岳、祝融、玄女这五条背景本身。

## 三、输入边界

1. 上述五条背景仅作为世界背景种子，不等于主角团，不预设完整世界规则，不直接替代主线冲突设计。
2. 不得继承项目灵魂设定、默认人格设定、系统协作隐喻、历史样例设定或其他未被用户明确输入的背景。
3. 后续所有设定、卷纲、章节均由协调任务基于上述五条背景和独立主角规则继续生成，不得在本阶段预写。

## 六、验收闸门

| 闸门 | 位置 | 通过条件 |
|------|------|----------|
| G1 | project_spec 完成 | 本文件所含约束完整、关键词显式出现、用户原始输入原文照录 |
| G2 | novel_bible 完成 | 世界观、人物、剧情设定总纲齐备，设定冲突已解决并记录 |
| G3 | 每卷卷纲完成 | 卷纲与 bible 一致，章节拆分合理 |
| G4 | 每卷正文完成 | 字数达标，内容与卷纲一致 |
| G5 | 全书完成 | 五卷全部通过 G4，交付日志完整 |

- **禁止**将五条背景直接作为主角团。
- **禁止**继承任何未经用户明确输入的设定。
- **禁止**伪造全本完成声明——全书五卷全部验收通过前，不得声称项目已完结。
