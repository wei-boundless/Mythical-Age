## 二、世界背景

### 2.1 世界基本形态

《洪荒时代》的世界是一个处于文明初萌阶段的广袤大陆。大陆按方位分为五域：东荒、中土、西荒、南荒、北荒。五域之间并非截然分隔，而是通过自然地貌的过渡带彼此连接。世界尚未形成统一的政治秩序，各域内部以聚落、城邦、部族为主要社会组织形式。

世界的基本物理规则与真实世界相近，但存在一种被称为"洪荒之力"的底层力量——它并非人格化的神祇，而是弥漫于天地之间的原始生机与元素能量。五域各自的自然环境与这种力量的分布密切相关。

### 2.2 五域与五条背景种子的关系

五条用户提供的背景——勾芒、河伯、四岳、祝融、玄女——是各自所在域的核心存在。它们不是神，不是统治者，也不是主角。它们更接近"域灵"或"域枢"：一种与所在域的自然本质深度绑定的、具有记忆与意志的古老存在。

| 域 | 域枢 | 本质 | 携带的记忆 |
|----|------|------|------------|
| 东荒 | 勾芒 | 青木——繁荣、萌发、东风 | 万物萌发的记忆 |
| 中土 | 河伯 | 河流——汇聚、渡口、祭辞 | 百川与古老祭辞的记忆 |
| 西荒 | 四岳 | 山脉——巍然、地脉、盟约 | 聚落与万城之盟的记忆 |
| 南荒 | 祝融 | 火焰——炽烈、光焰、锻造 | 人间烈火的记忆 |
| 北荒 | 玄女 | 夜幕——神秘、月辉、星图 | 渊深通玄的记忆 |

### 2.3 域枢的行为边界

- 域枢**不直接干预**人间事务。它们的存在方式是"承载"与"记忆"，而非"命令"与"统治"。
- 域枢可以与特定个体产生感应或共鸣，但这种联系是稀有的、非强制性的。
- 域枢之间并非敌对关系，但各自所承载的自然本质之间存在天然的张力（如木与火、水与山），这种张力构成了世界底层的动态平衡。
- 域枢**不是主角**，也不是主角团成员。它们可能作为背景力量、记忆来源或关键时刻的参照系出现，但叙事的中心始终是独立主角。

### 2.4 人类世界的基本状况

- 人类是洪荒世界的原生族群之一，分布在各域，以聚落、城邦、游牧部族等形式生存。
- 各域的人类社会因自然环境差异而发展出不同的生存方式、技艺传统与精神信仰。
- 人类对域枢的认知程度不一：有的域将域枢视为传说或信仰对象，有的域保留着与域枢沟通的古老仪式，有的域已几乎遗忘域枢的存在。
- 五域之间的人类社会存在贸易、迁徙、冲突与联盟，但尚未形成跨域的统一文明。

### 2.5 世界背景的开放边界

以下内容**不在本阶段确定**，留待后续卷纲和章节写作中逐步展开：
- 各域的具体城邦名称、政治制度、历史事件
- 洪荒之力的具体运作机制
- 域枢的起源
- 人类之外的其他智慧族群（如有）
- 世界的创世神话或宇宙论

## 三、独立主角生成原则

### 3.1 核心约束

- 主角必须是**独立于五条背景种子**的原创人物。
- 主角不是勾芒、河伯、四岳、祝融、玄女本身，也不是它们的化身、转世或直接造物。
- 主角是**人类**（或至少以人类身份展开叙事）。

### 3.2 主角生成原则

1. **出身原则**：主角应来自五域之一的具体聚落或城邦，具有可追溯的出身背景。出身域的选择将影响主角的初始视角、技能倾向和文化认知。
2. **关联原则**：主角可能与所在域的域枢存在某种感应或共鸣，但这种关联是**间接的、有限的、需要付出代价的**。主角不能直接调用域枢的力量，不能与域枢自由对话，更不能代表域枢行事。
3. **驱动原则**：主角的行动驱动力来自其个人经历、欲望、责任或创伤，而非域枢的指派或"天命"。主角的旅程是人的旅程。
4. **成长原则**：主角的能力成长应遵循"付出—获得"的基本逻辑。不存在天赋神授或突然觉醒的压倒性力量。
5. **局限原则**：主角的认知受其出身、经历和信息来源的限制。主角对世界的理解是逐步展开的，不应在叙事初期就掌握全局信息。

### 3.3 主角不可为之事

- 不可同时与五个域枢建立直接联系
- 不可在单卷内遍历五域（跨域旅行需有充分叙事理由和代价）
- 不可成为"天选之人"式的唯一救世主
- 不可在叙事初期就拥有超越常人的战斗能力或知识储备

### 3.4 主角具体设定

**主角的具体姓名、性别、年龄、出身域、初始状态、性格特征等，不在本阶段确定。** 这些内容将在人物设定总表（character_bible.md）中，基于上述原则生成。

## 四、主线冲突生成原则

### 4.1 冲突来源

主线冲突应从以下三个层面的交互中产生，而非从五条背景种子中直接导出：

1. **域间张力**：五域的自然本质差异（木—火—水—山—夜）在人类社会的政治、经济、信仰层面产生的摩擦与冲突。
2. **人与域枢的关系变迁**：人类对域枢的认知从"共存"到"遗忘"再到"重新发现"（或"利用"）的过程中产生的矛盾。
3. **主角的个人困境与外部世界的碰撞**：主角的成长轨迹与域间冲突的交汇点。

### 4.2 冲突设计原则

1. **渐进原则**：冲突应随卷序逐步升级。第一卷以局部冲突为主，第五卷才可能触及域级或世界级的对抗。
2. **灰度原则**：冲突各方应有可理解的动机，不存在纯粹的"邪恶势力"。域枢之间的自然张力不应被简化为善恶二元对立。
3. **代价原则**：每一次冲突升级都应有明确的代价——人的死亡、聚落的毁灭、知识的失落、信任的破裂。冲突不是免费的戏剧燃料。
4. **不可逆原则**：重大冲突造成的后果不可轻易逆转。世界在冲突中改变，且这种改变是累积性的。

### 4.3 主线冲突的开放边界

以下内容**不在本阶段确定**：
- 具体反派角色或敌对势力
- 各卷的具体冲突事件
- 冲突的最终解决方案
- 主角在冲突中的具体角色

这些内容将在剧情设定总纲（plot_bible.md）和各卷卷纲中逐步展开。

## 五、时间线

### 5.1 时间线结构

本作采用**叙事现在时**为主的时间结构。故事从主角的某个关键节点开始，向前推进。不使用大规模倒叙作为结构手段。

### 5.2 时间线分层

| 层级 | 内容 | 确定状态 |
|------|------|----------|
| 远古层 | 域枢的存在起点、世界初始状态 | 不在此阶段确定 |
| 记忆层 | 域枢所携带的"记忆"所指涉的过去事件 | 仅作为背景暗示，不展开 |
| 近世层 | 主角出生前数十年至数百年的五域历史 | 留待卷纲阶段按需构建 |
| 叙事层 | 主角的故事时间线（五卷覆盖的时间跨度） | 留待卷纲阶段确定 |

### 5.3 叙事层时间约束

- 五卷覆盖的时间跨度应在主角生命中具有意义的区间内，建议不超过二十年。
- 卷与卷之间允许时间跳跃，但每次跳跃应有明确的叙事理由和后果承接。
- 各域之间的旅行时间应保持合理，不得出现"数日之内横跨大陆"式的压缩。

## 六、伏笔账本

### 6.1 伏笔管理原则

伏笔账本用于追踪跨卷、跨章节的伏笔设置与回收。本阶段仅建立账本结构，不填入具体伏笔内容。

### 6.2 伏笔条目格式

每条伏笔记录应包含以下字段：

| 字段 | 说明 |
|------|------|
| 伏笔ID | 唯一标识，格式：FSH-卷序-序号 |
| 设置位置 | 卷序/章节/场景 |
| 伏笔内容 | 简要描述伏笔的具体内容 |
| 伏笔类型 | 人物伏笔 / 事件伏笔 / 世界观伏笔 / 关系伏笔 |
| 预期回收 | 计划回收的卷序或章节 |
| 实际回收 | 回收时的位置（回收后填写） |
| 状态 | 已设置 / 部分回收 / 已回收 / 废弃 |
| 备注 | 补充说明 |

### 6.3 伏笔约束

- 跨卷伏笔应在设置时即记录，不得事后补录。
- 单卷内伏笔由卷纲自行管理，不强制进入总账本。
- 废弃伏笔需注明废弃原因，不得无声删除。

## 七、风格规则

### 7.1 叙事风格

1. **视角**：以主角为绝对中心视角。不使用全知叙述者跳跃式切换视角。如需展示主角不在场的信息，应通过主角后续获知的渠道（传闻、信件、遗迹、他人转述等）间接呈现。
2. **时态**：统一使用中文叙事现在时（"他走向城门"而非"他走向了城门"），以增强临场感。
3. **距离**：叙述距离保持在主角的感知范围内。主角不知道的事，叙述不提前揭示。主角未察觉的情绪，叙述不代为分析。

### 7.2 语言风格

1. **基调**：庄重但不古奥。避免过度文言化，也避免现代口语的随意感。追求一种"有重量的朴素"。
2. **描写**：以具体感官细节为主，减少抽象形容词堆砌。环境描写应服务于氛围和人物状态，不做纯装饰性铺陈。
3. **对话**：对话应符合说话人的身份、出身域和认知水平。不同域的人物在措辞习惯上可有微妙差异，但不应使用方言拟音。
4. **命名**：人名、地名应具有统一的语感体系，避免中西混杂或风格跳跃。具体命名规则留待人物设定总表和卷纲阶段确定。

### 7.3 节奏规则

1. **章节长度**：每章建议 8000—12000 字，允许 ±20% 浮动。
2. **场景切换**：单章内场景不超过三个主要场景。场景切换应有明确的分隔标记。
3. **信息密度**：动作场景与信息场景交替分布，避免连续三章以上纯动作或纯信息。

### 7.4 暴力与情感描写规则

1. **暴力**：可描写暴力后果，但不做虐杀式细节渲染。战斗描写的重心应放在决策、动作与后果上，而非血腥展示。
2. **情感**：情感通过行动、对话和身体反应呈现，减少内心独白式的直接倾诉。主角的情感状态应让读者通过其行为推断，而非由叙述直接告知。

### 7.5 禁止事项

- 禁止使用现代科技隐喻（如"像程序一样运行"、"系统化"等）
- 禁止角色说出超越其时代认知水平的概念
- 禁止叙述者直接对读者喊话或进行道德评判
- 禁止在正文中使用括号注释或脚注式补充
