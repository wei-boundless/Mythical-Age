# 长篇小说项目规格：《洪荒时代》

## 项目目标

- **项目名称**：洪荒时代
- **项目类型**：长篇小说持续交付
- **交付形式**：分卷长篇，按卷规划、按章交付
- **目标体量**：暂定 1,000,000 字
- **核心原则**：只接收用户在长篇任务对话中明确给出的小说输入，不从项目默认设定中继承背景内容

---

## 输入边界

1. 小说背景、人物、规则、冲突、卷级路线，只能来自用户在长篇任务中的明确输入。
2. 项目中的灵魂系统、人格设定、角色分工、协作关系，只属于项目本身，不属于小说世界观。
3. 未经用户明确输入的设定，不得由启动提示、验收脚本、样例文档或历史产物提前补写。

---

## 运行约束

1. 先由对话框输入启动提示，再由协调任务生成后续设定与正文。
2. 不并行分卷。
3. 上一批次验收通过后，下一批次才能进入正文阶段。
4. 禁止把未生成内容标记为已完成。
