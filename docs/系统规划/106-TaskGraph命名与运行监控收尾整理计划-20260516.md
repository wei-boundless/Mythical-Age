# 106-TaskGraph命名与运行监控收尾整理计划-20260516

## 目标

把前端编辑器与运行监控链路中剩余的 `Coordination*` 历史命名继续压缩到 API 边界以内，避免界面层、状态层、组件文件名继续暴露旧术语。

## 本次范围

1. 将聊天监控面板文件从 `CoordinationRunPanel.tsx` 整理为 `TaskGraphRunPanel.tsx`。
2. 将前端运行监控与续跑相关的直接旧字段读取收口到 `frontend/src/lib/api.ts` 的 helper。
3. 将 `store` 与聊天页中已经完成语义迁移的状态名继续保持一致，不再回流旧命名。
4. 保持后端真实协议字段不改，只做前端边界隔离。

## 不在本次范围

1. 后端接口路径、JSON 字段名的重命名。
2. 非 TaskGraph 编辑器主链之外的大范围运行系统重构。
3. 与本次命名整理无关的 UI 改版。

## 实施步骤

1. 新增或补齐 TaskGraph 运行 helper，覆盖 run 列表、run id、live monitor run 提取。
2. 重命名聊天监控面板文件并更新所有引用。
3. 将运行页、聊天监控页中剩余 `coordination_run_id` 直接访问改为 helper。
4. 执行 `tsc`、相关 vitest、Edge 页面检查，确认聊天页和 TaskGraph 运行页可用。

## 完成标准

1. 前端主链不再直接 import/使用 `CoordinationRunPanel`。
2. 前端编辑器与聊天监控不再直接暴露“协调监控”“CoordinationRun”等旧文案。
3. `frontend` 编译通过，相关测试通过，页面可实测打开。
