# 图任务封装组合拓扑重构计划书

日期：2026-05-30  
状态：待审阅实施计划  
依据：`docs/系统框架/007-图任务系统架构设计书-20260528.md` 重定版

本文只写重构计划，不直接实施。实施前必须确认本计划与新版 `007` 一致。

## 0. 目标结论

本轮重构目标不是继续修补 `graph_module` 旧链路，也不是把 GraphHarness 做成任务图事实源。

目标是按新版设计书建立一条闭合主链：

```text
TaskGraphDefinition
  -> LayeredGraph
  -> StandardView / ComposableGraphView / FlowView
  -> ContractManifest
  -> GraphHarnessConfig
  -> ExecutionPackage
  -> GraphRuntime
  -> GraphLoop
  -> GraphNodeWorkOrder
  -> AgentHarness / Tool / HumanGate / ResourceService
  -> FlowPacket / Receipt / ArtifactRef / MemoryRef
  -> TimelineLedger / Checkpoint / Monitor
```

核心变化：

1. 取消父图/子图语义。
2. 图封装后成为 `EncapsulatedGraphUnit`。
3. 封装图和其它单元通过端口、边契约、packet、scope、receipt 组成组合拓扑。
4. 运行期只消费已发布运行合同，不读取编辑器草稿，不重新解释 graph_module。
5. 节点上下文由边契约生成 FlowPacket，不能默认把调度依赖边当通信边。
6. GraphLoopState 只存状态和 refs，不存完整 work order、完整 result、完整正文。
7. 写作任务图迁移为组合编排图 + 封装图单元，删除 `graph_module_runtime`、`sequential_graph_module_runtime`、`graph_module` executor 等旧语义。

### 1.2 必须切除的旧语义

当前仍存在旧组合语义：

```text
graph_module
graph_module_runtime
graph_module_composition
sequential_graph_module_runtime
graph_module_container
graph_module_run
graph_module_sequence
graph_module_committed_refs_only

```

主要位置：

```text
scripts/configure_writing_modular_novel_graph.py
backend/task_system/compiler/graph_harness_config_publisher.py
backend/task_system/graphs/composable_graph_builder.py
backend/task_system/graphs/composable_graph_models.py
backend/task_system/graphs/task_graph_standard_models.py
storage/tasks/task_graphs.json
storage/tasks/graph_harness_configs.json
storage/tasks/task_communication_protocols.json
```

问题：

```text
这些命名会继续暗示“图模块运行”“父子运行”“导入图运行”等旧心智。
新版设计要求封装图只是组合拓扑中的 graph_ref 单元。
外部组合拓扑只通过端口、边契约、FlowPacket 和 receipt 交接。
```

处理原则：

```text
保留能力，不保留旧概念。
graph_module 语义迁移为 EncapsulatedGraphUnit / graph_ref。
graph_module_runtime 语义迁移为 graph_ref runtime_contract。
graph_module_handoff 语义迁移为 encapsulated_graph_handoff / FlowPacket contract。
```

### 1.3 ContextMaterializer 的关键偏差

当前 `GraphContextMaterializer.inbound_context_for_node()` 通过 `_incoming_dependency_edges()` 读取 `SchedulerView.dependency_edges`，再从 `state.result_index` 取上游结果。

问题：

```text
调度依赖边被误用为通信边。
边的 scheduler_role 和 handoff role 没有分开。
result_delivery_policy 虽然存在，但仍以 result_index 为源构造 payload。
默认策略仍会落到 summary_and_refs。
```

新版设计要求：

```text
SchedulerView 只回答 ready / blocked。
FlowEdge / handoff contract 决定通信内容。
ContextMaterializer 必须读取 handoff edge、memory_read edge、artifact_context edge、file_read edge。
每条入边生成一个 FlowPacket 或 packet ref。
```

### 1.4 GraphLoopState 过厚

当前 `GraphLoopState` 包含：

```text
work_order_index: dict[str, dict]
result_index: dict[str, dict]
result_history: dict[str, tuple[dict]]
```

问题：

```text
work_order_index 容易保存完整 input_package。
result_index 容易保存完整 outputs.final_answer。
checkpoint、event、monitor 会重复携带大 payload。
图任务运行消耗会随节点数和正文长度膨胀。
```

新版设计要求：

```text
GraphLoopState 只保存状态、refs、receipt、checkpoint 指针。
完整 work order、node result、artifact、memory candidate 必须进入独立 store/repository。
Monitor 只读 projection，不复制正文。
```

### 1.5 写作任务配置仍有旧组合字段

`scripts/configure_writing_modular_novel_graph.py` 中仍有：

```text
_graph_module_node()
_master_edge()
graph_module_runtime
graph_module_composition
default_executor = graph_module
allowed_executors = ["graph_module"]
task/graph_module_commit
contract.writing.modular_novel.graph_module_handoff
summary_and_refs_only
```

问题：

```text
写作任务本应是新版任务图能力的实例。
如果写作图继续生成 graph_module 字段，后续实现会被旧概念牵引。
```

目标：

```text
写作组合编排图由 EncapsulatedGraphUnit 节点组成。
设计初始化图、章节循环图、收尾交付图都是可封装图定义。
组合编排图通过 FlowEdge 连接这些封装图公开端口。
交接合同使用 encapsulated_graph_handoff / flow_packet，不再使用 graph_module_handoff。
```

## 2. 目标数据模型

### 2.1 FlowUnit

新增或收敛标准单元视图：

```python
FlowUnit = {
    "unit_id": str,
    "unit_type": "atomic_node" | "graph_ref" | "reservoir" | "human_gate" | "tool_unit" | "monitor_unit" | "merge_gate",
    "title": str,
    "source_ref": dict,
    "interface_id": str,
    "contract_bindings": dict,
    "scope_policy": dict,
    "runtime_policy": dict,
    "lineage": dict,
}
```

映射：

```text
TaskGraphNodeDefinition(node_type=agent_role/tool/review_gate/...) -> atomic_node
TaskGraphNodeDefinition(node_type=encapsulated_graph | graph_ref) -> graph_ref
memory_repository / artifact_repository / issue_ledger -> reservoir
manual_gate / review_gate -> human_gate
runtime_monitor -> monitor_unit
```

### 2.2 EncapsulatedGraphUnit

封装图单元必须是 `FlowUnit.unit_type = graph_ref`。

标准字段：

```python
{
    "unit_type": "graph_ref",
    "ref": {
        "graph_id": "graph.writing.modular_novel.design_init",
        "version_ref": "published"
    },
    "interface_id": "interface.graph.design_init.v1",
    "contract_bindings": {
        "schema": {},
        "handoff": {},
        "runtime": {
            "graph_ref": {
                "linked_graph_id": "...",
                "version_ref": "published",
                "isolation_policy": "isolated_per_encapsulated_graph_run",
                "visibility_policy": "committed_only"
            }
        }
    }
}
```

禁止字段：

```text
graph_module_runtime
graph_module_composition
graph_module_container
graph_module_run
sequential_graph_module_runtime
```

### 2.3 FlowEdge

标准边字段：

```python
{
    "edge_id": str,
    "source_unit_id": str,
    "source_port_id": str,
    "target_unit_id": str,
    "target_port_id": str,
    "edge_type": "handoff" | "memory_read" | "memory_commit" | "artifact_context" | "revision_feedback" | "temporal_gate" | "split" | "merge",
    "scheduler_role": "dependency" | "conditional_dependency" | "context" | "commit" | "event" | "audit" | "none",
    "payload_contract_id": str,
    "contract_bindings": {
        "schema": {},
        "handoff": {},
        "memory": {},
        "artifact": {},
        "temporal": {}
    },
    "result_delivery_policy": "contract_payload_and_refs" | "refs_only" | "notification_only",
    "visibility_policy": dict,
    "failure_policy": dict,
    "lineage": dict
}
```

处理原则：

```text
scheduler_role 决定 ready。
handoff/memory/artifact contract 决定 FlowPacket。
不能再用 dependency_edges 代替 inbound_context_edges。
```

### 2.4 FlowPacket

新增 `FlowPacket` 模型，作为边交接事实。

```python
{
    "packet_id": str,
    "packet_type": str,
    "graph_run_id": str,
    "source_unit_id": str,
    "target_unit_id": str,
    "source_port_id": str,
    "target_port_id": str,
    "edge_id": str,
    "scope_id": str,
    "contract_id": str,
    "a2a_message_type": str,
    "payload_summary": str,
    "payload_refs": list[dict],
    "artifact_refs": list[dict],
    "memory_refs": list[dict],
    "result_refs": list[dict],
    "receipt_refs": list[dict],
    "visibility": dict,
    "status": "candidate" | "accepted" | "committed" | "rejected" | "failed",
    "lineage": dict,
}
```

规则：

```text
FlowPacket 默认只存 refs 和摘要。
正文进入 artifact/file repository。
记忆事实进入 memory repository。
候选不能直接变成 committed。
```

### 2.5 TimelineLedger

新增或收敛语义时序账本：

```python
{
    "event_id": str,
    "graph_run_id": str,
    "clock_seq": int,
    "scope_id": str,
    "event_type": str,
    "subject_ref": dict,
    "packet_refs": list[str],
    "receipt_refs": list[str],
    "checkpoint_ref": str,
    "created_at": float
}
```

事件类型：

```text
scheduler_evaluated
node_dispatch_requested
node_result_received
receipt_accepted
receipt_rejected
artifact_materialized
memory_candidate_written
memory_committed
revision_requested
batch_committed
merge_ready
checkpoint_linked
human_gate_opened
human_gate_resumed
encapsulated_graph_run_started
encapsulated_graph_run_completed
```

## 3. 目标权威链

### 3.1 保存权威

```text
TaskGraphDefinition
```

保存图语言和契约。前端编辑器只能写回它，不保存并列事实源。

### 3.2 编译权威

```text
LayeredGraph
StandardView / ComposableGraphView / FlowView
ContractManifest
GraphHarnessConfig
ExecutionPackage
```

编译器只能归一化、诊断、发布合同，不执行任务。

### 3.3 运行权威

```text
GraphRuntime
GraphLoop
GraphContextMaterializer
WorkOrderExecutor
AgentHarness
Resource services
```

GraphLoop 只推进状态。ContextMaterializer 只装配当前节点可见 packet。WorkOrderExecutor 只转交执行端。

### 3.4 记录权威

```text
FlowPacketStore
TimelineLedgerStore
WorkOrderStore
NodeResultStore
ArtifactRepository
MemoryRepository
CheckpointStore
RunMonitorProjection
```

记录层只保存事实和 refs，不能重新决定任务路径。

### Phase 7：封装图组合运行闭环

目标：封装图作为组合拓扑单元运行，不使用父子图概念。

改动文件：

```text
backend/harness/graph/work_order_executor.py
backend/harness/graph/runtime.py
backend/harness/graph/loop.py
backend/harness/graph/runner.py
backend/harness/graph/models.py
backend/query/runtime.py
```

工作项：

1. `executor_type=graph_ref` 进入封装图运行执行器。
2. 封装图运行启动新的 graph run，但语义是组合单元运行，不叫 child graph。
3. 外部组合拓扑只保存封装图运行 ref。
4. 封装图完成后生成 committed output FlowPacket。

## 6. 实施红线

1. 不新增父图/子图/child graph/parent resume 概念。
2. 不让 graph_module 旧字段进入新发布合同。
3. 不让 GraphLoop 根据节点名、任务族、契约名前缀补写作逻辑。
4. 不让 ContextMaterializer 默认从 dependency edge 复制上游 result。
5. 不让 GraphLoopState 存完整正文、完整 work order、完整 result。
6. 不让 A2A adapter 决定任务图结构。
7. 不让编辑器保存隐藏事实源。
8. 不保留已被新主链替代的旧兼容链路。
