## 4. 权限边界

所有写作团队 Agent runtime profile 只允许：

- `op.model_response`
- `op.memory_read`

统一阻止：

- `op.write_file`
- `op.edit_file`
- `op.shell`
- `op.python_repl`
- `op.memory_write_candidate`
- `op.delegate_to_agent`
- `op.rag_search`
- `op.web_search`

稳定资产写入只能通过 `memory_commit_world`、`memory_commit_outline`、`memory_commit_character`、`memory_commit_chapter` 和 `memory_finalize` 这些资源节点完成。

## 5. 前端验收点

1. 新建 `writing_team_long_novel` 模板时应出现 29 个节点、41 条边。
2. 基础资产层应显示 A/B 双方案节点和对应 judge 节点。
3. 章节阶段应显示 `writer_a_draft -> writer_b_review -> writer_a_revision -> writer_b_final_candidate -> novel_quality_judge`。
4. 偏差修正阶段应显示 world / outline / character 三组 router + repair A/B。
5. 所有稳定入库必须经过 memory resource 节点。
6. 图监控中当前运行节点应能落在上述节点之一，而不是退回主会话自由回答。
