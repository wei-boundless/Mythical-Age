# 写作任务设计文件夹

状态：设计稿，未进入配置

这个文件夹是给写作任务图重构用的结构化设计底稿。它不是配置文件，不直接驱动运行，只负责把节点、记忆库、协议、前端命名和实施顺序先钉死。

## 设计目标

1. 把写作流程改成稳定的主循环，而不是散乱的提示链。
2. 把基准库、动态记忆库、产物索引、问题账本、线程账本分开。
3. 把每个节点的职责、输入、输出、禁区、交接写成可执行细则。
4. 让前端图上的中文名、图层、职责和运行状态使用同一套命名来源。
5. 让后续配置只做落地，不再重新争论结构。

## 文件地图

- `00-主循环与分层.md`
- `01-记忆与协议.md`
- `02-节点细则总表.md`

- `04-实施顺序与验证.md`
- `05-起步阶段-用户输入到创作架构初始化.md`
- `06-图模板与协议复用.md`
- `07-世界观后设计层-人设剧情大纲协同.md`
- `08-三阶段任务图拆分.md`
- `09-三图拼接的分层总图.md`

## 这套设计的硬原则

- 只认 `accepted` / `committed` 结果向下游推进。
- 基准库默认稳定，只有正式审核与提交链路才允许改。
- 动态记忆库承接增量，不反向覆盖基准库。
- 候选稿、审稿意见、返修草稿都不能冒充 canon。
- 中文显示名必须有注册来源，不能靠前端临时拼接。
- 不同层级的页面必须拆开，不混在一页里。
