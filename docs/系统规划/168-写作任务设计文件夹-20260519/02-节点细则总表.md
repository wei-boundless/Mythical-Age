# 节点细则总表

## 1. 项目启动包整理者

- 角色：把用户目标、硬设定、风格要求整理成项目启动包。
- 输入：用户目标、约束、参考信息、待确认问题。
- 输出：`project_brief`。
- 交接：只交给世界观设定写手。
- 禁区：不写正文，不做世界观扩写，不抢分卷职责。

## 2. 世界观设定写手

- 角色：产出世界观候选。
- 输入：项目启动包、硬设定、用户风格要求。
- 输出：`world_candidate`。
- 交接：交给世界观审核员。
- 禁区：不写分卷表，不把大纲冒充世界观。

## 3. 世界观审核员

- 角色：判断世界观候选是否可进入主干。
- 输入：世界观候选、项目启动包。
- 输出：审核裁决、问题清单、通过版世界主干。
- 允许裁决：`pass / pass_with_notes / revise / fail_closed`。
- 禁区：不替作者扩写设定，不直接改基准库。

## 4. 世界观写入/提交管家

- 角色：把通过审核的世界观登记为世界观提交。
- 输入：世界观审核结果、候选引用。
- 输出：`world_commit`、世界观引用、写入回执。
- 交接：把可固化世界主干交给基准库管理员。
- 禁区：不跳过审核直接写库。

## 5. 基准记忆库管理员

- 角色：把项目最稳定的内容固化成基准库。
- 输入：项目启动包、世界观提交、冻结边界。
- 输出：`baseline_memory`。
- 必写内容：世界观主干、大纲主干、人物主干、冻结人物与关系事实、可改动库范围。
- 禁区：不写卷级策略，不把候选版本混入 canon。

## 6. 分卷规划者

- 角色：把基准库拆成卷级作战图。
- 输入：基准库、可改动库、项目目标。
- 输出：`volume_plan`。
- 必写内容：卷目标、章节范围、冲突节奏、卷首钩子、卷尾爆点、下一卷输入清单。
- 禁区：不改写冻结事实，不写正文。

## 7. 章节细纲规划者

- 角色：把当前卷拆成当前批次可直接开写的章纲。
- 输入：卷计划、基准库、动态记忆、上一批摘要、返修要求。
- 输出：`chapter_outline`。
- 必写内容：逐章目标、场景推进、冲突动作、信息释放点、章末钩子、连续性警戒点。
- 禁区：不只给一句梗概，不把整卷计划原样复制。

## 8. 章节作者

- 角色：写出当前批次的章节正文候选。
- 输入：章纲、接续包、线程包、约束包、记忆快照。
- 输出：`chapter_draft`。
- 必写内容：每章正文、承接说明、目标完成说明、人物与冲突推进、商业钩子、公开摘要。
- 禁区：不写下一批内容，不拿说明文字代替正文。

## 9. 章节审核员

- 角色：判断本批章节能否进入记忆。
- 输入：章节候选、章纲、基准库、动态记忆、问题账本。
- 输出：`chapter_review`。
- 必写内容：裁决、逐章连续性检查、设定冻结检查、商业阅读检查、返修要求。
- 禁区：不替作者重写正文。

## 10. 章节提交管家

- 角色：把通过审核的章节登记为单章可追踪提交。
- 输入：章节审核结果、正文引用、摘要引用。
- 输出：`chapter_commit`。
- 硬要求：每章都必须单独登记 `chapter_index / chapter_file_ref / chapter_summary_ref / chapter_facts_delta`。
- 禁区：不只登记一个笼统批次。

## 11. 章节推进审核员

- 角色：决定当前批次写完后继续下一批还是进入卷审。
- 输入：章节提交、字数进度、批次状态。
- 输出：`chapter_progress_decision`。
- 决策范围：继续本卷 / 进入卷审 / 失败关闭。
- 禁区：不补写正文，不绕过卷审。

## 12. 卷级总审核员

- 角色：把整卷当成连续阅读单元来审。
- 输入：本卷所有章节提交、卷计划、记忆快照、问题账本。
- 输出：`volume_review`。
- 必写内容：整卷连续性、基准一致性、人物与冲突弧线、商业阅读体验、卷内修订要求。
- 禁区：不把卷级问题压给单章返工。

## 13. 卷级提交管家

- 角色：把通过卷审的一卷写成卷级事实。
- 输入：卷审结果、章节引用、章节摘要引用。
- 输出：`volume_commit`。
- 禁区：不写基准库，只写卷级产物索引和卷级记忆。

## 14. 卷后复盘员

- 角色：从已成立正文中提炼新事实、新趋势、新风险。
- 输入：卷级提交、已提交章节、线程账本、问题账本。
- 输出：`volume_postmortem`。
- 禁区：不把“打算写”的内容当事实，不改主干。

## 15. 可改动库更新提案员

- 角色：把卷后复盘中的有效增量转成可改动库提案。
- 输入：卷后复盘、证据引用、冻结边界。
- 输出：`world_outline_extension_proposal`。
- 必写内容：世界增量、大纲增量、人物权重增量、证据引用、不得改写的冻结事实。
- 禁区：不改主角身份，不改已冻结关系。

## 16. 可改动库边界审核员

- 角色：审核提案有没有越过基准边界。
- 输入：增量提案、证据、冻结边界。
- 输出：`extension_review`。
- 允许裁决：`pass / pass_with_notes / revise_extension / reject`。
- 禁区：不把边界问题当成文风问题。

## 17. 可改动库提交管家

- 角色：把审核通过的增量写入动态记忆库。
- 输入：增量审核结果、允许写入的 delta。
- 输出：`extension_commit`。
- 交接：给下一卷路由器提供可读引用。
- 禁区：不把拒绝项写进动态库。

## 18. 下一卷推进审核员

- 角色：判断本卷结束后是开下一卷、整编交付，还是停机。
- 输入：卷级提交、可改动库更新、累计字数、目标字数。
- 输出：`next_volume_decision`。
- 决策范围：`next_volume / final_assemble / fail_closed`。
- 禁区：不吞掉未回收线程。

## 19. 交付包整编者

- 角色：把已提交的卷级成果整理成最终可交付包。
- 输入：基准库、卷级提交、增量提交、章节文件、章节摘要、交付要求。
- 输出：`final_manuscript`。
- 禁区：不重新编故事，只做汇总、排序、校验和打包。

## 20. 最终交付审核员

- 角色：检查整本书是否可以收尾。
- 输入：最终稿、基准库、动态记忆、卷级清单、未解决问题。
- 输出：`final_review`。
- 禁区：不把明显缺失伪装成可交付。

## 21. 任务收尾归档管家

- 角色：把任务收口成交付包并归档。
- 输入：最终审核结果、交付清单、归档引用。
- 输出：`delivery_package`。
- 禁区：不再生成新剧情内容。

## 22. 运行监测员

- 角色：只观察运行是否健康，不参与内容生产。
- 输入：运行快照、时序、阻塞、流式输出、监测记录。
- 输出：监测决策包、必要时的用户交互请求。
- 禁区：不写世界观、不写章节、不修改记忆库。

## 23. 失败关闭记录员

- 角色：把无法继续的情况安全记录下来。
- 输入：失败原因、安全状态、最后成功提交引用。
- 输出：`failure_report`。
- 禁区：不继续推进，不伪造成功。
