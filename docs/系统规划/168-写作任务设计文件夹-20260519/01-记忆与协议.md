# 记忆与协议

## 1. 记忆库定义

### 基准库

存放稳定主干、冻结人物、冻结关系、世界底层规则和长期主线。

- 默认只读
- 允许通过正式审核与提交链路更新
- 更新后要形成新版本，不允许悄悄覆盖

### 动态记忆库

存放卷后复盘、下一卷策略、人物权重变化、可吸收的大纲增量和世界增量。

- 可变
- 只接收审核通过的增量
- 不得反向覆盖基准库

### 产物索引库

存放候选稿、审核记录、提交清单、章节引用、卷级引用、交付引用。

- 只做索引和追踪
- 不承担 canon 存储职责

### 问题账本

存放阻塞、缺陷、返修要求、卷级问题、交付阻塞。

### 线程账本

存放伏笔、悬念、关系线、世界规则承诺、回收窗口。

### 中文名注册表

存放节点、角色、阶段、线程、仓库的中文显示名和别名。

## 2. 核心包

### continuity_packet

下一节点的接续包，只能基于已接受结果生成。

必须包含：

- 上一轮已接受结果引用
- 当前状态摘要
- 人物状态增量
- 世界状态增量
- 大纲状态增量
- 未闭合线程
- 禁止变更项

### revision_packet

返修包，必须同时带原稿和审核意见。

### constraint_packet

边界包，必须写清冻结事实、预算、禁区和不可越界范围。

### thread_packet

线程包，用来说明伏笔、悬念、关系推进和回收窗口。

### artifact_context_packet

产物包，只给精确引用，不灌全量正文。

### memory_snapshot

记忆快照，只读取授权仓库和授权版本。

## 3. 交接规则

1. 下游只消费 `accepted` / `committed`。
2. `candidate` 和 `review` 只能用于本轮返修，不得直接外溢。
3. 基准库冲突优先级高于动态库，除非动态库已被正式提交为新版本。
4. 章节批次交接必须包含单章引用，不能只给笼统批次名。
5. 伏笔线程必须有状态：`proposed / active / due / paid / deferred / cancelled / invalidated`。

## 4. 防污染规则

- 不把聊天历史当主事实源。
- 不把候选稿和审核意见混成一包。
- 不把未来批次全文塞进当前节点上下文。
- 不让摘要替代事实。
- 不让未审核内容进入下游记忆。
