# 主循环与分层

## 1. 主循环

写作任务的主循环应固定为：

`project_brief -> world_design -> world_review -> memory_commit_world -> baseline_memory_seed -> volume_plan -> chapter_outline -> chapter_draft -> chapter_review -> memory_commit_chapter -> chapter_progress_router -> volume_review -> volume_commit -> volume_postmortem -> world_outline_extension_proposal -> extension_review -> extension_commit -> next_volume_router -> final_assemble -> final_review -> memory_finalize`

监测节点 `runtime_monitor` 贯穿全程，`fail_closed` 作为终态兜底。

## 2. 分层

### 控制层

负责决定“谁在什么时候做什么”。

- `project_brief`
- `chapter_progress_router`
- `next_volume_router`
- `runtime_monitor`
- `final_review`

### 生产层

负责产出正文、细纲、规划、复盘、交付。

- `world_design`
- `volume_plan`
- `chapter_outline`
- `chapter_draft`
- `volume_postmortem`
- `world_outline_extension_proposal`
- `final_assemble`

### 审核层

负责裁决是否允许进入下一步。

- `world_review`
- `chapter_review`
- `volume_review`
- `extension_review`
- `final_review`

### 记忆层

负责把已确认内容写入正确仓库。

- `memory_commit_world`
- `baseline_memory_seed`
- `memory_commit_chapter`
- `volume_commit`
- `extension_commit`
- `memory_finalize`

## 3. 运行原则

1. 写作是循环推进，不是一次性生成。
2. 每一步只能读取自己被授权的输入包和记忆包。
3. 任何修订都必须带版本锚点和来源引用。
4. 同一轮中，候选、审核、提交必须严格分开。
5. 未来批次内容不得提前泄漏到当前批次输入。
6. 章节批次与卷级结果都必须可以追溯到单章、单卷。

## 4. 层级页面原则

前端展示也按层级拆开：

- 总览层
- 节点层
- 资源层
- 时间线层
- 运行监测层

每一层都是独立页面或独立卡片切换，不把不同层级塞进同一块混合视图里。
