# 前端中文名与图层

## 1. 中文名注册

写作任务图上的中文名必须来自统一注册层，不能只靠组件里临时写 label。

### 注册对象

- `node_id`
- `projection_id`
- `agent_id`
- `role_type`
- `phase_id`
- `thread_id`
- `repository_id`

### 优先级

前端展示按同一套优先级取名：

1. 显式注册的 `display_name_zh`
2. 投影定义中的中文标题
3. 节点标题
4. Agent 名称
5. 角色映射名
6. `node_id` 兜底

## 2. 图上显示规则

- 节点主标题显示中文名。
- 节点 ID 只放次级信息或 tooltip。
- 边和分组也要支持中文显示。
- 没有注册中文名的节点，应该在预检里暴露出来，而不是默默放行。

## 3. 页面分层

不同层级必须拆页面，不混页：

- 拓扑层：看节点和边
- 职责层：看节点细则和契约
- 时间线层：看运行顺序和阶段推进
- 记忆层：看基准库、动态库、账本和产物索引
- 监测层：看运行健康、阻塞和交互请求

## 4. 卡片切换

页面之间用卡片按钮切换，不把多个层级压成同一堆表格。

## 5. 命名一致性

同一个对象在拓扑、职责、时间线、监测、记忆视图里必须使用同一个中文主名，避免一处叫“基准库管理员”，另一处又只写 `baseline_memory_seed`。
