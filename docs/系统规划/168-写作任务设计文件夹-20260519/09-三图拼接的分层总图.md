# 三图拼接的分层总图

状态：设计稿，未进入配置

## 1. 核心判断

三张任务图不应该彼此孤立展示，而应该可以拼接成一张分层总图。

这张总图的作用不是把所有细节塞进一张密集拓扑里，而是把三张正式任务图按阶段拼起来，让人一眼看见：

- 先设计
- 再创作
- 再收尾

## 2. 为什么要拼接

如果只有三张独立图，会出现两个问题：

1. 用户只能看见局部，容易丢掉全局顺序。
2. 每张图单独理解没问题，但阶段之间的衔接感不强。

如果把三张图直接粗暴合并成一张平铺大图，又会出现：

1. 图太大，眼花缭乱。
2. 时序层级不清。
3. 设计、创作、收尾的责任边界混掉。

所以最佳方案是：**三图独立 + 顶层拼接**。

## 3. 拼接方式

### 3.1 顶层总图

顶层只显示三段：

```text
设计阶段图
  -> 正式创作阶段图
  -> 收尾阶段图
```

这条总图只表达阶段顺序，不展开每个阶段内部所有节点。

### 3.2 阶段内展开

每个阶段点击进入自己的子图：

- 设计图展开人设、剧情、大纲、对齐
- 创作图展开章纲、正文、审核、提交、推进
- 收尾图展开卷审、复盘、增量、交付

## 4. 总图的展示层次

### 第一层

只显示阶段名称和阶段边：

- 设计阶段
- 正式创作阶段
- 收尾阶段

### 第二层

进入某个阶段后再展开该阶段内部节点。

### 第三层

进入节点后再看：

- 输入
- 输出
- 协议
- 交接
- 记忆写入

这样可以保证层次越来越细，但不会一开始就把所有节点摊平。

## 5. 前端呈现建议

总图页面建议支持三种切换：

1. 总览模式
2. 阶段展开模式
3. 节点详情模式

其中：

- **总览模式**：只看三图拼接关系
- **阶段展开模式**：看某一张图的内部拓扑
- **节点详情模式**：看节点职责和协议

## 6. 为什么这比单图更好

### 更强的层次感

用户能先看到阶段，再看阶段里的结构，不会一上来被细节淹没。

### 更好的可读性

同一张总图既保留整体顺序，也不会失去阶段边界。

### 更好的可维护性

以后某个阶段内部节点变化，只需要改那一张子图，不需要重画总图。

### 更好的扩展性

如果未来再加阶段，比如“世界观后设计层”或“返修层”，只要挂到总图里作为新阶段即可。

## 7. 结论

你的想法可以直接落成结构原则：

**三张图独立建模，顶层再拼成一张分层总图。**

这样既不会丢层次，也不会让一张图大到失控。
