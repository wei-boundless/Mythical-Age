并行装配规则：

- `chapter_writer_a` 与 `chapter_writer_b` 同属 `phase.chapter_drafting`。
- 两者 `sequence_index = 1`。
- 两者 `execution_mode = parallel`。
- 两者应放入同一候选稿分发组，语义为 `chapter_draft_candidates`。
- `chapter_review` 等待两条 `chapter_draft` 产物边都完成后再运行。

返修装配规则：

- `quality_gate` 输出 `contract.writing_team.long_novel.quality_gate`。
- 当 `verdict = repair_required` 时，走 `edge.quality_gate.chapter_plan`。
- 返修目标是 `chapter_plan`，不是 `world_design`，避免从世界观重新开始。
- `loop_policy.max_attempts = 3`。
- `loop_policy.exit_condition = quality_gate_passed_or_human_stop`。

入库装配规则：

- 只有 `quality_gate.memory_commit_allowed = true` 时，`memory_commit` 才应提交长期资产。
- `memory_commit` 只写 approved world / outline / character / chapter summary / review decision。
- 未通过资产只能保留为运行 trace 或返修依据，不能进入正式项目记忆。

### 17.8 前端逐层填写顺序

在 TaskGraph Studio 中重新装配时，建议按以下顺序填写：

1. 蓝图层：选择 `writing_team_long_novel` 模板，确认 `graph_id`、任务域、入口、出口、图契约。
2. Agent 编组层：逐个节点确认 Agent 可调用，尤其是 `chapter_writer_a`、`chapter_writer_b` 是两个不同实例。
3. 拓扑层：确认 11 个节点和 13 条边，不遗漏 `world_review -> character_design`、`character_design -> chapter_plan`、`quality_gate -> chapter_plan`。
4. 职责与交接层：确认每个节点的 Projection、输入契约、输出契约和 artifact target。
5. 时序层：确认五个 phase，写作者 A/B 同 phase、同 sequence、parallel。
6. 记忆与产物层：确认只传 artifact refs、summary、review decision，不传原始内部对话。
7. 契约与质量门层：确认质量门具备 `verdict`、`repair_targets`、`memory_commit_allowed`。
8. 发布运行层：运行 preflight，编译 runtime spec，再启动第一轮小范围实测。

### 17.9 最小真实运行输入

首轮实测建议使用一个小输入，不直接要求完整长篇：

```text
请为一部长篇小说项目启动写作团队任务图。
题材：近未来城市悬疑。
目标：先完成世界观设计、世界观审核，并准备进入大纲与人物设定。
约束：不要写完整正文，只产出可交接的结构化资产。
```

预期行为：

- `world_design` 产出 `world/world_brief.md`。
- `world_review` 对世界观给出通过、返修或阻塞裁决。
- 若通过，`outline_design` 与 `character_design` 应进入 ready 或可调度状态。
- 若返修，不应进入大纲和人物设定。

### 17.10 完整链路验收输入

第二轮再测试完整链路：

```text
请继续运行长篇小说写作团队任务图。
目标：基于已通过的世界观、大纲和人物设定，为第一章生成章节细纲，两名章节写作者分别产出候选稿，由章节审稿人比较审查，再由质量门决定是否入库。
约束：如果质量门认为不达标，必须回到章节细纲修正，不要直接入库。
```

预期行为：

- `chapter_plan` 汇总世界观、大纲、人物设定，产出第一章细纲。
- `chapter_writer_a` 与 `chapter_writer_b` 并行产出两份候选稿。
- `chapter_review` 比较两份候选稿，给出采用建议和问题列表。
- `quality_gate` 只做裁决，不替代写作者扩写正文。
- `memory_commit` 只在 `memory_commit_allowed = true` 时写入项目资产。
