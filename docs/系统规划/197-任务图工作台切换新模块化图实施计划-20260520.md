# 任务图工作台切换新模块化图实施计划

## 目标

把任务系统前端从“普通任务图列表 + 旧图可编辑”切换到“新模块化任务图优先”的工作台体验。切换不改变通用架构原则：任务图、GraphUnit、契约绑定、批次契约、执行包都必须来自通用模型和后端编译结果，不允许为写作任务单独开运行私门。

## 当前判断

后端已经有模块化写作任务图：

- `graph.writing.modular_novel.master`
- `graph.writing.modular_novel.design_init`
- `graph.writing.modular_novel.chapter_cycle`
- `graph.writing.modular_novel.finalize`

前端已经有三栏工作台、GraphUnit inspector、批次契约 inspector、执行包面板，但入口选择仍按普通图列表顺序工作，用户容易进入旧图或看不到新图的父子运行结构。

## 实施步骤

1. 图选择切换
   - 为任务图列表和编辑器图下拉建立统一排序规则。
   - 对模块化主图、模块化子图、带 GraphUnit/批次契约的图给予优先展示。
   - 打开新模块化写作任务域时优先选择 `graph.writing.modular_novel.master`。

2. 工作台层级关系
   - 左侧图层栏明确显示当前父图、GraphUnit 子图入口和可钻取动作。
   - 选中 GraphUnit 时，右侧编辑台能直接进入绑定的子任务图。
   - 避免把旧字段作为主要配置入口，优先展示 `contract_bindings`、GraphUnit handoff、split policy。

3. 发布执行闭环
   - 发布页执行包继续调用真实后端 `/execution-packages`。
   - 执行包面板突出 GraphUnit 子图计划、split plan、批次生命周期、对象追溯。
   - 用户能从发布页判断“图上配置的子图、契约、批次是否进入真实运行包”。

4. 验证
   - 增加或调整前端单元测试覆盖图排序/优先选择逻辑。
   - 运行相关前端测试和构建。
   - 启动本地前端，用浏览器检查任务系统工作台能切到新模块化图。

## 不做事项

- 不为写作任务增加专用运行门禁或专用质量门。
- 不把旧图字段继续扩展成新的主配置模型。
- 不用前端假数据伪造 GraphUnit 或 split plan。
