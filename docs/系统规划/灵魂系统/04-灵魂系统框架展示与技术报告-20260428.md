## 0. 系统概览

当前灵魂系统是一套围绕“灵魂 seed 文件 + 结构化 SoulProfile + 投影卡片 + prompt 运行时视图”构建的身份与表达层。

它由四个主要部分组成：

```text
SoulRegistry
  读取和管理灵魂档案、seed 文件、active soul、自制灵魂。

ProjectionStore
  读取和管理投影卡片，保存 selected projection。

SoulProjectionBuilder
  根据 soul、task、skill、tool、memory、output 信息生成 SoulRuntimeView。

SoulPromptManifest
  记录运行时 prompt sections 的来源、长度和 hash。
```

当前系统的核心链路是：

```text
灵魂文件 / profile
  -> SoulRegistry
  -> /soul/catalog
  -> 前端展示与切换

投影卡片 catalog
  -> ProjectionStore
  -> /soul/projections
  -> 前端展示与选择

SoulProjectionRequest
  -> SoulProjectionBuilder
  -> SoulRuntimeView
  -> SoulPromptManifest
```

## 3. 核心数据模型

数据模型集中在：

```text
backend/soul/contracts.py
```

### 3.1 SoulProfile

`SoulProfile` 是当前代码里的结构化灵魂档案。

字段包括：

```text
soul_id
name
display_name
source
version
enabled
seed_path
description
background
personality_traits
expression_style
working_habits
preferred_role_types
preferred_task_modes
collaboration_tendencies
memory_preferences
risk_biases
guardrails
portrait
validation_errors
metadata
```

技术特点：

```text
1. 使用 dataclass(slots=True, frozen=True)。
2. 通过 to_dict() 输出普通 dict。
3. metadata 会被显式转成 dict。
4. builtin profiles 由 registry.py 中的 BUILTIN_PROFILES 生成。
5. custom profiles 从 soul/custom/{id}/profile.json 读取。
```

### 3.2 SoulRole

`SoulRole` 描述某个灵魂在任务中的角色信息。

字段：

```text
role_id
soul_id
role_type
agent_profile_id
task_mode
expected_artifact
allowed_skills
visible_tools
handoff_target
```

当前它是契约层对象，主要用于描述关系，还不是主运行链路中的核心存储对象。

### 3.3 SoulSkillView

`SoulSkillView` 是给投影装配使用的 skill 可见摘要。

字段：

```text
skill_id
title
capability_summary
use_when
input_boundary
output_boundary
forbidden_uses
current_task_reason
```

它表达“当前投影可以看见的 skill 信息”，不是 skill 执行器本身。

### 3.4 SoulToolView

`SoulToolView` 是给投影装配使用的 tool 可见摘要。

字段：

```text
tool_id
title
capability_summary
input_schema_summary
output_schema_summary
risk_summary
authorized
authorization_owner
```

`authorized` 会影响 `SoulRuntimeView.visible_tool_ids`，只有 authorized 为 true 的 tool id 会进入 visible tool 列表。

### 3.5 SoulProjectionRunPreview

`SoulProjectionRunPreview` 是投影运行预览对象。

字段包括：

```text
run_id
bundle_id
node_id
projection_id
status
role_control_mode
context_id
context_scope
resource_policy_ref
tool_requirement_refs
candidate_tool_ids
risk_tags
input_artifact_refs
output_artifact_refs
memory_scope
checkpoint_ref
trace_ref
state_policy
```

它在 `__post_init__()` 中做了三类校验：

```text
1. run_id 必须存在。
2. memory_scope 必须保持 none。
3. state_policy 不能包含 legacy task 分类字段、执行指令字段、权限授予字段。
```

### 3.6 SoulProjectionRequest

`SoulProjectionRequest` 是构建投影运行时视图的输入。

字段：

```text
task_id
soul_id
role_type
task_mode
agent_profile_id
projection_name
skill_views
tool_views
projection_run_preview
task_contract_summary
memory_policy_summary
output_contract_summary
```

`SoulProjectionBuilder.build()` 消费的就是这个对象。

### 3.7 PromptSection

`PromptSection` 表示一段即将进入 runtime view 的 prompt section。

字段：

```text
section_id
title
source_type
source_id
owner_layer
cache_scope
visible_to_model
content
```

`to_dict()` 会额外输出：

```text
chars: len(content)
```

### 3.8 SoulRuntimeView

`SoulRuntimeView` 是灵魂投影构建后的运行时视图。

字段：

```text
soul_id
role_type
task_mode
sections
visible_skill_ids
visible_tool_ids
authorization_owner
trace
```

它是 `SoulProjectionBuilder` 的主要产物之一。

### 3.9 SoulPromptManifest

`SoulPromptManifest` 是 prompt manifest。

字段：

```text
manifest_id
task_id
soul_id
projection_id
sections
prompt_hash
```

manifest 记录 section 元信息和整体 prompt hash，不保存额外执行状态。

## 4. 投影系统边界

投影系统在当前架构中的职责是：

```text
决定 agent 以什么运行时姿态承载任务。
```

它处理的是姿态、表达和呈现方式，不处理任务真相和权限事实。

### 4.1 投影系统管理什么

```text
ProjectionCard
  静态投影配置。
  保存 soul_id、role_type、姿态标签、默认表达方式、工作风格预设、注意力偏好。

SoulProjectionRequest
  运行时投影构建请求。
  接收 task、skill、tool、memory、output 的摘要输入。

SoulRuntimeView
  当前任务中真正给模型看的投影视图。

SoulPromptManifest
  记录本次投影视图由哪些 prompt sections 组成。
```

### 4.2 投影系统不管理什么

```text
不管理 TaskDefinition。
不管理 TaskGraph。
不管理 SkillRegistry。
不管理 ToolRegistry。
不管理工具授权。
不管理 memory 写入。
不管理 runtime directive。
```

### 4.3 与任务系统的关系

任务系统给投影系统提供：

```text
TaskRuntimeInstruction
OutputContract
TaskRole
TaskProjectionBinding
```

投影系统返回：

```text
ProjectionRuntimePosture
SoulRuntimeView
PromptManifest
```

### 4.4 与 Skill / Tool / 操作系统的关系

Skill / Tool / 操作系统给投影系统提供的不是权限，而是可见摘要：

```text
SkillRuntimeView
ToolRuntimeView
MemoryPolicy summary
ResourcePolicy summary
```

投影系统可以决定这些摘要以什么语言和姿态呈现给模型，但不能扩大可见范围，也不能改变授权结果。

### 4.5 ProjectionState 预留口

投影系统后续可以接入工作记忆，形成短期成长态：

```text
WorkingMemoryView
  -> ProjectionStateCandidate
  -> SoulProjectionBuilder
  -> ProjectionRuntimePosture
```

`ProjectionState` 只影响当前运行时姿态，不写回 `SoulProfile`，不授予工具权限，不改写任务流程。

## 5. SoulRegistry

实现文件：

```text
backend/soul/registry.py
```

### 5.1 常量配置

核心路径：

```text
ACTIVE_SEED_PATH = "soul/agent_core/ACTIVE_SEED.md"
CORE_PATH = "soul/agent_core/CORE.md"
AGENT_PROFILE_PATH = "soul/agent.md"
SEED_CATALOG_PATH = "soul/agent_core/SEED_CATALOG.md"
```

内置 seed 映射：

```text
BUILTIN_SEED_PATHS
BUILTIN_SOUL_NAMES
BUILTIN_PROFILES
```

自制灵魂禁止字段：

```text
tools
tool_permissions
worker_route
worker_routes
memory_write
memory_write_policy
control_kernel
runtime_directive
override_core
core_override
```

这些字段出现在 custom profile 中时，`validate_custom_profile()` 会返回错误。

### 5.2 文件读写

`read_text(path)` 支持多编码读取：

```text
utf-8
utf-8-sig
gb18030
gbk
utf-8 errors=ignore fallback
```

`write_text(path, content)` 使用 UTF-8 写入，并自动创建父目录。

### 5.3 active soul 判断

`active_soul_id()` 的判断顺序：

```text
1. 读取 ACTIVE_SEED.md。
2. 与 builtin seed 内容逐个比对。
3. 与 enabled custom soul 的 SOUL.md 内容比对。
4. 通过中文名在 ACTIVE_SEED.md 中反查。
5. 默认返回 hebo。
```

### 5.4 profiles

`profiles(include_disabled=False)` 会合并：

```text
builtin_profiles()
custom_profiles()
```

builtin profiles 来自代码内的 `BUILTIN_PROFILES`。

custom profiles 来自：

```text
backend/soul/custom/{soul_id}/profile.json
backend/soul/custom/{soul_id}/SOUL.md
backend/soul/custom/{soul_id}/portrait.png
```

### 5.5 catalog 输出

`build_catalog()` 输出：

```text
active_soul_key
active_soul_id
active_soul_name
injection_chain
static_files
seeds
soul_profiles
management
```

其中 `injection_chain` 当前固定为：

```text
10 当前灵魂契约 ACTIVE_SEED.md
20 通用静态准则 CORE.md
30 长期项目偏好 agent.md
```

`management` 当前返回：

```text
planes: ["management", "projection", "runtime"]
authorization_owner: "ControlKernel / ResourcePolicy"
prompt_manifest_enabled: true
custom_soul_dir: "soul/custom"
```

### 5.6 switch

`switch(soul_id)` 的行为：

```text
1. 查找 SoulProfile。
2. 检查 profile 存在且 enabled。
3. 读取 profile.seed_path。
4. 将 seed 内容写入 ACTIVE_SEED.md。
```

API 层随后会调用：

```text
runtime.refresh_indexes_for_path(ACTIVE_SEED_PATH)
```


```text

根据 request 内容生成 projection_id。

```

```text
检查 projection_id 存在。
写入 selected_projection_id。
```

`delete_projection_card()`：

```text
删除指定 projection card。
如果删除的是 selected projection，则自动选中剩余第一个 card。
```

## 7. SoulProjectionBuilder

实现文件：

```text
backend/soul/projection.py
```

### 7.1 输入

`SoulProjectionBuilder.build(request)` 输入：

```text
SoulProjectionRequest
```

构建前会：

```text
1. 通过 SoulRegistry 获取 profile。
2. 检查 profile 存在且 enabled。
3. 读取 profile.seed_path。
4. 读取 CORE.md。
5. 将 skill_views / tool_views 转成文本摘要。
```

### 7.2 输出

返回 dict：

```text
projection_id
runtime_view
prompt_manifest
profile
```

### 7.3 projection_id

`projection_id` 由 SHA1 截断生成，输入内容包括：

```text
task_id
soul_id
role_type
task_mode
agent_profile_id
projection_name
task_contract_summary
projection_run_preview fingerprint
```

截断长度：

```text
16 hex chars
```

### 7.4 PromptSection 组成

当前 builder 生成 8 个 section：

| section_id | title | source_type | owner_layer |
| --- | --- | --- | --- |
| `identity_view` | 灵魂身份 | `soul_seed` | `soul` |
| `static_common_rules` | 静态共同准则 | `core` | `soul_core` |
| `dynamic_task_contract` | 动态任务契约 | `task_contract` | `task` |
| `role_view` | 当前投影职责 | `task_contract` | `task` |
| `projection_run_view` | 投影子任务运行视图 | `projection_task_run` | `task` |
| `skill_view` | Skills 可见摘要 | `skill_contract` | `skill` |
| `tool_view` | Tools 可见摘要 | `tool_contract` | `resource_policy` |
| `memory_output_view` | 记忆与输出边界 | `memory_policy` | `runtime` |

### 7.5 tool 可见规则

`SoulRuntimeView.visible_tool_ids` 只包含：

```text
tool_view.authorized == True
```

`tool_view` section 文本中会展示每个 tool 的授权状态：

```text
授权：是
授权：否
```

### 7.6 trace

runtime view 的 trace 当前包含：

```text
projection_owner: "SoulProjectionBuilder"
authorization_owner: "ResourcePolicy"
profile_source: profile.source
```

## 8. PromptManifest

实现文件：

```text
backend/soul/prompt_assembly.py
```

核心函数：

```python
build_prompt_manifest(task_id, projection_id, runtime_view)
```

执行逻辑：

```text
1. 将 runtime_view.sections 的 content 用空行连接。
2. 对连接后的 prompt 文本计算 SHA256。
3. 将每个 PromptSection 转成 PromptSectionManifest。
4. 返回 SoulPromptManifest。
```

`PromptSectionManifest` 记录：

```text
section_id
source_type
source_id
owner_layer
cache_scope
visible_to_model
chars
```

## 11. 权限与边界

当前灵魂系统在代码中保留了几类边界：

### 11.1 custom profile 禁止权限字段

`FORBIDDEN_CUSTOM_KEYS` 禁止自制灵魂声明：

```text
tools
tool_permissions
worker_route
worker_routes
memory_write
memory_write_policy
control_kernel
runtime_directive
override_core
core_override
```

### 11.2 ProjectionRunPreview 禁止执行指令

`SoulProjectionRunPreview` 会拒绝：

```text
legacy_* 字段
task_kind / modality 旧分类字段
runtime_directive 等非 disabled 执行指令
permission / authorization / authorized 权限授予字段
```

### 11.3 Tool 授权归属

`SoulToolView.authorization_owner` 默认是：

```text
ResourcePolicy
```

`SoulRuntimeView.authorization_owner` 默认也是：

```text
ResourcePolicy
```

### 11.4 文件编辑路径校验

`SoulRegistry.resolve_editable_path()` 会：

```text
1. normalize path。
2. 检查路径是否属于 editable_paths。
3. resolve 后确认没有 path traversal。
```

## 12. 当前数据流总图

```text
前端请求 /soul/catalog
  -> backend/api/souls.py
  -> SoulRegistry.build_catalog()
  -> 读取 ACTIVE_SEED / CORE / agent.md / SEED_CATALOG / seeds / custom profiles
  -> 返回 catalog
  -> 前端展示 active soul、seed、portrait、static files
```

```text
前端请求 /soul/switch
  -> backend/api/souls.py
  -> SoulRegistry.switch(soul_id)
  -> 写入 ACTIVE_SEED.md
  -> runtime.refresh_indexes_for_path
  -> 返回新 catalog
```

```text
前端请求 /soul/projections
  -> backend/api/souls.py
  -> SoulRegistry.profiles()
  -> projection_store.list_projection_cards()
  -> reconcile_projection_store()
  -> 读取/补全 backend/soul/projections/catalog.json
  -> 返回 selected_projection_id + cards
```

```text
运行时构建投影视图
  -> SoulProjectionRequest
  -> SoulProjectionBuilder.build()
  -> PromptSection[]
  -> SoulRuntimeView
  -> build_prompt_manifest()
  -> SoulPromptManifest
```

## 13. 技术要点

```text
1. 灵魂系统当前以文件为主要存储介质。
2. builtin souls 写在代码常量和 seed markdown 中。
3. custom souls 写在 soul/custom/{id}/profile.json 与 SOUL.md 中。
4. active soul 通过 ACTIVE_SEED.md 表示。
5. projection cards 通过 soul/projections/catalog.json 表示。
6. prompt 装配不是直接返回纯文本，而是先形成 PromptSection[]。
7. PromptManifest 使用 SHA256 记录装配后 prompt 的 hash。
8. API 层负责路径校验、上传校验、投影卡片选择和 custom soul 写入。
9. 前端当前主要把灵魂作为可切换表达风格和系统框架图节点展示。
10. ResourcePolicy 在契约中被标记为工具授权归属方。
```
