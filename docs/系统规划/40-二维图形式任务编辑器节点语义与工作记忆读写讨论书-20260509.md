# 40-二维图形式任务编辑器节点语义与工作记忆读写讨论书

日期：2026-05-09

## 1. 讨论范围

本文件只记录当前已经讨论过的内容。

当前只讨论两个对象：

```text
Agent 节点
工作记忆库节点
```

以及它们之间的两类关系：

```text
Agent -> 工作记忆库：谁写
工作记忆库 -> Agent：谁读
```

其他对象暂不纳入本文件，包括但不限于：

- Artifact 节点
- Barrier 节点
- ManualGate 节点
- 调度节点
- DSL
- LangGraph 编译
- RunLoop 实施细节

这些内容后续可以逐项讨论，但当前不预设。

## 2. 当前共识

### 2.1 编辑器定位

编辑器不是单纯的画布工具。

当前讨论中的定位是：

```text
二维图形式任务编辑器
```

但本文件暂时只讨论其中的 Agent 节点和工作记忆库节点。

### 2.2 Agent 节点

Agent 节点建议使用圆节点表示。

语义：

```text
Agent 节点 = 一个任务组织中的执行主体
```

它代表某个 Agent 在当前任务能力结构中的角色。

当前已讨论属性：

- 它是执行者。
- 它可以向工作记忆库写入成果。
- 它可以从工作记忆库读取成果。
- 它不等于工作记忆库本身。

暂不展开：

- Agent prompt
- 投影系统
- 工具权限
- 契约
- 调度
- 失败策略

### 2.3 工作记忆库节点

工作记忆库节点建议使用方节点表示。

语义：

```text
工作记忆库节点 = 当前任务组织中的受控记忆资源
```

它不是 Agent。

它用于显式表达：

- 哪些 Agent 可以写入。
- 哪些 Agent 可以读取。
- 工作成果如何在 Agent 之间沉淀和复用。

当前已讨论属性：

- 它是任务过程中的工作记忆资源。
- 它应该进入编辑器，成为可见节点。
- 它用于控制 Agent 之间的记忆成果共享。

暂不展开：

- 记忆类型分类
- 晋升长期记忆
- 审核机制
- 生命周期
- 存储实现

## 3. 读写关系

### 3.1 Agent 写工作记忆库

图上表示为：

```text
Agent 圆节点 -> 工作记忆库方节点
```

含义：

```text
这个 Agent 可以向这个工作记忆库写入成果
```

需要后续讨论的问题：

- 写入是直接写，还是提交候选。
- 写入是否需要审核。
- 写入内容由谁决定。
- 写入失败如何处理。
- 是否允许多个 Agent 写同一个工作记忆库。

### 3.2 Agent 读工作记忆库

图上表示为：

```text
工作记忆库方节点 -> Agent 圆节点
```

含义：

```text
这个 Agent 可以读取这个工作记忆库中的成果
```

需要后续讨论的问题：

- 读取是全文、摘要还是引用。
- 读取是否需要筛选。
- 读取是否需要审批。
- 读取是否有次数或范围限制。
- 是否允许一个 Agent 读取多个工作记忆库。

## 4. 当前最小模型

当前讨论可暂定为：

```text
圆节点：Agent
方节点：工作记忆库
圆 -> 方：写入权限 / 写入关系
方 -> 圆：读取权限 / 读取关系
```

示例：

```text
章节写作 Agent  ->  章节工作记忆库
章节工作记忆库  ->  连续性审查 Agent
```

含义：

- 章节写作 Agent 可以向章节工作记忆库写入成果。
- 连续性审查 Agent 可以从章节工作记忆库读取成果。

## 5. 暂不决定的问题

以下问题先记录，不在当前文件中提前给结论：

1. 工作记忆库节点是否可以有多个。
2. 工作记忆库是否需要 owner。
3. 写入是否默认进入候选队列。
4. 读取是否默认只读摘要。
5. 读写关系是否同时影响任务执行顺序。
6. 工作记忆和长期记忆如何衔接。
7. 工作记忆库节点是否需要分类型。
8. 工作记忆库的可视化信息应该显示多少。

## 6. 当前边界

当前只确认一件事：

```text
工作记忆应该在二维任务编辑器中显式图化。
```

图化方式先从最小表达开始：

```text
Agent 圆节点
工作记忆库方节点
读写方向边
```

后续逐项讨论后，再决定是否扩展更多节点和边类型。
