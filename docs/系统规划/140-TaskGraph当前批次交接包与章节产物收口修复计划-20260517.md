# TaskGraph当前批次交接包与章节产物收口修复计划

日期：2026-05-17

## 一、问题判断

这次不是写手单点失误，而是 TaskGraph 运行时对“节点执行实例”的收口不够严格：

1. `chapter_outline -> chapter_draft` 边存在，但当前批次细纲没有稳定进入写手最终可读输入，导致写手在正文里声明“上游未提供细纲”。
2. `chapter_draft` 的失败重试产物先落盘到正式章节目录，再由质量门判 rejected，污染了正式产物区。
3. `stage_results` 以 `stage_id` 为主键覆盖，缺少 `dispatch_event_id + batch_start/batch_end/round` 实例身份，旧重试结果可能覆盖已推进状态。
4. `memory_commit_chapter` 没有按 `batch + source_draft + source_review` 做幂等，同一批通过稿可能重复 commit。
5. 正文产物把正文和承接说明、自检、摘要混在同一个 `draft_round_*.md`，下游和用户无法区分“可交付正文”和“节点回执/manifest”。

## 二、修复原则

1. 当前批次未提交前，细纲通过 artifact handoff 边交给写手；提交后才进入章节细纲索引供后续读取。
2. 模型可见上下文必须由运行时确定性装配，不能靠 agent 猜测引用含义。
3. 未 accepted 的业务产物不得占用正式产物路径。
4. 每个业务结果必须绑定执行实例身份，过期结果只能记录为 stale，不得更新 `stage_results`。
5. commit 节点必须按来源身份幂等，不能只按 stage 名收口。

## 三、实施范围

### 1. 交接包注入

- 为 `chapter_draft` 显式加入当前批次细纲 artifact context。
- 为 `chapter_review` 和 `memory_commit_chapter` 补充当前批次细纲上下文，保证审核和提交都按同一批 outline/draft/review 收口。
- 运行时 message 组装消费 `artifact_context_packet.expanded_text_by_input_key`，并在缺少必需 artifact 时 fail closed。

### 2. 执行实例身份

- 给 stage result 写入 `dispatch_event_id`、`request_id`、`batch_start_index`、`batch_end_index`、`round_index`。
- 在收到 result 时校验 result 对应的 `request_id/dispatch_event_id` 是否等于当前 active request。
- 过期 result 只写 `stale_stage_results` 与 timeline 事件，不覆盖 `stage_results`。

### 3. rejected 产物隔离

- artifact materializer 支持 `acceptance_status`。
- 对需要质量门的 coordination stage，先计算业务接受态，再把 rejected 产物写入 `rejected/<stage>/<batch>/<request>`，并且不把这些 refs 暴露为 accepted output refs。
- 正式目录只保留 accepted 产物。

### 4. 正文与 manifest 拆分

- `chapter_draft` 输出目标拆成正文主体和 manifest。
- 正文文件只保留 `【章节正文候选】` 或正文主体。
- 承接说明、目标完成、自检、公开摘要进入 manifest/receipt 文件。

### 5. commit 幂等

- 对 `memory_commit_chapter` 计算 commit identity：`volume_index + batch_start_index + batch_end_index + source_draft_refs + source_review_refs`。
- 已提交过同一 identity 时，重复 result 记录为 duplicate commit，不能再次推进或生成第二个正式 commit。

## 四、验收标准

1. `chapter_draft` message 中能看到“当前批次细纲”全文或明确可读片段。
2. 缺少 `chapter_outline` artifact 时，`chapter_draft` 不会启动模型写作。
3. rejected draft 不会出现在正式 `volume_xxx/chapters/chapter_xxx_xxx/` 目录。
4. stale dispatch result 不覆盖当前 `stage_results`。
5. 同一 `draft + review + batch` 只能产生一个 commit。
6. 正文文件不再混入 manifest 信息。

## 五、收尾

- 增加回归测试覆盖交接包注入、stale result、rejected artifact 隔离和 commit 幂等。
- 更新写作任务配置脚本并重新生成任务图配置。
