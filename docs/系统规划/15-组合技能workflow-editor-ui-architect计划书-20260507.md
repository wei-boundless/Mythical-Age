# 组合技能 `workflow-editor-ui-architect` 计划书

日期：2026-05-07

## 一、计划目标

基于《前端整体重设计审查报告》，为当前产品前端制定一份可直接实施的整体重设计方案。

本次方案不是单页美化，也不是局部补丁，而是围绕以下目标展开：

1. 保留“灵魂切换驱动风格”的核心机制。
2. 把当前前端从“展示感较强的神话界面”重构为“神话气质的工业化工作台”。
3. 显著降低壳层重量、卡片厚度、按钮体积和说明性信息密度。
4. 把任务系统与协调监控真正提升为核心生产界面。
5. 建立统一的信息架构、布局语义、按钮等级和组件风格基线。

## 二、设计定位

## 产品气质

新前端应当保持以下特征：

- 有灵魂
- 有神话感
- 有品牌记忆点
- 有系统感

但同时必须具备：

- 高效工作台属性
- 更强的任务导向
- 更大的核心工作区
- 更低的视觉噪声

## 目标关键词

新前端关键词定为：

- 神话气质
- 冷静
- 轻锋利
- 编排工作台
- 高密但不压迫
- 清晰可持续

## 非目标

本次不做：

- 改写后端任务系统
- 变更灵魂系统数据结构
- 重写任务/编排运行逻辑
- 追求营销型大视觉首页

## 三、核心判断

前端重设计应当遵守一个总原则：

**品牌气质放在壳层，工作效率放在主层。**

也就是：

- 灵魂系统决定色相、光感、纹理和情绪
- 工作台决定布局、密度、层级和交互效率

前端不能再让“品牌表达”压住“功能使用”。

## 四、总体结构方案

## 1. 三层前端结构

将整体前端拆分为三层：

### A. 品牌层

负责：

- 顶层品牌身份
- 灵魂切换视觉映射
- 全局基调和情绪

包含：

- 缩窄版 Navbar
- 全局色相与纹理变量
- 当前灵魂状态锚点

要求：

- 轻
- 稳
- 不抢高度

### B. 导航层

负责：

- 模块切换
- 会话切换
- 上下文抽屉入口

包含：

- 左侧主导航栏
- 会话列
- 模块抽屉或导航抽屉

要求：

- 可收起
- 可专注
- 不在生产页面长期双边包夹

### C. 工作层

负责：

- 编辑
- 运行
- 监控
- 审查

包含：

- ChatPanel
- TaskSystemView
- OrchestrationView
- CoordinationRunPanel

要求：

- 最大化工作面积
- 低装饰
- 主次明确

## 2. 页面模式划分

将页面分成三种运行模式：

### 模式一：会话模式

适用于：

- 主会话
- 检索
- 普通问答

特点：

- 保留会话栏
- 保留较多灵魂感
- 允许稍高呼吸感

### 模式二：工作模式

适用于：

- 任务系统
- 编排系统
- MCP 管理
- 健康系统

特点：

- 自动隐藏右侧工作台栏
- 左侧栏缩窄
- 顶部栏减高
- 主区域放大

### 模式三：专注模式

适用于：

- 协调任务编辑器
- 拓扑搭建
- 协调监控
- 大型表单和图形工作区

特点：

- 隐藏右侧模块栏
- 左侧会话栏最小化或折叠
- 工作区全宽
- 工具条、图、检查器优先

## 五、灵魂风格机制保留与改造

## 1. 保留内容

保留现有：

- `html[data-soul="..."]` 变量切换
- 灵魂主色 / 软色 / glow / texture / motion
- 灵魂人物画像资源
- 灵魂简介与人格锚点

## 2. 改造方向

灵魂机制从“大型人格面板”转为“全局风格引擎”。

### 保留为全局变量的内容

- 主色
- 次色
- 高亮色
- 边界光
- 面板氛围
- 微动画节奏

### 从主壳层中缩减的内容

- 过高的大型人物展示区
- 长说明文案
- 每页重复的灵魂解释

## 3. 新的灵魂呈现方式

### 默认态

在左侧栏顶部仅保留：

- 灵魂名
- 小型切换器
- 小头像或符号

### 扩展态

点击后打开灵魂抽屉，展示：

- 灵魂立绘
- 简介
- 投影入口
- 风格设定

这样既保留人格感，又不长期占用工作空间。

## 六、导航系统重构方案

## 1. 顶部 Navbar

### 当前问题

- 高度偏大
- 品牌标题体积过重
- 对工作页贡献有限

### 新方案

改成低高度品牌条：

- 左：品牌字标
- 中：当前模式或当前对象标题
- 右：新会话 / 全局状态 / 快捷入口

### 要求

- 高度减少约 30%
- 标题字号下调
- 状态 pills 缩小
- 让首屏先看到工作区

## 2. 左侧 Sidebar

### 当前问题

- 灵魂区偏大
- 会话卡片太厚
- 信息密度偏低

### 新方案

分成两级：

#### 一级窄导航

- 灵魂按钮
- 会话
- 任务系统
- 编排系统
- 记忆
- 健康
- 能力
- MCP
- 配置

默认宽度较窄，可显示图标和简短标签。

#### 二级上下文面板

根据当前模块切换具体内容：

- 会话模式：显示会话列表
- 灵魂模式：显示灵魂信息
- 任务模式：显示当前对象目录

也就是说：

不要把灵魂、会话、模块入口永久塞在同一大卡里。

## 3. RightRail

### 当前判断

RightRail 作为常驻模块导航，不适合继续保留原形态。

### 新方案

将 RightRail 改为：

- 全局抽屉入口
- 快速切换面板
- 非常驻上下文栏

默认关闭，仅在需要时呼出。

### 结果

- 释放横向空间
- 减少双边包夹
- 主区更稳

## 七、任务系统重设计方案

## 1. 任务系统页面定位

任务系统不是“管理信息页”，而是：

**任务定义、任务装配、协调搭建的工作台。**

因此它需要：

- 更少的说明
- 更强的对象聚焦
- 更清楚的编辑分层

## 2. 页面分层

建议任务系统改成三层：

### 顶层：任务对象上下文条

显示：

- 当前任务域
- 当前对象
- 当前层级
- 保存状态

不再放太多解释文字。

### 第二层：层级导航

- 任务域
- 装配
- 协调
- 契约

每个按钮只显示名称与简短状态，不再承载过多二级摘要。

### 第三层：对应工作面

进入哪一层，就显示哪一层的真实工作台。

## 3. 域内协调任务页面

这是重构重点。

建议结构固定为：

### A. 顶部对象条

- 当前协调任务名
- 当前状态
- 模板与草稿动作
- 保存与发布动作

### B. 中部拓扑主区

- 拓扑图独占第一视觉
- 工具条贴近图
- 图状态贴近图

### C. 下部双栏

- 左：任务池 / 节点模板
- 右：节点检查器 / 边检查器 / 总体设置

### D. 底部辅助面板

- 快速连线
- 校验问题
- 系统字段折叠区

### 原则

- 图优先
- 编辑器优先
- 摘要后置
- 系统字段折叠

## 4. 长篇小说模板在任务系统中的位置

长篇小说持续交付模板不应该再塞成普通按钮。

建议改成：

- “模板库”入口
- 模板选择器
- 预览模板拓扑
- 应用模板

也就是：

模板是编辑器资源，不是页面顶栏动作。

## 八、编排系统重设计方案

编排系统应与任务系统统一语言，但更偏：

- Agent 目录
- 运行装配
- 组装关系
- 执行权限

### 建议结构

#### 左：对象目录

- Agent
- Agent 组
- 运行 lane
- 默认策略

#### 中：主详情与编辑器

- 选中对象核心配置
- 运行绑定
- 任务关联

#### 右：诊断与引用关系

- 被哪些任务引用
- 当前默认灵魂
- 可见技能
- 风险提示

### 原则

- 编排系统不应该沿用“展示页式大卡片”
- 更像专业装配台

## 九、会话与协调监控重设计方案

## 1. ChatPanel

保留：

- 会话
- 协调监控

但重构目标是让它更像工作工具。

### 改法

- 空状态缩减装饰
- 会话头部缩窄
- token 指标更轻
- 任务上下文条更明显

## 2. CoordinationRunPanel

这是产品中最有潜力的页面之一。

### 新结构建议

#### 顶部：运行摘要条

- 协调任务名
- 当前节点
- 当前 Agent
- 状态

#### 中部：拓扑主画布

- 占最大空间
- 当前节点发光
- 当前 handoff 更明显

#### 下部：双栏

- 左：流式输出
- 右：产物清单

### 风格方向

- 更像监控台
- 更少“卡片展板感”
- 更强“执行过程感”

## 十、按钮与交互体系重构

## 1. 四级按钮体系

### L1 主按钮

- 发布
- 运行
- 保存

特征：

- 高对比
- 中等尺寸
- 明确主次

### L2 次按钮

- 创建
- 复制
- 应用模板
- 导入

特征：

- 轻于主按钮
- 不抢视觉核心

### L3 轻工具按钮

- 编辑
- 删除
- 切换
- 刷新

特征：

- 更窄
- 更低
- 更接近工具态

### L4 图内按钮

- 节点工具
- 边工具

特征：

- 图内浮动
- 小尺寸
- 不抢主图

## 2. 交互节奏

目标是：

- 主动作用按钮
- 次动作做成工具
- 页面少重复解释
- 让用户扫一眼就知道下一步

## 十一、视觉系统重构方案

## 1. 颜色

保留深色基底和灵魂色差异。

但要减少：

- 大面积高对比 glow
- 连续多层次发光边框
- 强纹理覆盖

改成：

- 壳层有气质
- 工作面更干净

## 2. 卡片

建议：

- 页面 section 不再普遍卡片化
- 真正需要容器边界时再用卡片
- 单层卡片替代卡片嵌卡片

## 3. 字体与排版

建议：

- 品牌字保留神话气质
- 系统正文更克制、更紧凑
- 减少过大的大标题
- 降低说明文字占比

## 4. 密度

系统页应提升信息密度，但不能拥挤。

实现方式：

- 控制留白
- 收窄按钮
- 缩小状态 pill
- 折叠系统字段
- 减少重复说明

## 十二、技术实施策略

## 1. 先重构壳层，再重构页面

顺序建议：

1. `page.tsx` 与全局壳层
2. `Navbar.tsx`
3. `Sidebar.tsx`
4. `RightRail.tsx`
5. `globals.css` 的布局 token 和按钮 token
6. `TaskSystemView.tsx`
7. `OrchestrationView.tsx`
8. `ChatPanel.tsx`
9. `CoordinationRunPanel.tsx`

## 2. 从全局样式中抽出设计 token

当前 `globals.css` 已经过大。

建议后续拆分：

- `tokens.css`
- `shell.css`
- `workbench.css`
- `coordination.css`
- `soul-theme.css`

## 3. 视图组件减负

当前大视图文件体量偏大：

- `globals.css` 约 19671 行
- `TaskSystemView.tsx` 约 2886 行
- `PlaygroundView.tsx` 约 1422 行
- `OrchestrationView.tsx` 约 1176 行

这意味着：

- 视觉逻辑与业务逻辑耦合偏高
- 样式修改成本大
- 页面容易持续膨胀

建议后续拆分页面内部组件：

- 任务系统：壳层、对象条、模板库、拓扑编辑器、检查器
- 编排系统：目录、详情、关系面板
- 协调监控：头部、图、输出、产物

## 十三、阶段化实施计划

## 阶段一：壳层减重

目标：

- 减少笨重感

交付：

- 缩顶栏
- 改左栏
- 移除常驻右栏
- 建立三种页面模式

## 阶段二：任务系统与编排系统重构

目标：

- 让生产工具页面真正像工作台

交付：

- 任务系统新布局
- 协调编辑器新布局
- 编排系统新布局

## 阶段三：监控与会话重构

目标：

- 让运行页更锋利

交付：

- ChatPanel 简化
- CoordinationRunPanel 工具化

## 阶段四：灵魂引擎收敛

目标：

- 保留识别度，降低压迫感

交付：

- 灵魂变量系统收敛
- 灵魂抽屉
- 统一 soul-driven token

## 十五、最终判断

这次前端重设计不应该理解为“推翻现有全部成果”，而应理解为：

**保留世界观级别的识别机制，重做工作台级别的结构秩序。**

保留的核心：

- 灵魂切换风格机制
- 神话气质
- 模块化工作台
- 协调任务图形化

必须重做的核心：

- 壳层布局
- 导航结构
- 按钮体系
- 页面密度
- 任务与编排的工作台语义

一句话总结：

**让前端从“有气质的复杂界面”变成“有气质的高效工作台”。**
