# 组合技能 workflow-editor-ui-architect 搭建计划书

日期：2026-05-07

## 背景

当前项目已经具备两个可互补的前端设计技能：

- `frontend-design`：擅长高完成度视觉设计与前端界面实现
- `ui-ux-pro-max`：擅长设计系统生成、行业匹配、信息架构与实现约束检索

这两个技能各自能力明确，但在复杂工作台、编辑器、控制台类界面上仍存在协作断层：

1. 直接使用 `frontend-design`，容易先进入视觉实现，前置的信息架构和设计系统约束不够显式。
2. 直接使用 `ui-ux-pro-max`，容易得到结构化建议，但最终界面风格和完成度不一定自动拉满。
3. 项目自身还有额外约束：优先结构性重构、不同层级页面分离、用卡片按钮做页面切换、必须实际检查结果、大改动先写计划书。

因此需要一个项目内的上层组合技能，用来编排这两个技能的工作顺序，并把项目约束固化进去。

## 目标

创建一个项目级组合技能 `workflow-editor-ui-architect`，用于以下场景：

- 工作流编辑器
- 协调任务搭建器
- 多栏工作台
- 控制台/后台复杂页面
- 既要做结构设计，又要做高质量视觉实现的前端重构任务

该技能不复制两个现有技能的大段内容，而是负责：

1. 定义触发范围
2. 规定组合工作流
3. 固化项目特有约束
4. 指导何时读取、何时调用现有技能资源

## 设计原则

1. 组合，不拼接。

   不把两个技能全文硬合并到一个 `SKILL.md` 里，而是只保留编排逻辑与项目约束。

2. 优先结构，后做视觉。

   先用 `ui-ux-pro-max` 明确设计系统、信息层级和反模式，再进入 `frontend-design` 的界面实现阶段。

3. 贴合项目真实场景。

   技能描述和流程要直接面向“任务系统 / 协调任务 / 编辑器 / 工作台 / 检查器 / 画布”这类问题，而不是泛化成普通 landing page 技能。

4. 控制上下文体积。

   `SKILL.md` 保持精简，把可选检查清单和项目约束拆到 `references/`，按需读取。

5. 不保留无用残留。

   只创建必要文件，不额外生成 README、安装说明、变更日志等无用附属文档。

## 技能结构

计划创建如下目录：

```text
.codex/skills/workflow-editor-ui-architect/
├── SKILL.md
└── references/
    └── workflow-editor-checklist.md
```

### 1. `SKILL.md`

职责：

- 提供技能元数据
- 定义何时触发
- 定义四阶段工作流
- 指出何时读取 `ui-ux-pro-max` 与 `frontend-design` 的内容
- 约束复杂前端任务必须先做结构设计，再做实现

### 2. `references/workflow-editor-checklist.md`

职责：

- 汇总项目内与复杂编辑器设计直接相关的约束
- 提供页面分层、卡片切换、工作台布局、检查项
- 作为实现前和验收前的核对清单

## 组合工作流

### 阶段一：任务定性

识别当前请求是否属于以下类型：

- 编辑器
- 工作台
- 多层级控制台
- 拓扑/流程/节点边配置界面
- 需要左中右分栏、主次导航、检查器、状态面板的界面

若属于上述类型，则触发本技能。

### 阶段二：结构与设计系统

先读取 `ui-ux-pro-max`，完成：

1. 产品类型判断
2. 行业/业务场景判断
3. 设计系统生成
4. 工作台模式、信息层级、反模式确认

输出物至少应包含：

- 页面分层
- 视图切换方式
- 主操作区与辅助区划分
- 色彩/字体/密度建议
- 不应采用的反模式

### 阶段三：高质量界面实现

再读取 `frontend-design`，完成：

1. 明确界面气质
2. 选择合适的视觉方向
3. 按既定结构实现界面
4. 完成状态、空态、交互反馈和响应式细节

这里 `frontend-design` 负责“做得成熟且好看”，但不能推翻第二阶段已经确认的结构。

### 阶段四：项目约束验收

实现前后都要对照项目约束核查：

1. 是否优先解决结构问题，而不是针对单点做补丁
2. 是否把不同层级页面拆开，而不是混在同一页
3. 是否使用卡片按钮完成分层切换
4. 是否实际检查前端结果
5. 如果改动较大，是否先补计划书并按计划实施

## 实施步骤

### 阶段一：新增计划书

新增本计划书，明确目标、结构、流程和验收方式。

### 阶段二：创建组合技能

在项目 `.codex/skills/` 下创建 `workflow-editor-ui-architect` 目录及 `SKILL.md`。

### 阶段三：补充引用式参考文件

新增 `references/workflow-editor-checklist.md`，放置项目相关检查项，避免把项目约束全部堆入主技能说明。

### 阶段四：结构核对

检查技能目录、文件命名、引用路径、说明是否自洽，确保后续可以直接在本项目中使用。

## 非目标

本次不做：

- 改写 `frontend-design` 原技能
- 改写 `ui-ux-pro-max` 原技能
- 为该技能额外开发脚本
- 在本次回合里直接重构业务前端页面

本次目标仅为建立一个可复用的项目级组合技能。
