# 编排系统架构设计

日期：2026-04-29  
定位：编排系统是 agent runtime 的控制大脑。它不负责理解全部业务细节，也不直接执行工具，而是把任务、资源、灵魂投影、记忆上下文、候选计划和执行边界统一收束成可验证的控制平面。  
当前施工目标：先完成单主 agent 的 preview 编排主链，不开放真实工具、worker、agent、文件写入、session 写回或 durable memory 写回。

---

## 0. 结论

编排系统的核心职责：

```text
接收各系统候选。
建立当前轮执行计划。
验证计划是否可采纳。
构建执行图预览。
在条件满足时生成 RuntimeDirective。
在执行完成后把结果交给 CommitGate / OutputBoundary。
```

当前阶段只做到：

```text
TaskOperationPreview
  -> CandidateCollector
  -> OrchestrationPlanPreview(single_agent)
  -> PlanValidationResult
  -> ExecutionGraphPreview
  -> AdoptionCandidate(blocked)
  -> AdoptionBlock(blocked)
  -> RuntimeDirectiveCandidate(candidate_only)
  -> RuntimeDirectiveBuildBlock(blocked)
  -> OperationGatePreflightPreview(blocked)
  -> DirectiveOnlyExecutorPreview(blocked)
  -> CommitGatePreview(blocked)
  -> ControlKernel
  -> blocked / preview_only
```

暂不做：

```text
真实工具执行。
真实 worker 执行。
真实 bounded agent 执行。
RuntimeDirective 可消费化。
CommitGate 放行写回。
multi_agent / hybrid topology adoption。
```

一句话：

```text
编排系统先成为可解释的大脑，再成为可执行的大脑。
```

## 1. 编排系统在全局架构中的位置

### 1.1 系统分工

```text
TaskSystem
  负责把用户请求整理成 TaskContract / TaskDefinition / TaskBinding / TaskPromptContract。

OperationSystem
  负责把资源需求整理成 OperationRequirement / ResourcePolicyPreview / ResourceRuntimeView。

SoulSystem
  负责把任务、资源、技能和角色边界投影成模型可见的 SoulRuntimeView / PromptManifestPreview。

MemorySystem
  负责提供记忆上下文候选、记忆读写边界和后续 MemoryWriteCandidate。

OrchestrationSystem
  负责采纳、排序、验证、建图、生成指令候选和控制写回边界。

ExecutionRuntime
  未来只按 RuntimeDirective 执行。

CommitGate / OutputBoundary
  负责结果是否写回、如何成为最终用户可见答案。
```

### 1.2 编排系统不做什么

```text
不直接解析自然语言成为工具调用。
不直接扩大 ResourcePolicy。
不直接修改 PromptManifest。
不直接写 session。
不直接写 durable memory。
不直接把 WorkerResult / AgentResult 变成 final answer。
不在 PlanValidator blocked 后回退到旧执行链。
```

## 3. 设计原则

### 3.1 单一执行真相

当前轮执行真相只能来自：

```text
ControlKernel adopted OrchestrationPlan
  -> ExecutionGraph
  -> RuntimeDirective
```

禁止：

```text
QueryPlanner 直接决定 execution_kind。
query_understanding.tool_name 直接进入 RuntimeToolBridge。
worker_plan 直接进入 EvidenceOrchestrator。
TaskCoordinator 直接从任务绑定执行并写结果。
SoulProjection 声明工具权限。
Memory restore 覆盖当前轮执行决定。
```

### 3.2 Candidate 不等于 Decision

所有上游系统只能生产候选：

```text
TaskOperationPreviewCandidate
LegacyQueryPlanCandidate
FollowupCandidate
MemoryContextCandidate
CapabilityCandidate
ToolCandidate
WorkerCandidate
OutputCommitCandidate
```

候选固定规则：

```text
authority = candidate_only
不能包含 RuntimeDirective
不能触发工具
不能写 session
不能写 memory
不能作为 final answer
```

### 3.3 控制平面与执行平面分离

```text
Control Plane:
  CandidateCollector
  OrchestrationCoordinator
  PlanValidator
  ExecutionGraphPreviewBuilder
  RuntimeDirectiveBuilder
  CommitPlanner

Execution Plane:
  ModelExecutor
  ToolExecutor
  WorkerExecutor
  BoundedAgentExecutor
```

当前只做 Control Plane preview。

### 3.4 Fail-Closed 默认

必须 blocked 的情况：

```text
缺 TaskContract。
缺 ResourcePolicy。
ResourcePolicy 仍是 preview。
PlanValidator 未通过。
ExecutionGraph 没有 adopted policy refs。
RuntimeDirective 缺失。
OperationGate preflight 已拒绝，真实 OperationGate 未通过。
CommitGate 未允许。
```

## 4. 目标架构

### 4.1 控制平面主链

```text
TaskOperationPreview
  -> CandidateCollector
  -> OrchestrationCoordinator
  -> OrchestrationPlanPreview
  -> PlanValidator
  -> ExecutionGraphPreviewBuilder
  -> ControlKernel
  -> blocked / preview_only
```

### 4.2 未来可执行主链

```text
OrchestrationPlanPreview
  -> AdoptionCandidate
  -> OrchestrationPlan
  -> AdoptedResourcePolicy
  -> ExecutionGraph
  -> RuntimeDirective
  -> OperationGate
  -> Executor
  -> ResultCandidate
  -> CommitCandidate
  -> CommitGate
  -> OutputBoundary
```

### 4.3 核心对象关系

```text
CandidateSet
  contains CandidateEnvelope[]

OrchestrationPlanPreview
  selects candidate refs
  references task / resource / prompt / topology refs
  owns stage preview order

PlanValidationResult
  checks plan consistency
  blocks runtime execution in phase 1

ExecutionGraphPreview
  describes future nodes without creating ExecutionNode

RuntimeDirectiveCandidate
  describes possible runtime directive but cannot be consumed

ControlKernelResult
  owns final preview control status
```

## 5. 模块设计

### 5.1 CandidateCollector

文件：

```text
backend/orchestration/collector.py
```

职责：

```text
收集任务、操作、灵魂、记忆、旧 planner、follow-up、capability 的候选。
统一包装为 CandidateEnvelope。
不选择。
不采纳。
不执行。
```

输入：

```text
TaskOperationPreview
history refs
session refs
legacy QueryPlan snapshot
followup snapshot
memory context snapshot
capability summary
```

输出：

```text
CandidateSet
```

第一阶段最小候选：

```text
task_operation_preview
single_agent_topology
resource_policy_preview
prompt_manifest_preview
legacy_query_plan_snapshot optional
```

验收：

```text
每个 CandidateEnvelope.authority == "candidate_only"。
candidate_count 可进入 ControlKernel diagnostics。
candidate payload 不含 executable directive。
```

### 5.2 OrchestrationCoordinator

文件：

```text
backend/orchestration/coordinator.py
```

职责：

```text
从 CandidateSet 组装 OrchestrationPlanPreview。
选择 single_agent topology。
建立 stage preview。
引用 task/resource/prompt/topology refs。
不执行。
```

当前阶段固定：

```text
topology = "single_agent"
stage_count = 1
executor_hint = "model"
runtime_executable = false
```

禁止：

```text
不根据模型文本直接开工具。
不根据 QueryExecutionPlan 直接生成 ExecutionNode。
不采纳 multi_agent / hybrid topology。
```

### 5.3 OrchestrationPlanPreview

文件：

```text
backend/orchestration/plan.py
```

建议字段：

```python
@dataclass(slots=True, frozen=True)
class OrchestrationPlanPreview:
    plan_id: str
    task_id: str
    topology_ref: str
    topology_mode: str = "single_agent"
    task_contract_ref: str = ""
    task_prompt_contract_ref: str = ""
    resource_policy_ref: str = ""
    prompt_manifest_ref: str = ""
    selected_candidate_refs: tuple[str, ...] = ()
    stages: tuple[OrchestrationStagePreview, ...] = ()
    preview_only: bool = True
    adopted: bool = False
    runtime_executable: bool = False
    authority: str = "orchestration_plan_preview"
```

约束：

```text
preview_only == true
adopted == false
runtime_executable == false
```

### 5.4 OrchestrationStagePreview

建议字段：

```python
@dataclass(slots=True, frozen=True)
class OrchestrationStagePreview:
```

当前阶段只生成一个 stage：

### 5.5 PlanValidator

文件：

```text
backend/orchestration/validation.py
```

职责：

```text

```

第一阶段检查项：

```text

task_prompt_contract_exists
```

### 5.6 ExecutionGraphPreviewBuilder

文件：

```text
backend/orchestration/graph_preview.py
```

职责：

```text
从 OrchestrationPlanPreview 和 PlanValidationResult 生成 ExecutionGraphPreview。
表达未来节点形状。
不创建 ExecutionNode。
```

原因：

```text
ExecutionNode.authority 必须是 runtime_directive。
preview 阶段不能伪造 runtime_directive。
```

```python
    node_type: str
    executor_hint: str
    stage_ref: str
    operation_refs: tuple[str, ...] = ()
    policy_refs: tuple[str, ...] = ()
    depends_on: tuple[str, ...] = ()
    executable: bool = False
    blocked_reason: str = "preview_only"
    authority: str = "preview_only"
```

```python
@dataclass(slots=True, frozen=True)
class ExecutionGraphPreview:
    graph_preview_id: str
    task_id: str
    plan_ref: str
    node_previews: tuple[ExecutionNodePreview, ...] = ()
    edge_previews: tuple[dict[str, Any], ...] = ()
    runtime_executable: bool = False
    blocked_reason: str = "preview_only"
```

### 5.7 RuntimeDirectiveCandidate


```text

  -> build_task_runtime_contract_preview()
  -> UnderstandingCandidateLayer.build_understanding_candidates()
  -> CandidateCollector.collect()
  -> OrchestrationCoordinator.build_preview_plan()
  -> PlanValidator.validate_preview()
  -> ExecutionGraphPreviewBuilder.build()
  -> AdoptionCandidate.build_blocked()
  -> AdoptionBlock.build_preview()
  -> RuntimeDirectiveCandidate.build()
  -> RuntimeDirectiveBuildBlock.build_preview()
  -> OperationGatePreflightPreview.build_blocked()
  -> DirectiveOnlyExecutorPreview.build_blocked()
  -> CommitGatePreview.build_blocked()
  -> ControlKernel.collect()
  -> emit task_operation_preview
  -> emit candidate_set_preview
  -> emit orchestration_plan_preview
  -> emit plan_validation
  -> emit execution_graph_preview
  -> emit adoption_candidate_preview
  -> emit runtime_directive_candidate_preview
  -> emit commit_gate_preview
  -> emit orchestration_control
  -> emit error preview_only
```

### 6.2 事件输出

第一阶段事件顺序固定：

```text

  contains understanding_candidate_preview entries

adoption_candidate_preview
adoption_block
runtime_directive_candidate_preview
runtime_directive_block

```

每个事件必须包含：

```text
preview_only = true
runtime_executable = false
runtime_directive_enabled = false
```

## 7. 与各系统的协调方式

### 7.1 与任务系统

输入：

```text
TaskContract
TaskDefinition
TaskBinding
TaskPromptContract
ExecutionTopologyPreview(mode=single_agent)
```

编排系统使用：

```text
task_id
task_mode
task_family
completion criteria
prompt contract ref
review policy
```

编排系统不得：

```text
修改 TaskContract。
把 TaskBinding 当作工具授权。
直接 mark task running / completed。
```

### 7.2 与操作系统

输入：

```text
OperationRequirement
ResourcePolicyPreview
ResourceRuntimeView
OperationRegistry refs
```

编排系统使用：

```text
resource_policy_ref
denied_operations
requires_approval_operations
preview_available operations
risk diagnostics
```

编排系统不得：

```text
把 ResourcePolicyPreview 当 AdoptedResourcePolicy。
绕过 OperationGate。
把 requires_approval 静默转 allow。
```

### 7.3 与灵魂系统

输入：

```text
SoulRuntimeView
PromptManifestPreview
TaskPromptContract sections
```

编排系统使用：

```text
prompt_manifest_ref
projection diagnostics
section source refs
```

编排系统不得：

```text
让 SoulProjection 改变权限。
让 prompt section 变成执行状态。
```

### 7.4 与记忆系统

当前只接候选，不执行写回。

输入：

```text
MemoryContextCandidate
MemoryRestoreCandidate
MemoryWriteCandidate future
```

第一阶段：

```text
memory candidate 只能进入 CandidateSet。
memory write 必须 blocked。
durable memory 写回必须等待 CommitGate / MemoryGate。
```

后续：

```text
MemoryPolicy
  -> memory read scope
  -> memory write candidate scope
  -> adopted memory write
```

### 7.5 与执行层

当前不接执行层。

未来接入条件：

```text
PlanValidator passed。
ResourcePolicy adopted。
RuntimeDirective generated。
OperationGate passed。
Executor supports directive type。
CommitGate remains deny by default for writes。
```

## 10. 验收标准

### 10.1 必须通过的结构断言

```text
CandidateEnvelope.authority == "candidate_only"
OrchestrationPlanPreview.preview_only == true
OrchestrationPlanPreview.adopted == false
OrchestrationPlanPreview.runtime_executable == false
PlanValidationResult.runtime_executable == false
ExecutionGraphPreview.runtime_executable == false
RuntimeDirectiveCandidate.authority == "candidate_only"
ControlKernelResult.status == "blocked"
ControlKernelResult.directives == []
ExecutionGraph.nodes == []
CommitCandidate.allowed == false
```

### 10.2 防回退断言

```text
QueryRuntime 不从 execution.execution_kind 进入 worker/direct_tool/model。
query_understanding.tool_name 不成为执行入口。
worker_plan 不直接进入 EvidenceOrchestrator。
RuntimeToolBridge 不被主链直接调用。
TaskCoordinator 不因 preview mark_running / mark_completed。
session_manager.append_messages 不绕过 CommitGate。
ResourcePolicyPreview 不被 executor 消费。
PromptManifestPreview 不保存 RuntimeDirective。
```

### 10.3 事件断言

```text
task_operation_preview 在 orchestration_control 前出现。
candidate_set_preview 在 orchestration_plan_preview 前出现。
plan_validation 在 execution_graph_preview 前出现。
adoption_candidate_preview 在 runtime_directive_candidate_preview 前出现。
adoption_block.blocked == true。
runtime_directive_block.blocked == true。
commit_gate_preview 在 orchestration_control 前出现。
orchestration_control 最终 blocked。
error.answer_channel == "orchestration_fail_closed"。
```

## 11. 切换规则

### 11.1 Shadow 阶段

```text
新编排系统只产 preview artifacts。
旧执行链不恢复。
真实主链 blocked。
```

### 11.2 只读试点条件

只有满足：

```text
PlanValidator 有硬规则。
AdoptedResourcePolicy 与 ResourcePolicyPreview 分离。
RuntimeDirective 只能由 ControlKernel / RuntimeDirectiveBuilder 生成。
OperationGate preflight 已拒绝 candidate，真实执行前必须由 OperationGate 通过。
Executor 只能消费 RuntimeDirective，不能消费旧 query plan。
CommitCandidate 默认 denied。
QueryRuntime 不再从旧 execution kind 直接执行。
```

才能进入只读执行试点。

### 11.3 Rollback 规则

如果新编排 preview 出错：

```text
回到 blocked / preview_only。
不得回旧 direct_tool / worker 执行链。
不得恢复 RuntimeToolBridge 自动推断执行。
```

## 13. 最终口径

```text
编排系统是 agent 的控制大脑。
任务系统告诉它要做什么。
操作系统告诉它能碰什么。
灵魂系统告诉它如何呈现给模型。
记忆系统给它上下文候选和写回候选。
编排系统只负责采纳、验证、建图和控制执行真相。
执行层只按 RuntimeDirective 做。
写回只由 CommitGate 放行。
```

当前最短施工口径：

```text
single_agent 编排 preview 主链已完成。
验证结果、adoption block、directive block 已可解释。
旧 query 不可达执行分支已清理。
OperationGate + directive-only executor 前置合同已完成。
最后才开放只读执行试点。
```
