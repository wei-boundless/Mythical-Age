# 任务系统重构实施计划

日期：2026-04-29  
定位：在 `00-设计原则继承与重构约束.md` 基础上，定义任务系统第一阶段的对象、边界、文件落点、接口和验证顺序。  
当前目标：先接通任务契约 preview 链路，不接真实执行。

---

## 0. 结论

任务系统第一阶段不要追求“能自动执行复杂任务”，而要先把下面这条链路做稳：

```text
UserRequest
  -> TaskContract
  -> TaskDefinition
  -> TaskBinding
  -> SkillRuntimeView
  -> ResourceRuntimeView
  -> TaskPromptContract
  -> ProjectionRequirement
  -> PromptManifestPreview
```

第一阶段完成后，系统应能回答：

```text
当前是什么任务。
它为什么被识别成这个任务。
它的默认 binding 是什么。
它能看到哪些 skill 摘要。
它需要哪些资源摘要。
它给模型看的任务契约长什么样。
它建议由什么投影姿态承载。
它为什么还不能真实执行。
```

当前不做：

```text
不接真实工具执行。
不接真实 worker 执行。
不接 durable memory 写回。
不让模型自由规划后直接执行。
不恢复旧 QueryRuntime if/else 分支。
```

---

## 1. 任务系统第一阶段定位

### 1.1 任务系统负责什么

```text
定义任务真相。
保存任务配置。
整理给模型看的任务契约。
记录任务生命周期和产物引用。
为 ControlKernel 提供可采纳的任务结构。
为 ProjectionSystem 提供低噪声输入。
```

### 1.2 任务系统不负责什么

```text
不直接执行工具。
不直接执行 worker。
不直接写 session / memory / artifact。
不直接决定最终 answer ownership。
不直接授予资源权限。
不直接拼最终 prompt。
```

### 1.3 与其他系统关系

```text
TaskSystem
  拥有任务真相和任务契约骨架。

SkillSystem
  提供当前任务可采用的方法摘要。

OperationSystem
  提供当前任务可见的资源边界摘要。

ProjectionSystem
  消费 TaskPromptContract，把它转成 SoulRuntimeView / PromptManifest。

ControlKernel
  采纳 TaskContract / TaskPromptContract / ResourcePolicy，生成 ExecutionGraph。

CommitGate
  任务完成后统一处理写回。
```

## 2. 第一阶段核心对象

### 2.1 TaskContract

当前轮任务根契约。

建议文件：

```text
backend/tasks/contracts.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class TaskContract:
    task_id: str
    session_id: str
    user_goal: str
    source: str = "user_request"
    task_family: str = "unknown"
    task_mode: str = "unknown"
    parent_task_id: str = ""
    bindings: dict[str, Any] = field(default_factory=dict)
    constraints: dict[str, Any] = field(default_factory=dict)
    requested_outputs: tuple[str, ...] = ()
    candidate_refs: tuple[str, ...] = ()
    refs: dict[str, Any] = field(default_factory=dict)
    status: str = "pending"
    authority: str = "task_contract"
```

约束：

```text
TaskContract 是任务真相，不是执行计划。
TaskContract 可以引用 candidates，但不能直接执行 candidates。
TaskContract 不保存 skill 正文、tool 实现、projection 静态卡片全文。
```

### 2.2 TaskDefinition

静态任务定义。

建议文件：

```text
backend/tasks/definitions.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class TaskDefinition:
    definition_id: str
    title: str
    task_family: str
    task_mode: str
    level: str
    goal_summary: str
    input_contract: dict[str, Any] = field(default_factory=dict)
    output_contract: dict[str, Any] = field(default_factory=dict)
    completion_criteria: tuple[str, ...] = ()
    default_skill_refs: tuple[str, ...] = ()
    default_operation_requirements: tuple[str, ...] = ()
    default_projection_role: str = ""
    review_policy: str = "optional"
    metadata: dict[str, Any] = field(default_factory=dict)
```

第一批定义：

```text
task.request_intake
task.information_search
task.local_material_read
task.information_synthesis
task.task_execution
task.inspection_and_correction
task.final_response
```

### 2.3 TaskBinding

当前任务定义的配置层。

建议文件：

```text
backend/tasks/bindings.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class TaskBindingRecord:
    binding_id: str
    definition_id: str
    enabled: bool = True
    source: str = "builtin"
    agent_profile_id: str = ""
    projection_selector: str = "task_default"
    skill_scope: tuple[str, ...] = ()
    denied_skills: tuple[str, ...] = ()
    operation_scope: tuple[str, ...] = ()
    denied_operations: tuple[str, ...] = ()
    memory_scope: str = "session_read"
    output_contract_id: str = ""
    review_policy: str = "optional"
    approval_policy: str = "default"
    guardrail_overrides: dict[str, Any] = field(default_factory=dict)
    metadata: dict[str, Any] = field(default_factory=dict)
```

约束：

```text
TaskBinding 是配置，不是权限事实。
TaskBinding 可以要求 operation_scope，但最终授权仍由 ResourcePolicy 决定。
TaskBinding 可以推荐 projection_selector，但最终运行时姿态仍由 ProjectionSystem 装配。
```

### 2.4 SkillRuntimeView

当前任务可见的方法摘要。

建议文件：

```text
backend/tasks/runtime_contracts.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class SkillRuntimeView:
    skill_id: str
    title: str
    task_reason: str
    method_summary: str
    input_boundary: str = ""
    output_boundary: str = ""
    forbidden_uses: tuple[str, ...] = ()
    required_operations: tuple[str, ...] = ()
```

约束：

```text
SkillRuntimeView 是摘要，不是 skill registry。
SkillRuntimeView 不声明工具授权。
SkillRuntimeView 不替代 TaskMethodContract。
```

### 2.5 ResourceRuntimeView

当前任务可见的资源边界摘要。

建议先放：

```text
backend/tasks/runtime_contracts.py
```

后续迁移到：

```text
backend/operations/runtime_view.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class ResourceRuntimeView:
    resource_id: str
    title: str
    capability_summary: str
    authorized: bool = False
    authorization_owner: str = "ResourcePolicy"
    risk_summary: str = ""
    denied_reason: str = ""
```

约束：

```text
ResourceRuntimeView 是给模型看的摘要。
ResourceRuntimeView.authorized 只能来自 ResourcePolicy preview。
模型看到 ResourceRuntimeView 不等于拿到执行权。
```

### 2.6 TaskPromptContract

给模型看的任务运行契约。

建议文件：

```text
backend/tasks/runtime_contracts.py
```

第一版字段：

```python
@dataclass(slots=True, frozen=True)
class TaskPromptContract:
    contract_id: str
    task_id: str
    definition_id: str
    binding_id: str
    task_section: str
    method_section: str
    resource_section: str
    projection_section: str
    output_section: str
    guardrail_section: str
    metadata: dict[str, Any] = field(default_factory=dict)
```

section 要求：

```text
task_section:
  当前任务目标、阶段、输入、完成条件。

method_section:
  当前可采用的方法摘要，来自 SkillRuntimeView。

resource_section:
  当前资源边界，来自 ResourceRuntimeView。

projection_section:
  需要什么承载姿态、表达密度、注意力重点。

output_section:
  输出格式、artifact refs、完成标准。

guardrail_section:
  禁止事项、失败处理、冲突策略、review 要求。
```

### 2.7 ProjectionRequirement

任务系统给投影系统的姿态需求。

第一版可以先不单独建文件，作为 `TaskPromptContract.projection_section` 和 metadata 输出。

后续独立为：

```text
backend/tasks/projection_requirements.py
```

字段草案：

```python
@dataclass(slots=True, frozen=True)
class ProjectionRequirement:
    task_id: str
    role_type: str
    posture_tags: tuple[str, ...] = ()
    expression_density: str = "normal"
    attention_focus: tuple[str, ...] = ()
    reason: str = ""
```

## 3. TaskMode 与定义映射

第一阶段不做复杂分类器，先用规则和显式命中。

### 3.1 TaskFamily

```text
qa
search
local_processing
execution
review
finalization
```

### 3.2 TaskMode

```text
request_intake
information_search
local_material_read
information_synthesis
task_execution
inspection_and_correction
final_response
```

### 3.3 识别规则

```text
搜索 / 查资料 / 联网 / 官方资料:
  information_search

读取 / 打开 / 总结文档 / 查看文件:
  local_material_read + information_synthesis

修改 / 实现 / 修复 / 落地:
  task_execution

检查 / 审查 / 有没有矛盾 / 风险:
  inspection_and_correction

总结 / 最终结论 / 输出方案:
  final_response
```

输出候选：

```text
TaskFamilyCandidate
TaskModeCandidate
```

采纳规则：

```text
显式用户指定 > 强关键词命中 > 上下文绑定 > 默认 request_intake。
不确定时进入 request_intake，不直接执行。
```

## 4. Preview 链路

第一阶段目标链路：

```text
build_task_contract(user_request)
  -> select_task_definition(contract)
  -> resolve_task_binding(definition)
  -> resolve_skill_runtime_views(definition, binding)
  -> resolve_resource_runtime_views(definition, binding)
  -> build_task_prompt_contract(...)
  -> build_projection_preview(...)
  -> build_prompt_manifest_preview(...)
```

### 4.1 API 预览接口

建议新增：

```text
GET /api/tasks/definitions
GET /api/tasks/definitions/{definition_id}
GET /api/tasks/definitions/{definition_id}/binding
PUT /api/tasks/definitions/{definition_id}/binding
POST /api/tasks/runtime-contract/preview
```

第一阶段可以先只做：

```text
GET definitions
GET binding
POST runtime-contract preview
```

### 4.2 Preview 输入

```json
{
  "session_id": "session-1",
  "user_goal": "读取 docs/... 并总结当前任务系统框架",
  "source": "manual_preview"
}
```

### 4.3 Preview 输出

```json
{
  "task_contract": {},
  "definition": {},
  "binding": {},
  "skill_runtime_views": [],
  "resource_runtime_views": [],
  "task_prompt_contract": {},
  "projection_requirement": {},
  "prompt_manifest_preview": {},
  "status": "preview_only"
}
```

## 5. 三个用例验收

### 5.1 联网搜索资料

用户请求：

```text
帮我联网搜索一下 Claude Code subagent 的官方资料。
```

期望：

```text
TaskDefinition:
  task.information_search

SkillRuntimeView:
  web search / source screening / evidence summary

ResourceRuntimeView:
  web_search authorized preview
  local_write denied
  shell denied

TaskPromptContract:
  强调来源、证据、未解决缺口。

ProjectionRequirement:
  evidence-first / concise / traceability
```

### 5.2 读取并总结本地文档

用户请求：

```text
读取 docs/系统规划/灵魂系统/05-讨论-20260428.md 并总结当前任务系统框架。
```

期望：

```text
Composite preview:
  local_material_read
  information_synthesis

ResourceRuntimeView:
  read_file authorized preview
  local_write denied
  shell denied

TaskPromptContract:
  读取 -> 抽取 -> 总结。
  不扩展到未读取材料之外。

ProjectionRequirement:
  analyst / structure-first / precise
```

### 5.3 修改文档后检查

用户请求：

```text
修改任务系统讨论文档，然后检查有没有前后矛盾。
```

期望：

```text
Composite preview:
  task_execution
  inspection_and_correction

ResourceRuntimeView:
  read_file authorized preview
  write_file requires_approval 或 preview_only
  shell denied 或 requires_approval

ReviewPolicy:
  required

ProjectionRequirement:
  execution stage: implementer
  review stage: risk_reviewer
```

第一阶段仍然：

```text
不真实修改文件。
只生成 staged preview。
```

## 7. 测试总原则

每个测试不只看输出文本，还要检查：

```text
task_contract.authority == "task_contract"
candidate.authority == "candidate_only"
binding.enabled == True
resource_runtime_view.authorization_owner == "ResourcePolicy"
task_prompt_contract 包含 6 个 section
prompt_manifest 记录 section 来源
preview status == preview_only
runtime_directive 不存在或 disabled
commit_candidate 默认 denied
```

---

## 8. 第一阶段完成标准

```text
1. 任务系统目录有稳定契约模型。
2. 第一批 basic task definitions 可注册。
3. 每个 definition 有默认 binding。
4. 三个真实用例可生成 runtime contract preview。
5. SkillRuntimeView 初步按任务裁剪。
6. ResourceRuntimeView 初步按任务和 binding 裁剪。
7. TaskPromptContract 成为 ProjectionSystem 输入。
8. PromptManifestPreview 可追踪 section 来源。
9. ControlKernel 仍 fail-closed，但能看到 task contract。
10. 没有真实工具执行、worker 执行、memory 写回。
```

## 9. 下一阶段入口

当第一阶段完成后，再进入操作系统重构：

```text
OperationRegistry
OperationRequirement
ResourcePolicy
ResourceRuntimeView
OperationGate
```

然后再做复合任务受控规划：

```text
CompositeTaskPolicy
CompositePlan
CompositePlanValidator
StagedTaskPromptContract
```

最后才接：

```text
ExecutionGraph -> RuntimeDirective -> ExecutionRuntime -> CommitGate
```

## 10. 当前定稿口径

```text
第一阶段不是让任务自动跑起来。
第一阶段是让任务先被正确理解、正确绑定、正确裁剪、正确呈现。
```

更短：

```text
先让任务契约闭合。
再让资源边界闭合。
最后才让执行链路闭合。
```
