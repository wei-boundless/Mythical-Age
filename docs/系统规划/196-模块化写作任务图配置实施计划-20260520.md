# 196-模块化写作任务图配置实施计划

日期：2026-05-20

## 1. 目标

配置一套新的长篇写作任务图，不再把所有节点塞进一张大图，而是采用升级后的通用任务图能力：

- 顶层编排图负责串接阶段。
- 设计初始化图负责世界观、人设、剧情与大纲入库。
- 章节批次循环图负责按通用批次契约推进 50 章。
- 收尾交付图负责整编、终审和归档。

这套配置必须使用通用任务图、GraphUnit、contract_bindings、unit_batch 和 runtime split policy，不允许新增写作专用后端私门。

## 2. 配置原则

1. 新图独立于旧 `graph.writing.simple_novel` 和旧单体 `graph.writing_team.long_novel`。
2. 节点职责通过已有 projection 和 task record 继承，不把开发说明塞给 Agent。
3. 章节数量、批次大小、审核与合并规则进入 `contract_bindings`。
4. 顶层父图只运行子图，不重复展开所有业务节点。
5. 子图必须是已发布图，父图 GraphUnit 编译时可以找到并展开。

## 3. 资产命名

- 顶层图：`graph.writing.modular_novel.master`
- 设计初始化子图：`graph.writing.modular_novel.design_init`
- 章节批次子图：`graph.writing.modular_novel.chapter_cycle`
- 收尾交付子图：`graph.writing.modular_novel.finalize`
- 通信协议：`protocol.writing.modular_novel`
- 图级契约：`contract.writing.modular_novel.*`

## 4. 执行步骤

1. 新增配置脚本 `scripts/configure_writing_modular_novel_graph.py`。
2. 复用现有写作团队 agent、projection、specific task records 和基础合同，新增模块化图与 GraphUnit 交接契约。
3. 给章节正文节点配置：
   - `unit_kind=chapter`
   - `requested_count=50`
   - `batch_size=5`
   - `range_start=1`
   - `batch_acceptance_policy=review_then_commit`
   - `merge_policy=wait_all_committed`
4. 运行配置脚本写入 `storage/tasks/*.json`。
5. 新增回归测试验证：
   - 四张图均存在且发布。
   - 父图编译出 3 个 GraphUnit。
   - 章节图编译出 10 个批次。
   - 批次配置只来自 `contract_bindings`。
6. 运行相关测试。

## 5. 完成标准

配置完成后，任务系统可以选择新的顶层图 `graph.writing.modular_novel.master`，发布执行包能看到三个子图运行计划；章节子图能看到 50 章拆成 10 个 5 章批次，并按审核后提交、全部提交后合并的通用契约执行。
