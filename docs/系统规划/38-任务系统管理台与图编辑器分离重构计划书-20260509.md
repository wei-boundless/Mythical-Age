# 38-任务系统管理台与图编辑器分离重构计划书

日期：2026-05-09

## 1. 判断

任务管理和任务图编辑是两个不同工作台，不应该继续混在同一个左侧任务域布局里：

- 任务管理台负责任务域、任务、契约、任务包、预检、RunLoop 等任务资产。
- 任务图编辑器负责打开一个任务，并构造该任务的 Agent 能力拓扑。
- 图编辑器不需要常驻任务域侧栏，否则编辑时左边一直被任务域占用，既浪费空间，也让用户误以为图编辑是在管理任务域。
- 图编辑器需要自己的任务加载器。用户进入编辑器后可以直接加载任务，然后编辑该任务的图。

## 2. 目标结构

任务系统前端保留两个顶层入口：

1. `任务管理`
   - 左侧：任务域。
   - 中间：当前任务域设置与任务列表。
   - 右侧：当前任务配置。
   - 入口动作：打开图编辑器。

2. `图编辑器`
   - 全宽编辑器，不渲染任务域 rail。
   - 顶部：任务加载器、任务图草稿选择、保存/发布状态。
   - 左侧：节点/模板/复用切换面板。
   - 中间：画布。
   - 右侧：检查器。

## 3. 编辑器任务加载逻辑

图编辑器内部提供任务加载器：

- 可选择全部任务，按任务域分组显示。
- 选择任务后：
  - 只设置编辑器自己的打开任务。
  - 只在编辑器内部定位该任务所属任务域。
  - 自动加载该任务族群下的任务图草稿。
  - 如果已有草稿，打开第一个或上次选中的图草稿。
  - 如果无草稿，显示空画布和“新任务图草稿”入口。
- 任务加载器只负责打开任务，不负责编辑任务定义。
- 任务定义、契约、RunLoop 等仍在任务管理台编辑。
- 编辑器不得读取任务管理台的当前任务域、当前任务、当前图草稿作为默认编辑对象。
- 任务管理台的选择态和图编辑器的打开态分离；管理台切换任务域不会改变编辑器正在编辑的任务图。
- 未在编辑器里显式打开任务时，编辑器保持空画布和空草稿状态。

## 4. 编辑器操作原则

- 默认点击节点只选择节点。
- 默认点击边只选择边。
- 连线必须显式进入连线模式。
- 添加节点必须来自明确按钮。
- 组合已有任务是可选复用能力，不是主流程。
- 空画布只显示文字和操作入口，不显示占位图案。
- 常驻功能只放高频动作；低频能力通过切换面板、折叠区和检查器承载。
- 编辑器只服务任务图编辑本身，不堆摘要卡、说明文案、状态故事和管理仓库信息。
- 管理页负责仓库式管理，编辑器内部只保留打开任务、选择草稿、构造节点/边、检查器和保存。

## 5. 实施阶段

### 阶段 1：外层布局分离

- `taskLayer` 文案改为 `任务管理 | 图编辑器`。
- `任务管理` 保留任务域 rail。
- `图编辑器` 不再渲染任务域 rail。
- `boundary-workbench` 根据当前顶层入口分支渲染。

### 阶段 2：图编辑器任务加载器

- 新增编辑器顶部加载条。
- 任务选择器使用全部任务。
- 选择任务时只更新编辑器专属状态，不同步 `selectedTaskId`、`selectedDomainId`、`selectedCoordinationId`。
- 新增编辑器专属 `editorTaskId`、`editorCoordinationId` 及其派生的任务域、任务图草稿、模板、协议。
- 新任务图草稿仍使用正式 API 和现有 `createCoordinationDraft`。

### 阶段 3：编辑器操作台

- 左侧改为 `节点 / 模板 / 复用` 切换面板。
- 节点面板默认打开。
- 复用已有任务默认不是主入口。
- 画布工具条只保留选择、连线、取消连线、适配视图。
- 删除编辑器摘要卡和元信息卡。
- 删除画布下方快速操作说明区。
- 将保存动作收敛到编辑器标题栏，不再分散成多块说明面板。

### 阶段 4：连线与检查器

- 引入明确编辑模式：`select | connect`。
- `linkingFromNodeId` 只在 connect 模式下生效。
- 右侧检查器按图级、节点级、边级互斥显示。
- 高级策略折叠。

### 阶段 5：验证

- `npm run lint` 通过。
- Edge + Playwright 检查：
  - 任务管理页仍可管理任务域/任务。
  - 图编辑器无任务域左侧 rail。
  - 图编辑器可直接加载任务。
  - 空图没有占位图案。
  - 选择节点不会连线。
  - 显式进入连线模式后才能创建边。
