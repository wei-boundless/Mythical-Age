# 简单长期记忆闭环修复计划书

日期：2026-05-11

## 目标

先不做复杂长期记忆架构，只把当前系统已有的长期记忆闭环修正到可用：

1. 用户明确问“你记得我的偏好吗 / 之后怎么称呼我 / 信息不足时怎么处理”时，必须进入 memory 路由，不走 RAG。
2. memory 路由必须读取 `long_term` 层，并把相关长期记忆放进运行时上下文。
3. runtime 必须声明 `op.memory_read`，让主 Agent 权限和任务合同一致。
4. 显式“记住：……”类偏好能被写入长期记忆候选，至少覆盖当前长测需要的稳定用户偏好。

## 当前问题

### 1. 识别层漏掉常见问法

`backend/understanding/memory_intent.py` 只覆盖了“你记得 / 我的偏好 / 复杂问题怎么回答”等表达，漏掉：

- “你之后应该怎么称呼我？”
- “如果信息不足，你应该怎么处理？”

这会导致问题走 `rag`，后续长期记忆永远不会被读取。

### 2. memory 任务没有强制长期层

`task.memory_recall` 能被选中，但默认任务记忆请求仍可能只包含 `conversation/state`，最终 `long_term_candidate_count = 0`。

### 3. runtime 操作声明缺口

`AgentRuntimeChainAssembler._operation_ids_for_runtime()` 没有把 memory 路由映射成 `op.memory_read`，导致权限、任务定义、运行时声明不一致。

### 4. 写入侧对“信息不足先说明缺什么”不稳定

当前长期记忆库已有“称呼我为岩”“复杂问题先给结论”，但没有“信息不足先明确缺什么，不要直接猜”。这说明显式写入链对部分中文偏好句仍不稳。

## 实施步骤

1. 扩展 memory intent：
   - 增加称呼类语义读取。
   - 增加信息不足处理类语义读取。
   - 两类都归入 `user/preference`。
2. 修正 memory 任务的记忆请求：
   - memory route / direct_memory / `task.memory_recall` 默认申请 `conversation/state/long_term`。
   - `allow_long_term_memory = true`。
3. 修正 runtime 操作声明：
   - memory route 返回 `op.memory_read`。
4. 修正显式写入策略：
   - “记住：如果信息不足，先明确告诉我缺什么，不要直接猜。”应生成 user preference 长期记忆。
5. 增加回归：
   - intent/route 覆盖称呼、信息不足、复杂问题。
   - runtime memory view 对称呼类问题有 long-term candidate。
   - runtime required ops 包含 `op.memory_read`。
   - 写入候选能覆盖信息不足偏好。
