# 109-工作记忆隔离清扫与监控增强计划-20260516

## 1. 目标

围绕 TaskGraph 运行时的工作记忆系统，完成三项结构性增强：

1. 收紧节点默认读取边界，避免同一任务图内节点无意共享工作记忆导致污染。
2. 在任务结束后执行自动清扫与存储瘦身，避免工作记忆长期堆积。
3. 在运行监控中显式展示节点实际读取了哪些工作记忆项，提升可观测性与排障能力。

## 2. 当前问题

### 2.1 默认读取边界不够硬

当前工作记忆虽然以 `task_run_id` 作为首要隔离键，但在同一 `task_run_id` 内：

- 如果节点声明了较宽的读取策略，或者没有显式声明足够严格的边界，
- `select_for_node(...)` 可能读取到不该被当前节点消费的已接受工作记忆项，
- 从而形成同图内部的隐式污染。

### 2.2 任务结束后只做生命周期归档，不做清扫

当前 `WorkingMemoryFinalizer` 会把记录转为 `archived / discarded / conflicted` 等状态，但：

- 不会删除明显无保留价值的终态项，
- 不会执行存储层优化，
- 长时间运行后数据库会不断膨胀。

### 2.3 监控看不到“读了什么”

运行态已经记录了 `WorkingMemoryReadLog`，但 TaskGraph 监控主要展示：

- `memory_write`
- `memory_commit`
- `memory_finalize`

却缺少最关键的：

- 当前节点读了哪些工作记忆项，
- 是否被拒读，
- 拒读原因是什么。

## 3. 目标策略

### 3.1 默认隔离策略

对工作记忆读取执行“默认最小授权”：

1. 若节点未显式声明 `readable_scopes`，默认只允许 `node_scope`。
2. `handoff_only` 可见性不得被默认读取，只有在策略显式允许时才可见。
3. `private_to_node` 继续只允许同 `owner_node_id` 读取。
4. `private_to_agent` 继续只允许同 `writer_agent_id` 读取。
5. 更宽范围的读取必须来自显式配置，而不是隐式放开。

### 3.2 自动清扫策略

在任务完成后：

1. 继续保留 finalization report。
2. 对 `draft / proposed / discarded / superseded` 等无保留价值项执行物理清扫。
3. 对数据库执行优化操作，降低长期堆积。
4. 清扫行为写入 finalization 结果，保证可追踪。

### 3.3 监控增强策略

在每个阶段构建工作记忆上下文时：

1. 生成 `memory_read` 运行操作记录。
2. 记录：
   - `read_log_id`
   - `selected_working_memory_refs`
   - `selected_item_previews`
   - `excluded_working_memory_refs`
   - `denied_reason`
3. 前端 TaskGraph 监控仅展示近期读取，并允许按节点识别。

## 4. 实施步骤

### 4.1 后端读取隔离

修改：

- `backend/memory_system/working_memory_service.py`

实施：

1. 增加默认 `readable_scopes=node_scope`。
2. 增加默认 `readable_visibilities` 约束。
3. 对 `handoff_only` 增加显式放行要求。

### 4.2 后端运行态读操作记录

修改：

- `backend/orchestration/runtime_loop/langgraph_coordination_runtime.py`

实施：

1. 在阶段工作记忆选择后生成标准化 `memory_read` operation。
2. 写入 `working_memory_operations`，供监控直接消费。

### 4.3 后端终态清扫

修改：

- `backend/memory_system/working_memory_store.py`
- `backend/memory_system/working_memory_finalizer.py`
- `backend/orchestration/runtime_loop/langgraph_coordination_runtime.py`

实施：

1. 增加按 `task_run_id` 的终态项清扫。
2. 增加数据库优化。
3. 将清扫摘要写入 finalization 结果。

### 4.4 前端监控增强

修改：

- `frontend/src/components/chat/TaskGraphRunPanel.tsx`

实施：

1. 让 `workingMemoryOperations` 支持 `memory_read`。
2. 展示读取 refs、拒读原因、节点归属。

## 5. 验证要求

至少覆盖以下回归点：

1. 无显式读取范围时，默认只能读 `node_scope`。
2. `handoff_only` 未显式授权时不可读。
3. 任务完成后，低价值终态项会被清扫。
4. TaskGraph 运行态会记录 `memory_read` 操作。
5. 前端监控能识别并展示 `memory_read`。
