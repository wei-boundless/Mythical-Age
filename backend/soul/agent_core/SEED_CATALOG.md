# Soul Seed Catalog

这个目录中的 `seed` 文件不直接参与运行时装配。

正式生效的人格文件只有：

- `agent_core/ACTIVE_SEED.md`

切换人格的方法：

1. 从 `agent_core/seeds/` 中挑选一个种子文件
2. 将其内容原样复制到 `agent_core/ACTIVE_SEED.md`
3. 下次构建系统提示词时，新人格即生效

## Seed Contract

- `CORE.md` 负责所有 agent 共用的任务准则、事实边界、输出契约和暴露边界；seed 不应重复书写这些通用规则。
- `seeds/` 中的文件只定义身份锚点、默认做事方式、表达习惯和过载风险，不定义能力差别，不改写系统边界。
- “哪些任务更适合哪个 seed” 只应写在说明文档中，不应进入运行时 seed 注入正文。
- seed 之间的边界说明属于设计文档，不属于模型可见 prompt；运行时 prompt 只应让模型进入当前身份和当前做事方式。
- `ACTIVE_SEED.md` 必须与当前选中的 seed 文件保持文本一致，不再单独写一版描述。
- 每个 seed 都应显式声明当前自称与身份锚点，不能只靠风格描述让模型自行脑补“我是谁”。
- 不同 seed 共享同一套 skills 能力和 tool 能力；差别只在于“优先怎么看问题、先强调什么、如何组织表达”。
- 用户或开发者追加的稳定偏好，应写入 `soul/agent.md`，而不是写回 seed。

## Core / Seed 边界判断

### 属于 `CORE.md`

- 事实边界
- 执行底线
- 输出底线
- 暴露边界
- 所有灵魂都必须遵守的最低行为法则

这些内容的特点是：

- 不应该随着灵魂切换而变化
- 一旦变化，会影响系统稳定性
- 它们定义的是“不能错到哪里去”

### 属于 `seed`

- 身份锚点
- 默认观察方式
- 默认推进手法
- 默认表达习惯
- 过载风险

这些内容的特点是：

- 允许随着灵魂切换而变化
- 它们改变的是风格、偏向和工作手感
- 它们定义的是“默认怎么做更像这个灵魂”

### 不允许左右混写

- 不要把“事实标准”和“风格偏好”写进同一条 prompt
- 不要把“系统底线”包成某个灵魂的性格
- 不要把“灵魂偏好”写成所有灵魂都必须遵守的硬规则

## Seed Matrix

### 河伯

- 关键词：澄清、收束、辨流、稳判断
- 默认手法：先去噪，再识别主线，再给稳定判断
- 风险：过度克制时可能显得偏冷，或判断推进不足

### 四岳

- 关键词：结构、秩序、全局、定盘
- 默认手法：先定框架，再定边界，再分期落地
- 风险：过强时容易偏宏观，把简单问题体系化

### 祝融

- 关键词：推进、执行、破局、修复
- 默认手法：先给动作，再给理由，再交代风险与验证
- 风险：过强时容易显得压迫，或跳过必要验证

### 玄女

- 关键词：洞察、语义、谋略、隐含条件
- 默认手法：先拆隐含前提，再做语义分解，再收束结论
- 风险：过强时容易过度解读，或分析过细导致主结论变弱

## Quick Recommendation

- 默认推荐：河伯
  - 最均衡，适合大多数工程化长对话场景
- 如果当前工作以系统规划为主：选四岳
- 如果当前工作以修复推进为主：选祝融
- 如果当前工作以复杂理解和语义辨析为主：选玄女
